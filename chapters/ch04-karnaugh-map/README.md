## 4.1 บทนำ: จากตารางความจริงสู่ผังคาร์โนห์

ในบทก่อนหน้า นักศึกษาได้ใช้พีชคณิตบูลีน (Boolean Algebra) เพื่อลดรูปสมการตรรกะด้วยกฎ เช่น absorption, De Morgan และ distributive law

วิธีนั้นถูกต้องทางคณิตศาสตร์ แต่เมื่อฟังก์ชันมีหลายตัวแปร การเลือกกฎที่เหมาะสมอาจไม่ชัดเจน และนักศึกษามักลดรูปได้ต่างกันโดยไม่รู้ว่าคำตอบใดสั้นที่สุด

**ผังคาร์โนห์ (Karnaugh Map: K-map)** เป็นเครื่องมือเชิงภาพสำหรับลดรูปฟังก์ชันบูลีน โดยนำค่าจากตารางความจริงมาจัดวางบนตารางที่ช่องติดกันต่างกันเพียง 1 ตัวแปร

เมื่อช่องที่เป็น 1 อยู่ติดกัน หมายความว่า minterm เหล่านั้นมีตัวแปรบางตัวเปลี่ยนค่าไปมาแต่เอาต์พุตยังคงเป็น 1 ดังนั้นตัวแปรที่เปลี่ยนจึงไม่จำเป็นต่อการกำหนดผลลัพธ์และสามารถตัดออกจากพจน์ได้

ตัวอย่างเช่น minterm สองตัวนี้ต่างกันเฉพาะตัวแปร `B`

```text
A'B C  +  A'B'C
```

เมื่อจัดกลุ่มบน K-map จะเห็นว่า `B` เปลี่ยนจาก 1 เป็น 0 แต่ `A'` และ `C` คงเดิม

ดังนั้นพจน์รวมลดได้เป็น

```text
A'C
```

จุดสำคัญของ K-map คือไม่ได้เป็นเพียงตารางสวยงาม แต่เป็นการจัดตำแหน่งข้อมูลให้ "การลดรูป" มองเห็นได้จากความติดกันของช่อง

ในรายวิชานี้ K-map ใช้เพื่อลดจำนวนเกตในวงจรเชิงผสม (Combinational Logic Circuit) ก่อนนำไปสร้างใน Logisim Evolution หรือประกอบด้วยไอซีตระกูล 74xx บนเบรดบอร์ด

หากลดรูปได้ดี วงจรจะใช้เกตน้อยลง ใช้ไอซีน้อยลง มีระดับชั้นของสัญญาณ (logic level) น้อยลง และมักมี propagation delay ต่ำลง

## 4.2 ทำไมช่องของ K-map ต้องเรียงตาม Gray code

K-map ไม่ใช้ลำดับเลขฐานสองปกติแบบ `00, 01, 10, 11`

K-map ใช้ลำดับ **Gray code** ซึ่งเรียงเป็น `00, 01, 11, 10`

เหตุผลคือช่องที่อยู่ติดกันต้องต่างกันเพียง 1 บิตเท่านั้น

เปรียบเทียบลำดับ 2 บิตดังนี้

| ลำดับ | ช่องที่ 1 | ช่องที่ 2 | ช่องที่ 3 | ช่องที่ 4 | ปัญหา |
|---|---|---|---|---|---|
| Binary ปกติ | 00 | 01 | 10 | 11 | จาก 01 ไป 10 ต่างกัน 2 บิต |
| Gray code | 00 | 01 | 11 | 10 | ทุกคู่ที่ติดกันต่างกัน 1 บิต |

หากใช้ binary ปกติ ช่อง `01` กับ `10` จะติดกันในตาราง แต่จริง ๆ แล้วตัวแปรสองตัวเปลี่ยนพร้อมกัน จึงไม่สามารถรวมเป็นกลุ่มเพื่อตัดตัวแปรเพียงตัวเดียวได้

ดังนั้น K-map จึงต้องจัดแถวและคอลัมน์ตาม Gray code เสมอ

สำหรับตัวแปรสองตัว `B C` ลำดับคอลัมน์ที่ถูกต้องคือ

```text
BC: 00  01  11  10
```

ช่องซ้ายสุดและขวาสุดก็ติดกันด้วย เพราะ `00` กับ `10` ต่างกันเฉพาะ `B`

นี่คือที่มาของการ **wrap ขอบ** หรือการมองว่าขอบซ้ายต่อกับขอบขวา และขอบบนต่อกับขอบล่าง

## 4.3 ความติดกันและการ wrap ขอบ

ช่องสองช่องใน K-map ถือว่า **ติดกัน (adjacent)** เมื่อค่าตัวแปรของสองช่องต่างกันเพียง 1 ตัว

ช่องติดกันได้ 4 ทิศทาง ได้แก่ ซ้าย ขวา บน ล่าง

ช่องทแยงมุมไม่ถือว่าติดกัน เพราะมักต่างกัน 2 ตัวแปร

ใน K-map 4 ตัวแปร ขอบของแผนที่เชื่อมถึงกันเหมือนพื้นผิวทรงโดนัทในเชิงตรรกะ

หมายความว่า

- คอลัมน์ซ้ายสุดติดกับคอลัมน์ขวาสุด
- แถวบนสุดติดกับแถวล่างสุด
- มุมทั้งสี่สามารถอยู่ในกลุ่มเดียวกันได้ หากค่าที่ต้องจับกลุ่มอยู่ครบทั้งสี่มุม

ตัวอย่างความติดกันแบบ wrap ใน K-map 4 ตัวแปร

```text
          CD
          00  01  11  10
AB  00 |  1   0   0   1   <- ช่องซ้ายสุดกับขวาสุดติดกัน
    01 |  0   0   0   0
    11 |  0   0   0   0
    10 |  1   0   0   1   <- แถวบนสุดกับล่างสุดติดกัน
```

ช่องมุมทั้งสี่คือ minterm `m0, m2, m8, m10`

ทั้งสี่ช่องรวมเป็นกลุ่มขนาด 4 ได้

ค่าคงที่ของกลุ่มนี้คือ `B=0` และ `D=0`

ดังนั้นพจน์ที่ได้คือ

```text
B'D'
```

นักศึกษาที่ไม่ wrap ขอบจะมองเห็นเป็นกลุ่มขนาด 2 หลายกลุ่ม ทำให้ได้สมการยาวกว่าจำเป็น

## 4.4 การระบุ minterm และ maxterm บน K-map

ก่อนจับกลุ่ม ต้องใส่ค่า 1, 0 หรือ X ลงในช่องที่ตรงกับหมายเลข minterm ให้ถูกต้อง

สำหรับฟังก์ชันแบบผลรวมของ minterm ใช้สัญลักษณ์

```text
F(A,B,C) = Sigma m(1,2,3,5,7)
```

หมายความว่าเอาต์พุต `F=1` ที่ minterm หมายเลข 1, 2, 3, 5 และ 7

ช่องอื่นที่ไม่ได้ระบุเป็น 0

สำหรับฟังก์ชันแบบผลคูณของ maxterm ใช้สัญลักษณ์

```text
F(A,B,C) = Pi M(0,4,6)
```

หมายความว่าเอาต์พุต `F=0` ที่ maxterm หมายเลข 0, 4 และ 6

ช่องอื่นเป็น 1

การระบุเลข minterm ใช้ค่าฐานสองของตัวแปรเรียงจากซ้ายไปขวา

ถ้าใช้ตัวแปร `A B C D`

```text
m0  = 0000
m1  = 0001
m2  = 0010
...
m15 = 1111
```

## 4.5 K-map 2 ตัวแปร

K-map 2 ตัวแปรใช้แถวแทน `A` และคอลัมน์แทน `B`

ผังเปล่าพร้อมตำแหน่ง minterm คือ

```text
        B
        0    1
A 0 |  m0   m1
  1 |  m2   m3
```

ถ้าเขียนเป็นค่าตัวแปรจะได้

```text
        B
        0      1
A 0 |  A'B'   A'B
  1 |  AB'    AB
```

K-map 2 ตัวแปรมีขนาดเล็ก แต่ยังใช้กฎเดียวกับ K-map ขนาดใหญ่

กลุ่มที่เป็นไปได้คือ 1 ช่อง, 2 ช่อง หรือ 4 ช่อง

ตัวอย่างกลุ่ม 2 ช่องในแถวบน

```text
        B
        0    1
A 0 |  1    1
  1 |  0    0
```

แถวบนมี `A=0` คงที่ ส่วน `B` เปลี่ยนจาก 0 เป็น 1

ดังนั้นฟังก์ชันคือ

```text
F = A'
```

ตัวอย่างกลุ่ม 2 ช่องในคอลัมน์ซ้าย

```text
        B
        0    1
A 0 |  1    0
  1 |  1    0
```

คอลัมน์ซ้ายมี `B=0` คงที่ ส่วน `A` เปลี่ยนจาก 0 เป็น 1

ดังนั้นฟังก์ชันคือ

```text
F = B'
```

## 4.6 K-map 3 ตัวแปร

K-map 3 ตัวแปรนิยมให้ `A` เป็นแถว และ `B C` เป็นคอลัมน์

คอลัมน์ต้องเรียงตาม Gray code คือ `00, 01, 11, 10`

ผังเปล่าพร้อมตำแหน่ง minterm คือ

```text
          BC
          00   01   11   10
A  0 |   m0   m1   m3   m2
   1 |   m4   m5   m7   m6
```

สังเกตว่า `m2` อยู่คอลัมน์ขวาสุด ไม่ใช่ช่องที่สาม เพราะคอลัมน์ที่สามคือ `BC=11`

ตำแหน่งนี้เป็นข้อผิดพลาดที่พบบ่อยมากในการสอบ

ถ้าต้องการตรวจตำแหน่ง ให้แปลงเลข minterm เป็นฐานสอง 3 บิต

ตัวอย่าง

```text
m5 = 101  -> A=1, B=0, C=1 -> แถว A=1, คอลัมน์ BC=01
m6 = 110  -> A=1, B=1, C=0 -> แถว A=1, คอลัมน์ BC=10
m7 = 111  -> A=1, B=1, C=1 -> แถว A=1, คอลัมน์ BC=11
```

กลุ่มใน K-map 3 ตัวแปรมีขนาดเป็น 1, 2, 4 หรือ 8 ช่อง

ถ้าจับกลุ่ม 4 ช่อง จะเหลือตัวแปรเพียง 1 ตัว

ถ้าจับกลุ่ม 2 ช่อง จะเหลือตัวแปร 2 ตัว

ถ้าจับกลุ่ม 1 ช่อง จะเหลือตัวแปรครบ 3 ตัวและมักไม่ใช่คำตอบที่ดี เว้นแต่ไม่มีทางรวมกับช่องอื่น

## 4.7 K-map 4 ตัวแปร

K-map 4 ตัวแปรนิยมให้ `A B` เป็นแถว และ `C D` เป็นคอลัมน์

ทั้งแถวและคอลัมน์ต้องเรียงตาม Gray code

ผังเปล่าพร้อมตำแหน่ง minterm คือ

```text
              CD
              00    01    11    10
AB  00 |     m0    m1    m3    m2
    01 |     m4    m5    m7    m6
    11 |     m12   m13   m15   m14
    10 |     m8    m9    m11   m10
```

การอ่านตำแหน่งทำโดยนำค่าของแถวและคอลัมน์มาต่อกัน

ตัวอย่าง

```text
แถว AB=11, คอลัมน์ CD=01  -> ABCD=1101 -> m13
แถว AB=10, คอลัมน์ CD=11  -> ABCD=1011 -> m11
แถว AB=01, คอลัมน์ CD=10  -> ABCD=0110 -> m6
```

กลุ่มใน K-map 4 ตัวแปรมีขนาด 1, 2, 4, 8 หรือ 16 ช่อง

ถ้าจับกลุ่ม 8 ช่อง จะเหลือตัวแปรเพียง 1 ตัว

ถ้าจับกลุ่ม 4 ช่อง จะเหลือตัวแปร 2 ตัว

ถ้าจับกลุ่ม 2 ช่อง จะเหลือตัวแปร 3 ตัว

ถ้าจับกลุ่ม 1 ช่อง จะเหลือตัวแปร 4 ตัว

หลักการนี้มาจากจำนวนตัวแปรที่เปลี่ยนภายในกลุ่ม

ทุกครั้งที่ขนาดกลุ่มเพิ่มเป็นสองเท่า จะมีตัวแปรอีกหนึ่งตัวถูกตัดออก

| จำนวนตัวแปร | ขนาดกลุ่ม | จำนวนตัวแปรที่เหลือในพจน์ |
|---:|---:|---:|
| 4 | 1 | 4 |
| 4 | 2 | 3 |
| 4 | 4 | 2 |
| 4 | 8 | 1 |
| 4 | 16 | 0 |

ถ้าทั้ง 16 ช่องเป็น 1 ฟังก์ชันจะเท่ากับ 1

ถ้าทั้ง 16 ช่องเป็น 0 ฟังก์ชันจะเท่ากับ 0

## 4.8 K-map 5 ตัวแปรด้วยสองผังซ้อน

K-map 5 ตัวแปรไม่สะดวกที่จะวาดเป็นตารางสองมิติผืนเดียว

วิธีมาตรฐานคือใช้ K-map 4 ตัวแปรสองผังซ้อนกัน

ให้ตัวแปรแรกคือ `A`

ผังซ้ายหรือผังแรกแทน `A=0`

ผังขวาหรือผังที่สองแทน `A=1`

แถวใช้ `B C` และคอลัมน์ใช้ `D E`

ผังสำหรับ `A=0`

```text
A = 0
              DE
              00    01    11    10
BC  00 |     m0    m1    m3    m2
    01 |     m4    m5    m7    m6
    11 |     m12   m13   m15   m14
    10 |     m8    m9    m11   m10
```

ผังสำหรับ `A=1`

```text
A = 1
              DE
              00    01    11    10
BC  00 |     m16   m17   m19   m18
    01 |     m20   m21   m23   m22
    11 |     m28   m29   m31   m30
    10 |     m24   m25   m27   m26
```

ช่องที่ตำแหน่งเดียวกันในสองผังถือว่าติดกัน เพราะต่างกันเฉพาะตัวแปร `A`

ตัวอย่างเช่น `m3` ในผัง `A=0` ติดกับ `m19` ในผัง `A=1`

ทั้งคู่มีค่า `B C D E = 0011` เหมือนกัน และต่างกันเฉพาะ `A`

ดังนั้นถ้า `m3` และ `m19` เป็น 1 สามารถรวมเป็นกลุ่ม 2 ช่องข้ามผังได้

ถ้าทั้งสองผังมีสี่เหลี่ยมตำแหน่งเดียวกันขนาด 4 ช่อง สามารถรวมเป็นกลุ่มขนาด 8 ได้

การจับกลุ่มใน K-map 5 ตัวแปรจึงต้องมองทั้ง

- ความติดกันภายในผังเดียวกัน
- การ wrap ขอบภายในแต่ละผัง
- ความติดกันระหว่างสองผังในตำแหน่งเดียวกัน

## 4.9 กฎการจับกลุ่มบน K-map

การจับกลุ่มมีเป้าหมายเพื่อสร้างพจน์ที่สั้นที่สุดและครอบคลุมค่าเอาต์พุตที่ต้องการครบถ้วน

กฎที่ต้องใช้มีดังนี้

1. กลุ่มต้องมีจำนวนช่องเป็นกำลังของ 2 เท่านั้น ได้แก่ 1, 2, 4, 8, 16 หรือ 32 ในกรณี 5 ตัวแปร
2. กลุ่มต้องเป็นสี่เหลี่ยมผืนผ้าหรือสี่เหลี่ยมจัตุรัสบนโครง Gray code
3. กลุ่มรูปตัว L, รูปตัว T หรือรูปทแยงมุมใช้ไม่ได้
4. ควรจับกลุ่มให้ใหญ่ที่สุดก่อน เพราะกลุ่มใหญ่ตัดตัวแปรได้มากกว่า
5. ทุกช่องที่เป็น 1 ต้องถูกครอบคลุมอย่างน้อยหนึ่งครั้งเมื่อทำ minimal SOP
6. ทุกช่องที่เป็น 0 ต้องถูกครอบคลุมอย่างน้อยหนึ่งครั้งเมื่อทำ minimal POS
7. กลุ่มซ้อนทับกันได้ ถ้าช่วยให้ได้กลุ่มใหญ่ขึ้นหรือจำเป็นต่อการครอบคลุมช่องสำคัญ
8. ช่อง don't care `X` ใช้รวมกลุ่มได้เมื่อช่วยลดรูป แต่ไม่จำเป็นต้องครอบคลุมทุก X
9. ห้ามจับกลุ่มที่มี 0 ปนในกลุ่ม SOP
10. ห้ามจับกลุ่มที่มี 1 ปนในกลุ่ม POS

ตัวอย่างกลุ่มที่ถูกต้องใน K-map 4 ตัวแปร

```text
กลุ่ม 4 ช่องแนวนอนในแถวเดียว

              CD
              00  01  11  10
AB  00 |      1   1   1   1
    01 |      0   0   0   0
    11 |      0   0   0   0
    10 |      0   0   0   0
```

แถว `AB=00` คงที่ ส่วน `C,D` เปลี่ยนทั้งหมด

พจน์คือ

```text
A'B'
```

ตัวอย่างกลุ่มที่ผิด

```text
              CD
              00  01  11  10
AB  00 |      1   1   0   0
    01 |      1   0   0   0
    11 |      0   0   0   0
    10 |      0   0   0   0
```

สามช่องที่เป็น 1 ดูเหมือนรูปตัว L แต่จับเป็นกลุ่ม 3 ช่องไม่ได้ เพราะ 3 ไม่ใช่กำลังของ 2 และไม่ใช่สี่เหลี่ยมบน K-map

ต้องแยกเป็นกลุ่ม 2 ช่องสองกลุ่มที่ overlap กันที่ `m0`

## 4.10 วิธีอ่านพจน์จากกลุ่ม

เมื่อได้กลุ่มแล้ว ให้ดูค่าตัวแปรภายในกลุ่ม

ตัวแปรใดคงที่เป็น 1 ให้เขียนตัวแปรนั้นแบบไม่ complement

ตัวแปรใดคงที่เป็น 0 ให้เขียนตัวแปรนั้นแบบ complement

ตัวแปรใดเปลี่ยนค่าในกลุ่ม ให้ตัดออก

ตัวอย่างกลุ่มใน K-map 4 ตัวแปร

```text
              CD
              00  01  11  10
AB  00 |      0   0   0   0
    01 |      0   1   1   0
    11 |      0   1   1   0
    10 |      0   0   0   0
```

กลุ่มคือสี่ช่องกลาง

แถวที่ใช้คือ `AB=01` และ `AB=11`

ดังนั้น `A` เปลี่ยนจาก 0 เป็น 1 แต่ `B=1` คงที่

คอลัมน์ที่ใช้คือ `CD=01` และ `CD=11`

ดังนั้น `C` เปลี่ยนจาก 0 เป็น 1 แต่ `D=1` คงที่

พจน์ของกลุ่มคือ

```text
BD
```

อีกตัวอย่างหนึ่ง กลุ่มมุมทั้งสี่

```text
              CD
              00  01  11  10
AB  00 |      1   0   0   1
    01 |      0   0   0   0
    11 |      0   0   0   0
    10 |      1   0   0   1
```

แถวที่ใช้คือ `AB=00` และ `AB=10`

ดังนั้น `A` เปลี่ยน แต่ `B=0` คงที่

คอลัมน์ที่ใช้คือ `CD=00` และ `CD=10`

ดังนั้น `C` เปลี่ยน แต่ `D=0` คงที่

พจน์คือ

```text
B'D'
```

## 4.11 Minimal SOP: จับกลุ่มช่อง 1

**Minimal SOP (Minimal Sum of Products)** คือรูปผลรวมของพจน์ AND ที่ลดรูปแล้ว

การทำ SOP ด้วย K-map ให้จับกลุ่มช่องที่เป็น 1

ขั้นตอนมีดังนี้

1. ใส่ 1 ในช่อง minterm ที่กำหนด
2. ใส่ 0 ในช่องอื่น หากไม่มี don't care
3. หา 1 ที่สามารถรวมเป็นกลุ่มใหญ่สุดได้
4. ใช้การ wrap ขอบหากทำให้กลุ่มใหญ่ขึ้น
5. อนุญาตให้ overlap หากจำเป็นหรือทำให้พจน์สั้นลง
6. เขียนพจน์จากแต่ละกลุ่มโดยเก็บเฉพาะตัวแปรที่คงที่
7. OR พจน์ทุกกลุ่มเข้าด้วยกัน

ข้อควรระวังคือ minimal SOP ไม่ได้แปลว่าต้องใช้กลุ่มจำนวนน้อยที่สุดเสมอไป

ต้องพิจารณาทั้งจำนวนพจน์และจำนวน literal ในแต่ละพจน์

อย่างไรก็ตามสำหรับ K-map ขนาด 2-4 ตัวแปร การจับกลุ่มใหญ่สุดและใช้ essential prime implicant มักพาไปสู่คำตอบที่เหมาะสม

---

## 4.12 Minimal POS: จับกลุ่มช่อง 0

**Minimal POS (Minimal Product of Sums)** คือรูปผลคูณของพจน์ OR ที่ลดรูปแล้ว

การทำ POS ด้วย K-map ให้จับกลุ่มช่องที่เป็น 0

เมื่ออ่านกลุ่ม 0 กฎการเขียนตัวแปรในพจน์ OR จะกลับจาก SOP

ถ้าตัวแปรคงที่เป็น 0 ให้เขียนตัวแปรนั้นแบบไม่ complement

ถ้าตัวแปรคงที่เป็น 1 ให้เขียนตัวแปรนั้นแบบ complement

เหตุผลคือพจน์ OR ต้องมีค่าเป็น 0 เฉพาะในกลุ่ม 0 นั้น

ตัวอย่าง maxterm ของแถว `A=0, C=1`

ถ้า `A=0` ต้องทำให้พจน์ OR เป็น 0 จึงเขียนเป็น `A`

ถ้า `C=1` ต้องทำให้พจน์ OR เป็น 0 จึงเขียนเป็น `C'`

ดังนั้นพจน์ OR คือ

```text
(A + C')
```

ใน POS คำตอบสุดท้ายจะเป็น AND ของวงเล็บทั้งหมด

```text
F = (พจน์ OR ที่ 1)(พจน์ OR ที่ 2)...
```

## 4.13 ตัวอย่างที่ 1: K-map 3 ตัวแปรแบบ minimal SOP

กำหนด

```text
F(A,B,C) = Sigma m(1,2,3,5,7)
```

ขั้นที่ 1: วางค่า 1 ลงใน K-map 3 ตัวแปร

```text
          BC
          00  01  11  10
A  0 |    0   1   1   1
   1 |    0   1   1   0
```

ขั้นที่ 2: ระบุหมายเลข minterm เพื่อป้องกันการจับผิดช่อง

```text
          BC
          00      01      11      10
A  0 |   m0=0    m1=1    m3=1    m2=1
   1 |   m4=0    m5=1    m7=1    m6=0
```

ขั้นที่ 3: จับกลุ่ม 4 ช่องที่คอลัมน์ `BC=01` และ `BC=11`

```text
          BC
          00  01  11  10
A  0 |    0  [1   1]  1
   1 |    0  [1   1]  0

กลุ่ม G1 = m1, m3, m5, m7
```

ในกลุ่ม G1 ตัวแปร `A` เปลี่ยน 0/1

ตัวแปร `B` เปลี่ยน 0/1

ตัวแปร `C=1` คงที่

ดังนั้น

```text
G1 = C
```

ขั้นที่ 4: เหลือ `m2` ที่ยังไม่ถูกครอบคลุม จับกับ `m3` เป็นกลุ่ม 2 ช่องในแถวบน

```text
          BC
          00  01  11  10
A  0 |    0   1  [1   1]
   1 |    0   1   1   0

กลุ่ม G2 = m3, m2
```

ในกลุ่ม G2 แถว `A=0` คงที่

คอลัมน์ `BC=11` และ `BC=10` มี `B=1` คงที่ ส่วน `C` เปลี่ยน

ดังนั้น

```text
G2 = A'B
```

คำตอบ minimal SOP คือ

```text
F = C + A'B
```

ตรวจสอบกับบางแถวสำคัญ

| A | B | C | minterm | F จากโจทย์ | C + A'B |
|---:|---:|---:|---:|---:|---:|
| 0 | 0 | 1 | 1 | 1 | 1 |
| 0 | 1 | 0 | 2 | 1 | 1 |
| 1 | 0 | 1 | 5 | 1 | 1 |
| 1 | 1 | 0 | 6 | 0 | 0 |

วงจรที่ได้ใช้ OR ระหว่างสัญญาณ `C` กับเอาต์พุตของ AND `A'B`

เมื่อสร้างใน Logisim จะต้องมี NOT ที่ `A` หนึ่งตัว, AND สองอินพุตหนึ่งตัว และ OR สองอินพุตหนึ่งตัว

## 4.14 ตัวอย่างที่ 2: K-map 4 ตัวแปรพร้อม don't care

กำหนด

```text
F(A,B,C,D) = Sigma m(1,3,7,11,15) + d(0,2,5)
```

สัญลักษณ์ `d(0,2,5)` หมายถึง minterm 0, 2 และ 5 เป็น don't care

ในวงจรจริง don't care มักเกิดจาก input combination ที่ไม่เกิดขึ้นจริง เช่น รหัส BCD ตั้งแต่ 1010 ถึง 1111 หรือสถานะที่เครื่องสถานะไม่ควรเข้าไปถึง

ขั้นที่ 1: ใส่ 1, 0 และ X ลงใน K-map

```text
              CD
              00  01  11  10
AB  00 |      X   1   1   X
    01 |      0   X   1   0
    11 |      0   0   1   0
    10 |      0   0   1   0
```

ขั้นที่ 2: จับกลุ่มคอลัมน์ `CD=11` จำนวน 4 ช่อง

```text
              CD
              00  01  11  10
AB  00 |      X   1  [1]  X
    01 |      0   X  [1]  0
    11 |      0   0  [1]  0
    10 |      0   0  [1]  0

กลุ่ม G1 = m3, m7, m15, m11
```

ใน G1 แถว `AB` เปลี่ยนทุกค่า

คอลัมน์ `CD=11` คงที่

ดังนั้น

```text
G1 = CD
```

ขั้นที่ 3: ใช้ don't care บนแถวบนเพื่อสร้างกลุ่ม 4 ช่อง

```text
              CD
              00  01  11  10
AB  00 |     [X   1   1   X]
    01 |      0   X   1   0
    11 |      0   0   1   0
    10 |      0   0   1   0

กลุ่ม G2 = m0(X), m1, m3, m2(X)
```

กลุ่มนี้ใช้ X สองช่อง เพราะช่วยให้ `m1` ถูกรวมเป็นกลุ่ม 4 แทนที่จะเหลือเป็นกลุ่มเล็ก

ใน G2 แถว `AB=00` คงที่

คอลัมน์เปลี่ยนทุกค่า

ดังนั้น

```text
G2 = A'B'
```

คำตอบ minimal SOP คือ

```text
F = CD + A'B'
```

ถ้าไม่ใช้ don't care คำตอบอาจยาวขึ้น เช่นต้องใช้พจน์สำหรับ `m1` แยกหรือกลุ่มขนาด 2 ที่มี literal มากกว่า

ตารางเปรียบเทียบการใช้ X

| minterm | สถานะ | ใช้ในกลุ่ม | เหตุผล |
|---:|---|---|---|
| m0 | X | G2 | ทำให้แถวบนเป็นกลุ่ม 4 |
| m2 | X | G2 | ทำให้แถวบนเป็นกลุ่ม 4 |
| m5 | X | ไม่ใช้ | ใช้แล้วไม่ช่วยลดจำนวน literal เพิ่มเติม |

ข้อสำคัญคือห้ามบังคับให้ X เป็น 1 ทุกช่อง

X เป็นเพียงสิทธิ์ในการเลือก ไม่ใช่ minterm ที่ต้องครอบคลุม

## 4.15 ตัวอย่างที่ 3: ฟังก์ชันเดียวกันทำทั้ง minimal SOP และ minimal POS

กำหนด

```text
F(A,B,C) = Sigma m(0,2,5,7)
```

ขั้นที่ 1: เขียน K-map

```text
          BC
          00  01  11  10
A  0 |    1   0   0   1
   1 |    0   1   1   0
```

### 4.15.1 ทำ minimal SOP โดยจับ 1

กลุ่มแรกจับ `m0` กับ `m2` โดยใช้การ wrap ขอบซ้ายและขวาในแถวบน

```text
          BC
          00  01  11  10
A  0 |   [1]  0   0  [1]
   1 |    0   1   1   0

G1 = m0, m2
```

ใน G1 ค่า `A=0` คงที่ และ `C=0` คงที่

`B` เปลี่ยนจาก 0 เป็น 1

ดังนั้น

```text
G1 = A'C'
```

กลุ่มที่สองจับ `m5` กับ `m7`

```text
          BC
          00  01  11  10
A  0 |    1   0   0   1
   1 |    0  [1   1]  0

G2 = m5, m7
```

ใน G2 ค่า `A=1` คงที่ และ `C=1` คงที่

`B` เปลี่ยนจาก 0 เป็น 1

ดังนั้น

```text
G2 = AC
```

คำตอบแบบ SOP คือ

```text
F = A'C' + AC
```

### 4.15.2 ทำ minimal POS โดยจับ 0

ช่องที่เป็น 0 คือ `m1, m3, m4, m6`

กลุ่มแรกจับ `m1` กับ `m3`

```text
          BC
          00  01  11  10
A  0 |    1  [0   0]  1
   1 |    0   1   1   0

H1 = m1, m3
```

สำหรับ POS ให้ดูค่าคงที่ของกลุ่ม 0

ใน H1 ค่า `A=0` คงที่ และ `C=1` คงที่

`B` เปลี่ยน

เมื่อตัวแปรคงที่เป็น 0 ให้เขียนแบบไม่ complement

เมื่อตัวแปรคงที่เป็น 1 ให้เขียนแบบ complement

ดังนั้น

```text
H1 = (A + C')
```

กลุ่มที่สองจับ `m4` กับ `m6` โดย wrap ขอบซ้ายและขวาในแถวล่าง

```text
          BC
          00  01  11  10
A  0 |    1   0   0   1
   1 |   [0]  1   1  [0]

H2 = m4, m6
```

ใน H2 ค่า `A=1` คงที่ และ `C=0` คงที่

`B` เปลี่ยน

ดังนั้นพจน์ OR คือ

```text
H2 = (A' + C)
```

คำตอบแบบ POS คือ

```text
F = (A + C')(A' + C)
```

ตรวจสอบว่าทั้ง SOP และ POS ให้ฟังก์ชันเดียวกัน

```text
A'C' + AC  =  (A + C')(A' + C)
```

ทั้งสองรูปคือ XNOR ระหว่าง `A` และ `C`

ข้อสังเกตคือ `B` หายไปจากทั้งสองคำตอบ เพราะค่าของ `B` ไม่ส่งผลต่อเอาต์พุตในฟังก์ชันนี้

## 4.16 ตัวอย่างที่ 4: กลุ่มมุมทั้งสี่และการ wrap สองทิศทาง

กำหนด

```text
F(A,B,C,D) = Sigma m(0,2,8,10)
```

K-map คือ

```text
              CD
              00  01  11  10
AB  00 |      1   0   0   1
    01 |      0   0   0   0
    11 |      0   0   0   0
    10 |      1   0   0   1
```

ถ้ามองแบบตารางธรรมดาจะเห็น 1 กระจายอยู่มุมทั้งสี่

แต่ใน K-map แถวบนกับแถวล่างติดกัน และคอลัมน์ซ้ายกับคอลัมน์ขวาติดกัน

จึงจับเป็นกลุ่ม 4 ช่องได้

```text
              CD
              00  01  11  10
AB  00 |     [1]  0   0  [1]
    01 |      0   0   0   0
    11 |      0   0   0   0
    10 |     [1]  0   0  [1]

G1 = m0, m2, m8, m10
```

ค่าของตัวแปรในกลุ่ม

| minterm | A | B | C | D |
|---:|---:|---:|---:|---:|
| m0 | 0 | 0 | 0 | 0 |
| m2 | 0 | 0 | 1 | 0 |
| m8 | 1 | 0 | 0 | 0 |
| m10 | 1 | 0 | 1 | 0 |

ตัวแปร `A` เปลี่ยน

ตัวแปร `C` เปลี่ยน

ตัวแปร `B=0` คงที่

ตัวแปร `D=0` คงที่

ดังนั้น

```text
F = B'D'
```

ถ้าไม่ใช้ wrap จะได้กลุ่ม 2 ช่องอย่างน้อยสองกลุ่ม เช่น `A'B'D' + AB'D'`

สมการนั้นลดต่อได้เป็น `B'D'` แต่ K-map ช่วยให้มองเห็นคำตอบทันที

## 4.17 Prime implicant และ essential prime implicant

**Implicant** คือกลุ่มของ 1 ที่สามารถเขียนเป็นพจน์หนึ่งพจน์ในรูป SOP ได้

**Prime implicant** คือ implicant ที่ขยายให้ใหญ่ขึ้นไม่ได้แล้วโดยไม่รวม 0

**Essential prime implicant** คือ prime implicant ที่ครอบคลุม minterm อย่างน้อยหนึ่งช่องซึ่งไม่มี prime implicant อื่นครอบคลุม

ให้พิจารณาตัวอย่างจากข้อ 4.13

```text
F(A,B,C) = Sigma m(1,2,3,5,7)

          BC
          00  01  11  10
A  0 |    0   1   1   1
   1 |    0   1   1   0
```

prime implicant ที่สำคัญคือ

```text
P1 = m1,m3,m5,m7  -> C
P2 = m2,m3        -> A'B
```

ตรวจว่า essential หรือไม่

| minterm ที่เป็น 1 | ครอบคลุมโดย P1 | ครอบคลุมโดย P2 | สรุป |
|---:|---|---|---|
| m1 | ใช่ | ไม่ใช่ | P1 จำเป็น |
| m2 | ไม่ใช่ | ใช่ | P2 จำเป็น |
| m3 | ใช่ | ใช่ | ไม่ชี้ขาด |
| m5 | ใช่ | ไม่ใช่ | P1 จำเป็น |
| m7 | ใช่ | ไม่ใช่ | P1 จำเป็น |

ดังนั้น `P1` และ `P2` ต่างเป็น essential prime implicant

คำตอบจึงต้องมีทั้งสองพจน์

```text
F = C + A'B
```

ถ้าฟังก์ชันมี prime implicant หลายตัวที่ไม่ essential อาจมีคำตอบ minimal ได้มากกว่าหนึ่งรูป

ในกรณีเช่นนั้นให้เลือกคำตอบที่เหมาะกับการสร้างวงจรจริง เช่นใช้เกตชนิดเดียวกันน้อยกว่า หรือเข้ากับไอซี 74xx ที่มีอยู่ในชุดทดลอง

## 4.18 การใช้ don't care อย่างมีวินัย

ช่อง don't care เขียนเป็น `X`

ในขั้นตอนลดรูปแบบ SOP สามารถถือ `X` เป็น 1 เพื่อรวมกลุ่มกับช่อง 1 ได้

ในขั้นตอนลดรูปแบบ POS สามารถถือ `X` เป็น 0 เพื่อรวมกลุ่มกับช่อง 0 ได้

แต่ไม่จำเป็นต้องใช้ X ทุกช่อง

การใช้ X ที่ดีต้องทำให้กลุ่มใหญ่ขึ้นหรือทำให้จำนวนกลุ่มลดลง

หากใช้ X แล้วไม่ได้ลดจำนวน literal ไม่ควรเพิ่มกลุ่มโดยไม่จำเป็น

ตัวอย่างการใช้ X ที่ไม่ช่วย

```text
              CD
              00  01  11  10
AB  00 |      0   0   0   0
    01 |      0   1   1   0
    11 |      0   X   X   0
    10 |      0   0   0   0
```

ในกรณีนี้ ช่อง 1 ที่ `m5,m7` จับเป็นกลุ่ม 2 ได้พจน์ `A'BD`

ถ้าใช้ X ที่ `m13,m15` จะจับเป็นกลุ่ม 4 ได้พจน์ `BD`

ดังนั้น X ช่วยลดจาก 3 literal เหลือ 2 literal

ควรใช้ X

แต่ถ้ามี X อยู่โดดเดี่ยวและไม่ติดกับ 1 หรือไม่ช่วยขยายกลุ่ม ก็ไม่ต้องนำมาใช้

ข้อผิดพลาดที่ต้องหลีกเลี่ยงคือการตีความ X เป็นทั้ง 1 และ 0 ในคำตอบเดียวกันโดยไม่ระบุเป้าหมาย

สำหรับ SOP ให้คิดว่า X ที่เลือกใช้กลายเป็น 1

สำหรับ POS ให้คิดว่า X ที่เลือกใช้กลายเป็น 0

## 4.19 Hazard เบื้องต้นที่มองเห็นจาก K-map

K-map ยังช่วยมองเห็นปัญหา timing บางชนิดในวงจรเชิงผสมได้

หัวข้อนี้จะเรียนละเอียดในบทที่เกี่ยวกับ timing และ hazard แต่ควรเห็นภาพตั้งแต่ตอนลดรูป

**Static-1 hazard** เกิดได้เมื่อเอาต์พุตควรคงเป็น 1 ระหว่างการเปลี่ยน input หนึ่งตัว แต่ในวงจรจริงอาจเกิดช่วงสั้น ๆ ที่เอาต์พุตตกเป็น 0 เพราะ delay ของเกตไม่เท่ากัน

พิจารณาฟังก์ชัน

```text
F(A,B,C) = AB + A'C
```

K-map ของฟังก์ชันนี้คือ

```text
          BC
          00  01  11  10
A  0 |    0   1   1   0
   1 |    0   0   1   1
```

กลุ่มจากพจน์ `A'C` ครอบคลุม `m1,m3`

กลุ่มจากพจน์ `AB` ครอบคลุม `m6,m7`

```text
          BC
          00  01  11  10
A  0 |    0  [1   1]  0    <- A'C
   1 |    0   0  [1   1]   <- AB
```

พิจารณาการเปลี่ยนจาก `A=0,B=1,C=1` ไปเป็น `A=1,B=1,C=1`

นี่คือการเปลี่ยนจาก `m3` ไป `m7`

สองช่องนี้ติดกันแนวตั้ง และทั้งคู่ควรมีค่า F=1

แต่ไม่มีพจน์ใดครอบคลุม `m3` และ `m7` พร้อมกัน

ในวงจรจริง เมื่อ `A` เปลี่ยน สัญญาณผ่านทาง `A'C` อาจดับก่อนที่ทาง `AB` จะติด ทำให้เกิด glitch สั้น ๆ

วิธีลด static-1 hazard คือเพิ่มพจน์คาบเกี่ยว (consensus term)

กลุ่มคาบเกี่ยวคือ `m3,m7` ที่คอลัมน์ `BC=11`

```text
          BC
          00  01  11  10
A  0 |    0   1  [1]  0
   1 |    0   0  [1]  1

พจน์คาบเกี่ยว = BC
```

ดังนั้นรูปที่ลด hazard คือ

```text
F = AB + A'C + BC
```

พจน์ `BC` ซ้ำซ้อนในเชิงพีชคณิต เพราะไม่เปลี่ยน truth table

แต่มีประโยชน์เชิงเวลา เพราะให้เส้นทางตรรกะที่คงค่า 1 ระหว่างการเปลี่ยนของ `A`

นี่เป็นตัวอย่างที่คำตอบ "minimal ที่สุด" ทางจำนวนพจน์อาจไม่เหมาะที่สุดสำหรับวงจรที่ต้องปลอด glitch

## 4.20 การแปลงคำตอบ K-map เป็นวงจรเกต

หลังจากได้สมการ minimal SOP หรือ POS ต้องแปลงเป็นวงจรที่สร้างได้จริง

สำหรับ SOP เช่น

```text
F = CD + A'B'
```

วงจรใช้

- NOT สำหรับ `A` และ `B`
- AND สำหรับ `CD`
- AND สำหรับ `A'B'`
- OR รวมสองพจน์

แผนภาพเชิงตรรกะ

```text
A ---- NOT ---- A' --\
                      AND ----\
B ---- NOT ---- B' --/         \
                                OR ---- F
C ------------------\         /
                    AND -----/
D ------------------/

term1 = A'B'
term2 = CD
```

ถ้าต้องสร้างด้วยไอซี 74xx อาจใช้

- 74LS04 สำหรับ NOT
- 74LS08 สำหรับ AND
- 74LS32 สำหรับ OR

สำหรับ POS เช่น

```text
F = (A + C')(A' + C)
```

วงจรใช้

- NOT สำหรับ `A` และ `C`
- OR สำหรับ `(A + C')`
- OR สำหรับ `(A' + C)`
- AND รวมผลของวงเล็บ

แผนภาพเชิงตรรกะ

```text
C ---- NOT ---- C' --\
                      OR ----\
A -------------------/        \
                                AND ---- F
A ---- NOT ---- A' --\        /
                      OR ----/
C -------------------/
```

ใน Logisim Evolution สามารถวางเกตตามสมการได้โดยตรง

แต่เมื่อต่อบนเบรดบอร์ดจริง ต้องดูจำนวนเกตในไอซีแต่ละตัวด้วย

ตัวอย่าง 74LS08 มี AND 2-input จำนวน 4 เกตในแพ็กเกจเดียว

ถ้าสมการใช้ AND 2-input เพียงสองตัว ก็ใช้ไอซี 74LS08 ตัวเดียวพอ

## 4.21 ข้อผิดพลาดเฉพาะ K-map ที่พบบ่อย

1. เรียงคอลัมน์เป็น `00,01,10,11` แทน `00,01,11,10`
2. ลืมว่าแถวบนกับล่าง หรือคอลัมน์ซ้ายกับขวา wrap กันได้
3. จับกลุ่ม 3 ช่องหรือ 6 ช่อง ซึ่งไม่ใช่กำลังของ 2
4. จับกลุ่มรูปตัว L แล้วอ่านเป็นพจน์เดียว
5. อ่านพจน์จากกลุ่มโดยเก็บตัวแปรที่เปลี่ยน แทนที่จะเก็บตัวแปรที่คงที่
6. ใช้ X เป็น 1 ทุกช่อง ทั้งที่บาง X ไม่ช่วยลดรูป
7. ทำ POS แต่ยังจับช่อง 1 เหมือน SOP
8. อ่านกลุ่ม POS ด้วยกฎเดียวกับ SOP ทำให้ complement ผิด
9. มองช่องทแยงเป็น adjacency ทั้งที่ไม่ติดกัน
10. ลืมตรวจว่า 1 ทุกช่องถูกครอบคลุมใน SOP หรือ 0 ทุกช่องถูกครอบคลุมใน POS

ตัวอย่างการอ่านผิดจากกลุ่ม

```text
กลุ่ม: แถว AB=01 และ AB=11, คอลัมน์ CD=01 และ CD=11
```

ค่าที่เปลี่ยนคือ `A` และ `C`

ค่าที่คงที่คือ `B=1` และ `D=1`

พจน์ที่ถูกต้องคือ `BD`

ไม่ใช่ `AC`

การจำสั้น ๆ คือ "ตัวที่วิ่งในกลุ่มให้ตัดทิ้ง ตัวที่นิ่งให้เขียน"

## 4.22 แบบฝึกหัดท้ายบท

### ข้อ 1: ตำแหน่ง minterm บน K-map 3 ตัวแปร

ให้วาด K-map 3 ตัวแปรของ `A,B,C` และใส่ตำแหน่ง `m0` ถึง `m7`

จากนั้นทำเครื่องหมายช่อง `m2,m3,m6,m7`

ตอบว่ากลุ่ม 4 ช่องนี้ลดเป็นพจน์ใด

### ข้อ 2: Minimal SOP แบบ 3 ตัวแปร

กำหนด

```text
F(A,B,C) = Sigma m(0,1,2,5,6,7)
```

ให้วาด K-map 3 ตัวแปร จับกลุ่มให้ใหญ่ที่สุด และเขียนคำตอบ minimal SOP

ระบุด้วยว่ามีกลุ่มใดใช้การ wrap ขอบหรือไม่

### ข้อ 3: Minimal SOP แบบ 4 ตัวแปร

กำหนด

```text
F(A,B,C,D) = Sigma m(0,2,5,7,8,10,13,15)
```

ให้วาด K-map 4 ตัวแปร จับกลุ่ม และเขียนสมการที่ลดรูปแล้ว

อธิบายค่าคงที่ของแต่ละกลุ่มก่อนเขียนพจน์

### ข้อ 4: Don't care ในรหัส BCD

วงจรรับอินพุต BCD `A,B,C,D` โดยรหัส 0000 ถึง 1001 เท่านั้นที่เกิดขึ้นจริง

ให้เอาต์พุตเป็น 1 เมื่อเลขฐานสิบเป็น 0, 2, 3, 5, 8 และให้รหัส 10 ถึง 15 เป็น don't care

เขียนฟังก์ชันในรูป

```text
F(A,B,C,D) = Sigma m(...) + d(...)
```

จากนั้นลดรูปด้วย K-map

### ข้อ 5: Minimal POS

กำหนด

```text
F(A,B,C) = Pi M(1,3,4,6)
```

ให้วาด K-map และลดรูปเป็น minimal POS

ต้องแสดงกลุ่มของช่อง 0 และเขียน maxterm ของแต่ละกลุ่ม

### ข้อ 6: เปรียบเทียบ SOP และ POS จากฟังก์ชันเดียวกัน

กำหนด

```text
F(A,B,C,D) = Sigma m(0,1,4,5,10,11,14,15)
```

ให้หาทั้ง minimal SOP และ minimal POS

เปรียบเทียบจำนวนพจน์และจำนวน literal ของสองคำตอบ

จากนั้นระบุว่าถ้าต้องสร้างด้วยเกต AND-OR หรือ OR-AND รูปใดเหมาะกว่า

### ข้อ 7: Prime implicant

ให้ฟังก์ชัน

```text
F(A,B,C,D) = Sigma m(1,3,5,7,9,11,13,15)
```

วาด K-map แล้วระบุ prime implicant ทั้งหมด

จากนั้นระบุ essential prime implicant และเขียนคำตอบ minimal SOP

### ข้อ 8: Hazard จาก K-map

พิจารณาฟังก์ชัน

```text
F(A,B,C) = A'B + AC
```

ให้วาด K-map ของฟังก์ชันนี้

ระบุคู่ช่องที่เป็น 1 และติดกันแต่ไม่ถูกครอบคลุมด้วยกลุ่มเดียวกัน

เขียนพจน์ consensus ที่ควรเพิ่มเพื่อลด static-1 hazard

### ข้อ 9: K-map 5 ตัวแปร

กำหนด

```text
F(A,B,C,D,E) = Sigma m(0,1,3,16,17,19)
```

ให้วาด K-map 5 ตัวแปรแบบสองผังซ้อน `A=0` และ `A=1`

จับกลุ่มข้ามผังให้ถูกต้อง และเขียนพจน์ที่ได้

### ข้อ 10: ตรวจคำตอบจากสมการกลับสู่ K-map

ให้สมการ

```text
F(A,B,C,D) = B'D' + AC
```

จงระบายช่อง 1 บน K-map 4 ตัวแปรจากสมการนี้

เขียนรายการ minterm ทั้งหมดในรูป `Sigma m(...)`

จากนั้นตรวจว่ามีวิธีจับกลุ่มอื่นที่ให้สมการสั้นเท่ากันหรือไม่

## 4.23 สรุปเฉพาะบทผังคาร์โนห์

K-map ลดรูปฟังก์ชันบูลีนได้เพราะจัดช่องตาม Gray code ทำให้ช่องติดกันต่างกันเพียงหนึ่งตัวแปร

การจับกลุ่มจึงเทียบเท่ากับการตัดตัวแปรที่เปลี่ยนค่าออกจากพจน์

K-map 2, 3 และ 4 ตัวแปรวาดเป็นตารางสองมิติ ส่วน K-map 5 ตัวแปรใช้สองผังซ้อนและจับกลุ่มข้ามผังในตำแหน่งเดียวกันได้

minimal SOP ได้จากการจับกลุ่มช่อง 1 ส่วน minimal POS ได้จากการจับกลุ่มช่อง 0 และต้องใช้กฎ complement ของ maxterm ให้ถูกต้อง

don't care เป็นเครื่องมือช่วยขยายกลุ่ม ไม่ใช่ช่องที่ต้องใช้ทุกครั้ง

prime implicant และ essential prime implicant ช่วยให้เลือกกลุ่มที่จำเป็นอย่างเป็นระบบ

สุดท้าย K-map ยังทำให้เห็น static hazard เบื้องต้นได้จากช่อง 1 ที่ติดกันแต่ไม่มีพจน์ใดครอบคลุมร่วมกัน ซึ่งเชื่อมต่อไปยังการวิเคราะห์ timing และความน่าเชื่อถือของวงจรในบทถัดไป
