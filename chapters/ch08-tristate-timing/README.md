## 8.1 บทนำ: เมื่อลอจิกไม่ได้มีแค่ 0 และ 1

ในบทก่อนหน้า วงจรเชิงผสมที่นักศึกษาออกแบบมีลักษณะสำคัญอย่างหนึ่งคือ output ของเกตทุกตัวต้องเป็น `0` หรือ `1` เสมอ

แต่ในระบบคอมพิวเตอร์จริง มีสถานการณ์ที่อุปกรณ์หลายตัวต้องต่อร่วมกับสายสัญญาณชุดเดียวกัน เช่น data bus ระหว่าง CPU, memory และ I/O interface

ถ้าอุปกรณ์ทุกตัวขับสายพร้อมกัน สายหนึ่งเส้นอาจถูกดึงขึ้นเป็น `1` โดยอุปกรณ์หนึ่ง และถูกดึงลงเป็น `0` โดยอีกอุปกรณ์หนึ่งในเวลาเดียวกัน

เหตุการณ์เช่นนี้ไม่ใช่ "ผลลัพธ์ผิดเล็กน้อย" แต่เป็นการต่อเอาต์พุตสองตัวชนกันโดยตรง ทำให้มีกระแสสูงผิดปกติ เกิดความร้อน แรงดันอยู่ในช่วงไม่แน่นอน และอาจทำให้ไอซีเสียหาย

ดังนั้นวงจรดิจิทัลจึงต้องมีสถานะที่สาม คือ **High impedance** หรือ **Hi-Z** ซึ่งหมายถึง output ปล่อยสาย ไม่ได้ขับ `0` และไม่ได้ขับ `1`

อุปกรณ์ที่สร้างสถานะนี้ได้เรียกว่า **ไตรสเตดบัฟเฟอร์ (tristate buffer)** หรือ **three-state buffer**

บทนี้จึงเชื่อมจากความถูกต้องเชิงตรรกะไปสู่ความถูกต้องเชิงฮาร์ดแวร์จริง โดยเน้นคำถามต่อไปนี้

- ใครเป็นผู้ขับบัสในแต่ละช่วงเวลา
- เส้นทางสัญญาณใดช้าที่สุดในวงจร
- output หนึ่งตัวขับ input ได้กี่ตัวโดยยังรักษาระดับแรงดันได้
- สัญญาณรบกวนทำให้ `0` กับ `1` ถูกตีความผิดได้เมื่อใด
- glitch ที่เกิดจาก propagation delay สามารถทำให้วงจรทำงานผิดได้อย่างไร

ใน CPU จริง คำถามเหล่านี้เกี่ยวข้องกับการอ่าน register, การเขียน memory, การเลือกแหล่งข้อมูลเข้า ALU และการจับข้อมูลด้วย clock ในวงจรเชิงลำดับ

## 8.2 ไตรสเตดบัฟเฟอร์: output ที่ปล่อยสายได้

**บัฟเฟอร์ (buffer)** ปกติเป็นวงจรที่ให้ output เท่ากับ input

```text
Y = A
```

ถ้า `A = 0` จะได้ `Y = 0`

ถ้า `A = 1` จะได้ `Y = 1`

แต่ไตรสเตดบัฟเฟอร์มี input ควบคุมเพิ่มอีกหนึ่งเส้น เรียกว่า **enable** หรือสัญญาณอนุญาตให้ขับ output

เมื่อ enable ทำงาน วงจรจะส่งค่า input ไปยัง output

เมื่อ enable ไม่ทำงาน output จะกลายเป็น `Z` หรือ Hi-Z

`Z` ไม่ใช่ค่าลอจิกที่สามสำหรับคำนวณบูลีน แต่เป็นสภาวะทางไฟฟ้าที่ output มีความต้านทานสูงมากจนแทบไม่ดึงสายขึ้นหรือลง

ถ้าสายที่ถูกปล่อยเป็น `Z` ไม่มีวงจรอื่นขับอยู่ แรงดันบนสายนั้นอาจลอยและอ่านค่าไม่แน่นอน

ดังนั้นบัสที่มี tristate ต้องมีการควบคุม enable ที่ชัดเจน หรือใช้ pull-up/pull-down ในกรณีที่ต้องการค่าเริ่มต้น

### 8.2.1 ตารางความจริงของ active-high enable

ไตรสเตดแบบ **active-high enable** หมายถึง enable ทำงานเมื่อ `EN = 1`

| EN | A | Y | ความหมายทางไฟฟ้า |
|---:|---:|:---:|---|
| 0 | 0 | Z | output ปล่อยสาย |
| 0 | 1 | Z | output ปล่อยสาย |
| 1 | 0 | 0 | output ดึงสายลงต่ำ |
| 1 | 1 | 1 | output ดึงสายขึ้นสูง |

สัญลักษณ์แบบย่อสามารถเขียนได้ดังนี้

```text
        ┌─────────┐
A ─────▶│         │
        │ BUF 3S  ├──── Y
EN ────▶│         │
        └─────────┘

EN = 1 : Y = A
EN = 0 : Y = Z
```

ใน Logisim Evolution สามารถวางองค์ประกอบแบบ controlled buffer หรือ tri-state buffer แล้วกำหนดขา enable ให้เป็น active high ได้

### 8.2.2 ตารางความจริงของ active-low enable

ไตรสเตดแบบ **active-low enable** หมายถึง enable ทำงานเมื่อสัญญาณควบคุมเป็น `0`

มักเขียนชื่อสัญญาณด้วยขีดบน เช่น `OE'` หรือ `/OE` โดย `OE` มาจาก output enable

| /OE | A | Y | ความหมายทางไฟฟ้า |
|---:|---:|:---:|---|
| 0 | 0 | 0 | output ดึงสายลงต่ำ |
| 0 | 1 | 1 | output ดึงสายขึ้นสูง |
| 1 | 0 | Z | output ปล่อยสาย |
| 1 | 1 | Z | output ปล่อยสาย |

สัญลักษณ์มักมีวงกลมเล็กที่ขา enable เพื่อบอกว่า active low

```text
        ┌─────────┐
A ─────▶│         │
        │ BUF 3S  ├──── Y
/OE ──o▶│         │
        └─────────┘

/OE = 0 : Y = A
/OE = 1 : Y = Z
```

ไอซีตระกูล 74xx หลายตัวใช้ enable แบบ active low เพราะในฮาร์ดแวร์จริงสัญญาณควบคุมจำนวนมากถูกออกแบบให้ดึงลงเป็นการเปิดใช้งานได้ง่ายและทนต่อ noise บางรูปแบบ

## 8.3 บัส: การแชร์สายโดยให้มีผู้ขับครั้งละหนึ่งตัว

**บัส (bus)** คือกลุ่มของสายสัญญาณที่ใช้ร่วมกันเพื่อส่งข้อมูลหลายบิตในเวลาเดียวกัน

ตัวอย่างเช่น data bus ขนาด 8 บิตประกอบด้วยสาย `D7..D0`

ถ้า CPU, RAM และอุปกรณ์ I/O ทุกตัวต่อเข้ากับ data bus เดียวกัน แต่ละอุปกรณ์ต้องมีวิธีตัดตัวเองออกจากบัสเมื่อไม่ได้เป็นผู้ส่งข้อมูล

ไตรสเตดบัฟเฟอร์จึงเป็นอุปกรณ์สำคัญในการสร้างบัสแบบแชร์สาย

```text
                 8-bit shared data bus
              D7 D6 D5 D4 D3 D2 D1 D0
              │  │  │  │  │  │  │  │
       ┌──────┴──┴──┴──┴──┴──┴──┴──┴──────┐
       │                                    │
  ┌────▼────┐                          ┌────▼────┐
  │ RAM OUT │── tristate drivers ─────▶│          │
  └────┬────┘                          │   CPU    │
       │                               │          │
     /OE_RAM                           └────▲────┘
                                             │
  ┌────┴────┐                                │
  │ I/O OUT │── tristate drivers ────────────┘
  └────┬────┘
       │
     /OE_IO
```

หลักการสำคัญคือ ในช่วงอ่านข้อมูลจาก RAM ต้องให้ `/OE_RAM = 0` และ `/OE_IO = 1`

ในช่วงอ่านข้อมูลจาก I/O ต้องให้ `/OE_IO = 0` และ `/OE_RAM = 1`

ถ้าทั้งสองตัวถูก enable พร้อมกันและข้อมูลไม่ตรงกัน จะเกิด **bus contention** หรือการชนกันของบัส

### 8.3.1 ตารางสถานะของบัส 1 บิตที่มีผู้ขับสองตัว

พิจารณาบัสหนึ่งเส้นชื่อ `D`

มีอุปกรณ์สองตัวต่อผ่านไตรสเตดบัฟเฟอร์ คือ `M` จาก memory และ `P` จาก peripheral

| EN_M | M | EN_P | P | ค่า D | สถานะบัส |
|---:|---:|---:|---:|:---:|---|
| 0 | 0 | 0 | 0 | Z | ไม่มีผู้ขับ สายลอย |
| 0 | 1 | 0 | 1 | Z | ไม่มีผู้ขับ สายลอย |
| 1 | 0 | 0 | 0 | 0 | memory ขับบัส |
| 1 | 1 | 0 | 0 | 1 | memory ขับบัส |
| 0 | 0 | 1 | 0 | 0 | peripheral ขับบัส |
| 0 | 0 | 1 | 1 | 1 | peripheral ขับบัส |
| 1 | 0 | 1 | 0 | 0 | ขับพร้อมกันแต่ค่าเท่ากัน ยังไม่ควรทำ |
| 1 | 1 | 1 | 1 | 1 | ขับพร้อมกันแต่ค่าเท่ากัน ยังไม่ควรทำ |
| 1 | 0 | 1 | 1 | X | bus contention |
| 1 | 1 | 1 | 0 | X | bus contention |

สัญลักษณ์ `X` ในที่นี้หมายถึงไม่สามารถรับประกันแรงดันจริงได้

การที่สองอุปกรณ์ขับค่าเท่ากันไม่ได้ทำให้การออกแบบถูกต้อง เพราะรอบถัดไปข้อมูลอาจไม่เท่ากัน และการเปิด output สองตัวชนกันยังเพิ่มโหลดโดยไม่จำเป็น

### 8.3.2 ตัวถอดรหัสกับสัญญาณ enable ของบัส

ในการเลือกอุปกรณ์หนึ่งตัวจากหลายตัว นิยมใช้ decoder สร้างสัญญาณ enable แบบ one-hot

ถ้ามีอุปกรณ์ 4 ตัวบนบัสเดียวกัน ใช้ address select สองบิต `S1 S0`

```text
S1 S0 ──▶ 2-to-4 decoder ──▶ EN0 EN1 EN2 EN3

EN0 = 1 เมื่อ S1S0 = 00
EN1 = 1 เมื่อ S1S0 = 01
EN2 = 1 เมื่อ S1S0 = 10
EN3 = 1 เมื่อ S1S0 = 11
```

วงจรที่ถูกต้องต้องรับประกันว่า `EN0..EN3` มีค่า `1` ได้เพียงหนึ่งเส้นในแต่ละช่วงเวลา

แต่ในฮาร์ดแวร์จริง การเปลี่ยนจาก `01` ไป `10` อาจทำให้ output ของ decoder เปลี่ยนไม่พร้อมกัน

ดังนั้นการควบคุมบัสในระบบเร็วสูงมักเพิ่มช่วงเวลาสั้น ๆ ที่ทุก enable ปิดอยู่ เรียกว่า **turnaround time** หรือ **dead time**

```text
ช่วงอ่านจากอุปกรณ์ 1 ไปอุปกรณ์ 2

เวลา       t0       t1       t2       t3
EN1        1────────0────────────────────
EN2        0────────────────────1────────
BUS        data1────Z──────────data2─────
                     ↑ dead time
```

ช่วง `Z` สั้น ๆ ช่วยป้องกันไม่ให้ output ของอุปกรณ์เดิมยังไม่ทันปิดในขณะที่อุปกรณ์ใหม่เริ่มขับบัส

## 8.4 ความหมายของเวลาหน่วงในวงจรดิจิทัล

วงจรดิจิทัลในสมการบูลีนดูเหมือนเปลี่ยนค่าทันที

แต่ในไอซีจริง transistor ต้องใช้เวลาในการชาร์จและคายประจุที่โหนดภายใน

จึงเกิดเวลาหน่วงระหว่าง input เปลี่ยนกับ output เปลี่ยน

คำสำคัญที่ต้องแยกให้ออกมีดังนี้

| สัญลักษณ์ | ชื่อ | ความหมาย |
|---|---|---|
| `t_pd` | propagation delay | เวลาจาก input เปลี่ยนจน output เปลี่ยนถึงระดับที่ถือว่าเป็นค่าใหม่ |
| `t_PLH` | propagation delay low-to-high | delay เมื่อ output เปลี่ยนจาก `0` ไป `1` |
| `t_PHL` | propagation delay high-to-low | delay เมื่อ output เปลี่ยนจาก `1` ไป `0` |
| `t_r` | rise time | เวลาที่สัญญาณไต่จากระดับต่ำไปสูง เช่น 10% ถึง 90% |
| `t_f` | fall time | เวลาที่สัญญาณตกจากระดับสูงไปต่ำ เช่น 90% ถึง 10% |
| `t_ZH` | Hi-Z to high | delay จากปล่อยสายไปเป็นขับ `1` |
| `t_HZ` | high to Hi-Z | delay จากขับ `1` ไปเป็นปล่อยสาย |

โดยทั่วไป `t_r` และ `t_f` ถูกเรียกรวมว่า **transition time** หรือเวลาที่สัญญาณใช้ในการเปลี่ยนระดับแรงดันจริงระหว่าง LOW และ HIGH

transition time ยาวเกินไปทำให้สัญญาณอยู่ในช่วงไม่แน่นอนนานขึ้น เพิ่มโอกาสรับ noise และอาจทำให้ input ของเกตถัดไปสวิตช์ผิดจังหวะ

ใน datasheet ของไอซีจริง มักให้ค่า delay เป็นช่วง เช่น typical และ maximum

การออกแบบเชิงวิศวกรรมต้องใช้ค่า maximum สำหรับกรณี worst case ไม่ใช่ค่า typical

### 8.4.1 tPLH และ tPHL ไม่จำเป็นต้องเท่ากัน

พิจารณา inverter หนึ่งตัว

เมื่อ input เปลี่ยน `0 -> 1` output จะเปลี่ยน `1 -> 0` จึงเกี่ยวข้องกับ `t_PHL`

เมื่อ input เปลี่ยน `1 -> 0` output จะเปลี่ยน `0 -> 1` จึงเกี่ยวข้องกับ `t_PLH`

```text
Input A      ___───────────────___
             0      1          0

Output Y     ───────___───────────
             1      0          1
                 │  │       │  │
                 │<-tPHL--->│  │
                          │<-tPLH->│
```

เหตุผลที่ `t_PLH` และ `t_PHL` ต่างกันได้คือ transistor ฝั่งดึงขึ้นและดึงลงมีคุณสมบัติทางไฟฟ้าไม่เหมือนกัน รวมถึงโหลด capacitive ที่ output ต้องชาร์จหรือคายประจุ

## 8.5 การคำนวณ delay ของวงจรหลายชั้น

วงจรเชิงผสมที่มีหลายเกตเรียงต่อกันต้องคำนวณ delay ตามเส้นทางจาก input ไป output

เส้นทางที่ใช้เวลานานที่สุดเรียกว่า **worst-case path** หรือ **critical path**

ค่า delay รวมตามเส้นทางหนึ่งคำนวณโดยบวก delay ของเกตที่สัญญาณผ่าน

ถ้า datasheet แยก `t_PLH` และ `t_PHL` ต้องดูทิศทางการเปลี่ยนของ output แต่ละเกต

ในระดับรายวิชานี้ เมื่อต้องประมาณเบื้องต้นสามารถใช้ `t_pd(max)` ของเกตแต่ละชนิดแล้วบวกกันตามเส้นทางที่ยาวที่สุด

### 8.5.1 ระดับชั้นของลอจิกกับ delay

สมการที่ลดจำนวนเกตได้ไม่ได้แปลว่าเร็วที่สุดเสมอไป

ตัวอย่างเช่นสมการแบบ sum-of-products สองระดับอาจมี AND หนึ่งชั้นและ OR หนึ่งชั้น

แต่ถ้าใช้ NAND เพียงชนิดเดียว อาจต้องเพิ่ม inverter หรือต่อ NAND หลายชั้น ทำให้ delay เพิ่มขึ้น

```text
วงจร 2 ระดับ

A ─┐
   AND ─┐
B ─┘    │
        OR ── F
C ─┐    │
   AND ─┘
D ─┘

delay path = t_AND + t_OR
```

```text
วงจรหลายระดับ

A ─┐
   NAND ─┐
B ─┘     NAND ─┐
C ───────┘     NAND ── F
D ─────────────┘

delay path อาจผ่าน NAND 3 ชั้น
```

ในการต่อไอซี 74xx บน breadboard สายยาวและโหลด input หลายตัวจะเพิ่ม capacitance ทำให้ขอบสัญญาณช้าลง จึงควรวางไอซีที่เกี่ยวข้องใกล้กันและใช้สายให้สั้นเท่าที่เหมาะสม

## 8.6 Fan-in และ fan-out

**Fan-in** คือจำนวน input ของเกตหนึ่งตัว

เช่น AND 2 input มี fan-in = 2 และ NAND 4 input มี fan-in = 4

fan-in มากทำให้เกตหนึ่งตัวรับตัวแปรได้หลายตัว แต่ภายในไอซีอาจมี delay มากขึ้นเพราะ transistor หลายตัวต่ออนุกรมหรือมี capacitance มากขึ้น

**Fan-out** คือจำนวน input ของเกตอื่นที่ output หนึ่งตัวสามารถขับได้โดยยังรักษาระดับแรงดันและกระแสตามมาตรฐาน

fan-out ไม่ควรเดาจากจำนวนสายที่ต่อได้บน breadboard แต่ต้องคำนวณจากกระแส output ของไอซีและกระแส input ที่โหลดต้องการ

สำหรับการคำนวณแบบ conservative ใช้สองกรณีคือสถานะ HIGH และ LOW

```text
fan-out HIGH = IOH(max ที่แหล่งจ่ายออกได้) / IIH(max ต่อ input หนึ่งตัว)
fan-out LOW  = IOL(max ที่ดึงลงได้)        / IIL(max ต่อ input หนึ่งตัว)

fan-out ที่ใช้ได้ = ค่าที่น้อยกว่า
```

ต้องระวังเครื่องหมายใน datasheet เพราะกระแสบางตารางใช้ค่าติดลบสำหรับกระแสที่ไหลออกจาก output

ในการคำนวณเบื้องต้นให้ใช้ขนาดของกระแสเป็นบวก แล้วเลือกค่าที่จำกัดที่สุด

### 8.6.1 Fan-out ใน TTL กับ CMOS

ตระกูล TTL input ใช้กระแสไบอัสมากกว่า CMOS โดยเฉพาะเมื่อ input เป็น LOW

จึงมักมี fan-out จำกัดโดยกระแส `IIL`

ตระกูล CMOS input มีความต้านทานสูงมาก กระแส DC input ต่ำมาก จึงมี fan-out เชิง DC สูง

แต่ในวงจรเร็ว fan-out ของ CMOS ถูกจำกัดโดย capacitance ของ input หลายตัวรวมกัน เพราะ output ต้องชาร์จและคายประจุโหลดเหล่านั้นทุกครั้งที่สัญญาณเปลี่ยน

ดังนั้น CMOS ไม่ได้แปลว่าต่อ input ได้ไม่จำกัดในงานที่ต้องการความเร็ว

## 8.7 Noise margin: ช่องว่างความปลอดภัยของระดับแรงดัน

ไอซีดิจิทัลไม่ได้ตีความแรงดันเป็น `0` หรือ `1` ที่ค่าเดียวพอดี

แต่กำหนดช่วงแรงดันที่รับประกันว่าเป็น LOW หรือ HIGH

ค่าที่ใช้บ่อยมีดังนี้

| สัญลักษณ์ | ความหมาย |
|---|---|
| `VOH` | output voltage high: แรงดัน output ต่ำสุดที่ยังรับประกันว่าเป็น HIGH |
| `VOL` | output voltage low: แรงดัน output สูงสุดที่ยังรับประกันว่าเป็น LOW |
| `VIH` | input voltage high: แรงดัน input ต่ำสุดที่อ่านเป็น HIGH |
| `VIL` | input voltage low: แรงดัน input สูงสุดที่อ่านเป็น LOW |

**Noise margin** คือส่วนต่างแรงดันที่วงจรทนต่อสัญญาณรบกวนได้โดยไม่ตีความผิด

```text
Noise margin high: NMH = VOH(min) - VIH(min)
Noise margin low : NML = VIL(max) - VOL(max)
```

ตัวอย่างระดับแรงดัน TTL 5 V ที่พบในตำราเบื้องต้นมักใช้ค่าโดยประมาณ

| พารามิเตอร์ | ค่าโดยประมาณ |
|---|---:|
| `VOH(min)` | 2.4 V |
| `VIH(min)` | 2.0 V |
| `VOL(max)` | 0.4 V |
| `VIL(max)` | 0.8 V |

ดังนั้น

```text
NMH = 2.4 - 2.0 = 0.4 V
NML = 0.8 - 0.4 = 0.4 V
```

หมายความว่าถ้า output HIGH มี noise ตกลงมาไม่เกินประมาณ 0.4 V ยังควรอ่านเป็น HIGH ได้

ถ้า output LOW มี noise ยกขึ้นไม่เกินประมาณ 0.4 V ยังควรอ่านเป็น LOW ได้

สำหรับ CMOS ที่ใช้ไฟเลี้ยง 5 V ระดับ threshold มักสัมพันธ์กับสัดส่วนของ `VCC` ทำให้ noise margin ดีกว่า TTL ในหลายกรณี

แต่ CMOS input ห้ามปล่อยลอย เพราะ input impedance สูงมาก ทำให้รับประจุหรือ noise จากสิ่งแวดล้อมได้ง่าย

## 8.8 การใช้พลังงานของวงจรลอจิก

พลังงานในวงจรดิจิทัลมีทั้งส่วนคงที่และส่วนที่ขึ้นกับการสวิตช์

ใน TTL มีกระแสภายในไหลอยู่แม้สัญญาณไม่เปลี่ยน จึงมี static power สูงกว่า CMOS โดยทั่วไป

ใน CMOS อุดมคติ เมื่อ output คงที่ จะมีกระแสรั่วไหลน้อยมาก

แต่เมื่อสัญญาณเปลี่ยน สายและ input capacitance ต้องถูกชาร์จและคายประจุ ทำให้เกิด dynamic power

สูตรประมาณของ dynamic power คือ

```text
P_dynamic ≈ C_load × VCC^2 × f_switch
```

เมื่อ `C_load` คือ capacitance รวมที่ output ต้องขับ, `VCC` คือไฟเลี้ยง และ `f_switch` คือความถี่การสวิตช์

ผลสำคัญคือ fan-out มากขึ้นทำให้ `C_load` มากขึ้น และสัญญาณเปลี่ยนเร็วขึ้นทำให้ใช้พลังงานมากขึ้น

### 8.8.1 เปรียบเทียบ TTL กับ CMOS แบบย่อ

| ประเด็น | TTL | CMOS |
|---|---|---|
| input current | มากกว่า CMOS | ต่ำมากใน DC |
| static power | สูงกว่า | ต่ำมาก |
| dynamic power | มี แต่ไม่เด่นเท่า CMOS สมัยใหม่ | เพิ่มตาม `C V^2 f` |
| noise margin | ปานกลาง | มักสูงกว่าเมื่อใช้ระดับ CMOS |
| input floating | ไม่ควรปล่อยลอย | ห้ามปล่อยลอยอย่างยิ่ง |
| ตัวอย่าง 74xx | 74LS, 74ALS | 74HC, 74HCT |

เมื่อต่อวงจรจริงในห้องปฏิบัติการ ต้องไม่ผสม TTL และ CMOS โดยไม่ตรวจระดับแรงดันและความเข้ากันได้ของ input threshold

ตัวอย่างเช่น 74HC ที่ใช้ไฟ 5 V ต้องการ `VIH` สูงกว่า TTL output บางกรณี จึงอาจต้องใช้ 74HCT ถ้าต้องรับสัญญาณจาก TTL

## 8.9 Hazard: glitch ที่เกิดจาก delay ต่างกัน

**Hazard** คือสภาวะที่ output ของวงจรเชิงผสมเกิด pulse สั้น ๆ ทั้งที่ตามสมการบูลีนควรคงค่าเดิม

pulse สั้นนี้มักเรียกว่า **glitch**

Hazard ไม่ปรากฏในตารางความจริง เพราะตารางความจริงถือว่าสัญญาณเปลี่ยนพร้อมกันและเกตไม่มี delay

แต่ในวงจรจริง สัญญาณจาก input หนึ่งตัวอาจเดินผ่าน inverter, AND และ OR ในจำนวนชั้นไม่เท่ากัน

เมื่อเส้นทางหนึ่งเปลี่ยนเร็วกว่าสัญญาณอีกเส้นทางหนึ่ง output อาจตกหรือเด้งชั่วคราว

### 8.9.1 Static-1 hazard

static-1 hazard เกิดเมื่อ output ควรคงเป็น `1` แต่กลับตกเป็น `0` ชั่วขณะ

มักพบในวงจรแบบ sum-of-products เช่น AND หลายตัวรวมด้วย OR

ตัวอย่างมาตรฐานคือ

```text
F = AB + A'C
```

ถ้า `B = 1` และ `C = 1`

เมื่อ `A = 0` จะได้

```text
F = 0·1 + 1·1 = 1
```

เมื่อ `A = 1` จะได้

```text
F = 1·1 + 0·1 = 1
```

ตามตรรกะ `F` ควรเป็น `1` ทั้งก่อนและหลัง `A` เปลี่ยน

แต่เส้นทาง `A'` ต้องผ่าน inverter ก่อนเข้า AND ทำให้พจน์ `A'C` อาจดับก่อนที่พจน์ `AB` จะติด

จึงเกิดช่องว่างสั้น ๆ ที่ทั้งสองพจน์เป็น `0`

```text
ก่อนแก้ hazard: F = AB + A'C, B=C=1, A เปลี่ยน 0 -> 1

A        ___────────────
A'       ───────___─────
AB       ________───────
A'C      ──────_________
F        ──────__───────
              glitch
```

### 8.9.2 Static-0 hazard

static-0 hazard เกิดเมื่อ output ควรคงเป็น `0` แต่กลับขึ้นเป็น `1` ชั่วขณะ

มักพบในวงจรแบบ product-of-sums เช่น OR หลายตัวรวมด้วย AND

หลักคิดคล้าย static-1 hazard แต่พิจารณาช่อง `0` บน K-map และการคาบเกี่ยวของ maxterm

### 8.9.3 Dynamic hazard

dynamic hazard เกิดเมื่อ output ควรเปลี่ยนครั้งเดียว เช่น `0 -> 1` แต่กลับแกว่งหลายครั้งก่อนนิ่ง

มักเกิดในวงจรหลายระดับที่มีเส้นทาง delay หลายเส้นซ้อนกัน

```text
output ที่ควรเกิด

Y ideal   ______────────────

output ที่มี dynamic hazard

Y real    ______─_─_────────
```

การแก้ dynamic hazard มักทำได้โดยลดโครงสร้างหลายระดับ, ใช้การออกแบบสองระดับที่ปลอด hazard สำหรับการเปลี่ยน input ทีละตัว หรือใช้ register จับค่าเมื่อสัญญาณนิ่งแล้ว

## 8.10 หา static hazard ด้วย K-map

K-map ไม่ได้ใช้แค่ลดรูปสมการให้สั้นที่สุด แต่ยังใช้มองหา hazard ได้ด้วย

สำหรับวงจร sum-of-products ให้พิจารณาช่องที่เป็น `1`

ถ้าช่อง `1` ที่อยู่ติดกันถูกครอบด้วยคนละกลุ่ม และสองกลุ่มนั้นไม่คาบเกี่ยวกัน จะมีโอกาสเกิด static-1 hazard เมื่อ input เปลี่ยนระหว่างสองช่องนั้น

วิธีแก้คือเพิ่มกลุ่มคาบเกี่ยวที่ครอบสองช่องดังกล่าว แม้พจน์นั้นจะซ้ำซ้อนเชิงพีชคณิต

พจน์ที่เพิ่มนี้เรียกว่า **consensus term**

### 8.10.1 K-map ของ F = AB + A'C

ใช้ตัวแปร `A, B, C`

วาง `A` เป็นแถว และ `BC` เป็นคอลัมน์ Gray code

```text
        BC
        00  01  11  10
      +----------------
A=0  |  0   1   1   0
A=1  |  0   0   1   1
```

พจน์ `A'C` ครอบช่องแถว `A=0` คอลัมน์ `BC=01,11`

พจน์ `AB` ครอบช่องแถว `A=1` คอลัมน์ `BC=11,10`

ช่อง `A=0,BC=11` กับ `A=1,BC=11` อยู่ติดกันในแนวดิ่ง เพราะต่างกันเฉพาะ `A`

แต่สองช่องนี้อยู่คนละกลุ่มเดิมและไม่มีการคาบเกี่ยวกัน จึงเกิด static-1 hazard เมื่อ `A` เปลี่ยน โดย `B=C=1`

เพิ่มกลุ่มแนวดิ่งที่คอลัมน์ `BC=11`

กลุ่มนี้ให้พจน์ `BC`

```text
        BC
        00  01  11  10
      +----------------
A=0  |  0   1  [1]  0
A=1  |  0   0  [1]  1
                  ↑
             consensus group = BC
```

สมการที่ปลอด static-1 hazard สำหรับการเปลี่ยน `A` ทีละตัวคือ

```text
F_safe = AB + A'C + BC
```

พจน์ `BC` เป็นพจน์ซ้ำซ้อนเชิงตรรกะ เพราะตามทฤษฎี consensus

```text
AB + A'C + BC = AB + A'C
```

แต่ในเชิงเวลา `BC` ช่วยค้ำ output ให้เป็น `1` ระหว่างที่ `AB` และ `A'C` สลับสถานะ

```text
หลังแก้ hazard: F = AB + A'C + BC, B=C=1, A เปลี่ยน 0 -> 1

A        ___────────────
A'       ───────___─────
AB       ________───────
A'C      ──────_________
BC       ───────────────
F        ───────────────
          ไม่มี glitch เพราะ BC ค้ำไว้
```

## 8.11 Setup time และ hold time: จุดเชื่อมไปวงจรเชิงลำดับ

บทนี้ยังไม่ลงรายละเอียด flip-flop แต่ต้องรู้ข้อจำกัดเวลาที่ทำให้วงจรเชิงผสมต่อกับ register ได้ถูกต้อง

**Setup time (`t_setup`)** คือเวลาขั้นต่ำที่ข้อมูล input ของ flip-flop ต้องนิ่งก่อนขอบ clock

**Hold time (`t_hold`)** คือเวลาขั้นต่ำที่ข้อมูล input ต้องนิ่งต่อหลังขอบ clock

```text
                 clock edge
                    │
CLK      ___________│──────────
                    │
D        ======= stable =======
          <setup>   │ <hold>
                    │
```

ถ้า output ของวงจรเชิงผสมมาถึงช้าเกินไป จะละเมิด setup time

ถ้า output ของวงจรเชิงผสมเปลี่ยนเร็วเกินไปหลัง clock edge อาจละเมิด hold time

แนวคิด critical path ในบทนี้จึงถูกใช้ต่อในบทวงจรเชิงลำดับเพื่อคำนวณ clock period ขั้นต่ำ

สูตรพื้นฐานสำหรับ setup constraint คือ

```text
T_clk >= t_clk-to-Q(max) + t_comb(max) + t_setup + t_skew
```

ส่วน hold constraint พิจารณา delay ขั้นต่ำ

```text
t_clk-to-Q(min) + t_comb(min) >= t_hold + t_skew
```

นักศึกษายังไม่จำเป็นต้องใช้สูตรเหล่านี้ลึกในบทนี้ แต่ควรเข้าใจว่า propagation delay ไม่ใช่เพียงตัวเลขประกอบ datasheet แต่เป็นตัวกำหนดความเร็วสูงสุดของระบบ clock

## 8.12 ตัวอย่างที่ 1: คำนวณ worst-case delay ของวงจรเลือกข้อมูล

โจทย์: วงจรหนึ่งเลือกข้อมูลจากสองแหล่งด้วย multiplexer ที่สร้างจากเกตพื้นฐาน

สมการคือ

```text
Y = S'A + SB
```

ใช้ inverter, AND และ OR โดยมีค่า delay สูงสุดดังนี้

| เกต | `t_pd(max)` |
|---|---:|
| NOT | 5 ns |
| AND | 8 ns |
| OR | 7 ns |

ต้องหา worst-case propagation delay จาก input แต่ละตัวถึง output `Y`

แผนภาพวงจร

```text
S ──┬──── NOT ──┐
   │           AND ──┐
A ─┴────────────┘    │
                     OR ── Y
S ──────────────┐    │
               AND ──┘
B ──────────────┘
```

เส้นทางจาก `A` ถึง `Y`

```text
A -> AND -> OR
delay = 8 + 7 = 15 ns
```

เส้นทางจาก `B` ถึง `Y`

```text
B -> AND -> OR
delay = 8 + 7 = 15 ns
```

เส้นทางจาก `S` ผ่านพจน์ `SB`

```text
S -> AND -> OR
delay = 8 + 7 = 15 ns
```

เส้นทางจาก `S` ผ่านพจน์ `S'A`

```text
S -> NOT -> AND -> OR
delay = 5 + 8 + 7 = 20 ns
```

ดังนั้น worst-case path คือเส้นทาง `S -> NOT -> AND -> OR`

```text
t_pd,worst = 20 ns
```

ถ้าวงจรนี้อยู่ระหว่าง register สองตัวและละเลย delay อื่นทั้งหมด ความถี่ clock ทางทฤษฎีที่เกิดจากวงจรเชิงผสมนี้เพียงอย่างเดียวประมาณได้จาก

```text
f_max ≈ 1 / 20 ns = 50 MHz
```

แต่ในการออกแบบจริงต้องบวก `t_clk-to-Q`, `t_setup`, clock skew และ margin เพิ่มเสมอ

## 8.13 ตัวอย่างที่ 2: ออกแบบการควบคุม tristate bus 4 อุปกรณ์

โจทย์: มีอุปกรณ์ 4 ตัว `R0, R1, R2, R3` ต้องส่งข้อมูล 8 บิตเข้าบัส `DATA[7:0]`

ใช้สัญญาณ select `S1 S0` เพื่อเลือกเพียงหนึ่งตัว

แต่ละอุปกรณ์มี output ผ่าน tristate buffer แบบ active-high enable

ต้องเขียนตารางควบคุม enable และอธิบายว่ากรณีใดทำให้ bus contention

ตาราง one-hot enable

| S1 | S0 | EN0 | EN1 | EN2 | EN3 | ผู้ขับ DATA bus |
|---:|---:|---:|---:|---:|---:|---|
| 0 | 0 | 1 | 0 | 0 | 0 | R0 |
| 0 | 1 | 0 | 1 | 0 | 0 | R1 |
| 1 | 0 | 0 | 0 | 1 | 0 | R2 |
| 1 | 1 | 0 | 0 | 0 | 1 | R3 |

แผนภาพ

```text
          S1 S0
            │
            ▼
      ┌───────────┐
      │ 2-to-4    │
      │ decoder   │
      └┬──┬──┬──┬─┘
       │  │  │  │
      EN0 EN1 EN2 EN3
       │  │  │  │
R0[7:0]▶3S │  │  │
R1[7:0]───▶3S │  │
R2[7:0]──────▶3S │
R3[7:0]─────────▶3S
       │  │  │  │
       └──┴──┴──┴──── DATA[7:0]
```

ถ้า decoder ทำงานถูกต้อง จะมี enable เป็น `1` เพียงเส้นเดียว

bus contention เกิดเมื่อมี enable เป็น `1` มากกว่าหนึ่งเส้น เช่น

```text
EN0 = 1, EN2 = 1
R0[3] = 0, R2[3] = 1
```

ที่บิต `DATA[3]` จะมี output หนึ่งตัวดึงลง LOW และอีกตัวดึงขึ้น HIGH พร้อมกัน

ผลที่ควรทำใน Logisim คือสังเกตสัญลักษณ์ error หรือ unknown บนสาย

ผลที่ต้องหลีกเลี่ยงใน hardware คือกระแสสูงและการร้อนของไอซี

เพื่อให้ปลอดภัยระหว่างเปลี่ยนตัวขับจาก `R0` ไป `R2` ควรสร้างลำดับควบคุมเป็น

```text
เวลา       t0        t1        t2
EN0        1─────────0─────────0
EN2        0─────────0─────────1
DATA       R0────────Z─────────R2
```

ช่วง `t1` คือเวลาที่ทุก output ปล่อยบัส

## 8.14 ตัวอย่างที่ 3: คำนวณ fan-out และ noise margin

โจทย์: output ของไอซีตัวหนึ่งต้องขับ input ของเกตตระกูลเดียวกันหลายตัว

datasheet ให้ค่ากระแสแบบ worst case ดังนี้

| พารามิเตอร์ | ความหมาย | ค่า |
|---|---|---:|
| `IOH(max)` | output จ่ายกระแสเมื่อ HIGH ได้ | 0.4 mA |
| `IIH(max)` | input หนึ่งตัวใช้กระแสเมื่อ HIGH | 20 uA |
| `IOL(max)` | output ดึงกระแสเมื่อ LOW ได้ | 8 mA |
| `IIL(max)` | input หนึ่งตัวใช้กระแสเมื่อ LOW | 0.4 mA |

คำนวณ fan-out ฝั่ง HIGH

```text
fan-out HIGH = 0.4 mA / 20 uA
             = 0.4 mA / 0.020 mA
             = 20
```

คำนวณ fan-out ฝั่ง LOW

```text
fan-out LOW = 8 mA / 0.4 mA
            = 20
```

ดังนั้น fan-out ที่ใช้ได้คือ

```text
fan-out = min(20, 20) = 20 input
```

ถ้าวงจรต้องขับ input 24 ตัว ต้องเพิ่ม buffer หรือแบ่งโหลดออกเป็นสองกลุ่ม

ต่อไปตรวจ noise margin จากค่าระดับแรงดัน

| พารามิเตอร์ | ค่า |
|---|---:|
| `VOH(min)` | 2.7 V |
| `VIH(min)` | 2.0 V |
| `VOL(max)` | 0.5 V |
| `VIL(max)` | 0.8 V |

```text
NMH = VOH(min) - VIH(min)
    = 2.7 - 2.0
    = 0.7 V

NML = VIL(max) - VOL(max)
    = 0.8 - 0.5
    = 0.3 V
```

ผลคือฝั่ง LOW มี margin น้อยกว่า

ถ้าสาย LOW รับ noise บวกเกิน 0.3 V อาจเข้าใกล้ขอบเขตที่ input ไม่รับประกันว่าเป็น LOW

ในการต่อ breadboard ที่ใช้สายยาว ควรลดความยาวสายสัญญาณ, ต่อกราวด์ร่วมให้แน่น, ใส่ decoupling capacitor ใกล้ไอซี และหลีกเลี่ยงการปล่อย input ลอย

## 8.15 ตัวอย่างที่ 4: วิเคราะห์และแก้ static-1 hazard จาก K-map

โจทย์: ให้ฟังก์ชัน

```text
F(A,B,C) = Σm(3,5,7)
```

ต้องสร้างสมการ SOP ที่ลดรูปด้วย K-map แล้วตรวจว่ามี static-1 hazard หรือไม่

เขียน K-map โดยใช้คอลัมน์ `BC = 00, 01, 11, 10`

```text
        BC
        00  01  11  10
      +----------------
A=0  |  0   0   1   0
A=1  |  0   1   1   0
```

minterm ที่เป็น `1` คือ

```text
m3 = A'BC
m5 = AB'C
m7 = ABC
```

จัดกลุ่มเพื่อลดรูป

กลุ่มที่ 1: ช่อง `m3` กับ `m7` อยู่ในคอลัมน์ `BC=11` ต่างกันเฉพาะ `A`

```text
พจน์ = BC
```

กลุ่มที่ 2: ช่อง `m5` กับ `m7` อยู่ในแถว `A=1` คอลัมน์ `BC=01,11` ต่างกันเฉพาะ `B`

```text
พจน์ = AC
```

สมการลดรูปคือ

```text
F = BC + AC
```

หรือ

```text
F = C(A + B)
```

ตรวจ hazard: สองกลุ่มคาบเกี่ยวกันที่ `m7`

ดังนั้นการเปลี่ยนจาก `m3` ไป `m7` มีพจน์ `BC` ค้ำไว้ และการเปลี่ยนจาก `m5` ไป `m7` มีพจน์ `AC` ค้ำไว้

แต่ต้องตรวจช่อง `m3` กับ `m5` ด้วย

`m3 = 011` และ `m5 = 101` ต่างกันสองตัวแปรคือ `A` และ `B`

การเปลี่ยนสองตัวแปรพร้อมกันไม่ใช่กรณี static hazard แบบ single-input change ที่ K-map รับประกัน

ดังนั้นสมการนี้ไม่มี static-1 hazard สำหรับการเปลี่ยน input ทีละตัว

เปรียบเทียบกับฟังก์ชัน

```text
G(A,B,C) = AB + A'C
```

K-map ของ `G` มีช่อง `1` ติดกันที่ `BC=11` แต่กลุ่มเดิมไม่คาบเกี่ยว จึงต้องเติม `BC`

```text
G_safe = AB + A'C + BC
```

ขั้นตอนตรวจ hazard ที่ควรใช้ในโจทย์สอบคือ

1. วาด K-map และจัดกลุ่มตามสมการที่ใช้จริงในวงจร
2. ดูช่อง `1` ที่ติดกันทุกคู่
3. ถ้าคู่ใดไม่ได้อยู่ในกลุ่มเดียวกันหรือกลุ่มที่คาบเกี่ยวกัน ให้เติมกลุ่ม consensus
4. แปลงกลุ่มที่เติมเป็นพจน์เพิ่มในสมการ
5. ย้ำว่าพจน์ที่เติมไม่เปลี่ยนตารางความจริง แต่เปลี่ยนพฤติกรรมช่วงเปลี่ยนสัญญาณ

## 8.16 แบบฝึกหัดท้ายบท

1. วาดสัญลักษณ์และตารางความจริงของ tristate buffer แบบ active-high enable และ active-low enable พร้อมอธิบายความหมายของ `Z` ด้วยคำของตนเอง

2. บัสหนึ่งเส้นมีไดรเวอร์สองตัว `A_DRV` และ `B_DRV` โดย `A_DRV` ต้องขับข้อมูลเมื่อ `SEL=0` และ `B_DRV` ต้องขับข้อมูลเมื่อ `SEL=1` จงออกแบบสัญญาณ enable ที่ไม่ทำให้ทั้งสองตัวขับพร้อมกัน และวาด timing diagram ตอน `SEL` เปลี่ยนจาก `0` เป็น `1`

3. วงจรมีเส้นทางจาก input `X` ไป output `Y` สองเส้นทาง เส้นทางที่หนึ่งผ่าน NOT, AND, OR และเส้นทางที่สองผ่าน NAND, NAND โดยมี delay `NOT=4 ns`, `AND=9 ns`, `OR=7 ns`, `NAND=6 ns` จงหา worst-case delay และระบุ critical path

4. ไอซีตัวหนึ่งมี `IOH=0.8 mA`, `IIH=40 uA`, `IOL=12 mA`, `IIL=0.6 mA` จงคำนวณ fan-out ฝั่ง HIGH, fan-out ฝั่ง LOW และ fan-out ที่ใช้ได้จริง

5. ให้ `VOH(min)=4.4 V`, `VIH(min)=3.5 V`, `VOL(max)=0.3 V`, `VIL(max)=1.5 V` จงคำนวณ `NMH` และ `NML` พร้อมอธิบายว่าฝั่งใดทน noise ได้น้อยกว่า

6. วิเคราะห์ฟังก์ชัน `F(A,B,C)=A'B + AC` ว่ามี static-1 hazard หรือไม่ โดยวาด K-map ระบุตำแหน่งช่องที่เป็นปัญหา และเขียนสมการหลังเติม consensus term ถ้ามี

7. สำหรับฟังก์ชัน product-of-sums `F=(A+B)(A'+C)` จงอธิบายว่า static-0 hazard อาจเกิดเมื่อ input ใดเปลี่ยน และควรพิจารณาช่อง `0` บน K-map อย่างไร

8. อธิบายเหตุผลที่วงจร combinational ที่มี glitch อาจยังใช้งานได้ถ้า output ถูกจับด้วย flip-flop หลังจากสัญญาณนิ่งแล้ว แต่จะเป็นอันตรายถ้า glitch ไปเป็น clock หรือ asynchronous reset

9. ใน Logisim Evolution ให้สร้างวงจร `F=AB+A'C` และวงจร `F=AB+A'C+BC` แล้วใส่ค่า delay ให้เกตแต่ละตัว เปรียบเทียบ waveform เมื่อ `B=C=1` และ `A` เปลี่ยนจาก `0` เป็น `1`

10. ออกแบบ data bus 4 บิตที่มีแหล่งข้อมูล 3 แหล่ง โดยใช้ tristate buffer และ decoder สร้าง enable จงวาดแผนภาพวงจรระดับบล็อก ตาราง enable และระบุกรณี bus contention ที่ต้องห้ามอย่างน้อย 2 กรณี

## 8.17 สาระที่ต้องนำไปใช้ต่อในบทวงจรเชิงลำดับ

เมื่อนักศึกษาออกแบบวงจรเชิงผสมในบทถัดไป ต้องไม่มองเฉพาะตารางความจริงและสมการลดรูป

วงจรที่นำไปต่อกับ flip-flop, register และ counter ต้องมี output นิ่งทันก่อน clock edge และต้องไม่สร้าง glitch ที่กลายเป็นสัญญาณควบคุมสำคัญ

ไตรสเตดบัฟเฟอร์ช่วยให้หลายอุปกรณ์แชร์บัสได้ แต่ต้องมีสัญญาณ enable แบบ one-hot และมีช่วงปิด output ระหว่างเปลี่ยนผู้ขับ

propagation delay บอกว่าเส้นทางข้อมูลช้าที่สุดใช้เวลานานเท่าใด

fan-out และ noise margin บอกว่า output หนึ่งตัวขับวงจรปลายทางได้จริงหรือไม่

hazard analysis บอกว่าสมการที่ถูกต้องเชิงบูลีนอาจยังมีพฤติกรรมผิดชั่วขณะเมื่อสัญญาณเดินทางไม่พร้อมกัน

ความเข้าใจทั้งสี่เรื่องนี้เป็นฐานของการคำนวณ clock period, setup time, hold time และการออกแบบ datapath/control path ในคอมพิวเตอร์ดิจิทัล
