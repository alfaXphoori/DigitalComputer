## 5.1 ปัญหาที่ทำให้ต้องใช้วิธีการใช้ตาราง

ในบทก่อนหน้า ผู้เรียนใช้ผังคาร์โนห์ (Karnaugh Map - K-map) เพื่อลดรูปสมการบูลีนจากตารางความจริง
วิธีนั้นดีมากเมื่อมีตัวแปร 2, 3 หรือ 4 ตัว เพราะผู้เรียนมองเห็นช่องที่อยู่ติดกันด้วยตา
และสามารถจับกลุ่มขนาด 1, 2, 4, 8 หรือ 16 ช่องได้โดยอาศัยลำดับ Gray code
แต่เมื่อจำนวนตัวแปรเพิ่มเป็น 5 ตัวขึ้นไป K-map เริ่มกลายเป็นงานที่ผิดพลาดง่าย
เพราะต้องมองเป็นแผนที่สองแผ่นหรือหลายแผ่นที่เชื่อมกันตามตำแหน่งตัวแปรที่เพิ่มเข้ามา

ตัวอย่างเช่น ฟังก์ชัน 6 ตัวแปรมีแถวอินพุตทั้งหมด 64 แถว
ถ้าวาด K-map จะต้องมี 64 ช่อง และการจับกลุ่มต้องพิจารณาการติดกันในหลายมิติ
ช่องที่อยู่คนละแผ่นอาจติดกันทางตรรกะได้ แม้ไม่ได้อยู่ติดกันบนกระดาษ
สำหรับนักศึกษาปี 1 การพลาดเพียงหนึ่งกลุ่มอาจทำให้สมการที่ได้ไม่ minimal
หรือเลวร้ายกว่านั้นคือครอบคลุม minterm ผิดและทำให้วงจรทำงานผิดฟังก์ชัน

วิธีการใช้ตาราง หรือ **Quine-McCluskey method** เป็นวิธีลดรูปฟังก์ชันบูลีนเชิงระบบ
คำว่าเชิงระบบหมายถึงทุกขั้นตอนมีกฎชัดเจนและทำซ้ำได้
ไม่ต้องอาศัยสายตาในการมองกลุ่มบนแผนที่
จึงเหมาะกับฟังก์ชันที่มีตัวแปรมากกว่า 5 ตัว
เหมาะกับการตรวจข้อสอบที่ต้องการแสดงขั้นตอน
และเป็นพื้นฐานของแนวคิดในโปรแกรมลดรูปตรรกะ (logic minimization program)

แก่นของวิธีนี้คือการรวม minterm ที่ต่างกันเพียง 1 บิต
เมื่อต่างกันเพียง 1 บิต ตัวแปรตำแหน่งนั้นจะถูกตัดออกได้ตามกฎบูลีน

```text
AB'C + ABC = AC(B' + B) = AC
```

ถ้าเขียนเป็นรูปบิต เช่น `101` และ `111`
สองพจน์นี้ต่างกันที่บิตกลางเพียงตำแหน่งเดียว
จึงรวมเป็น `1-1`
เครื่องหมาย `-` อ่านว่า "ตำแหน่งนี้ไม่สนใจค่า" หรือ variable นั้นถูกตัดออกจากพจน์แล้ว
ไม่ใช่สัญญาณไม่ทราบค่าในวงจรจริง

บทนี้จึงไม่เน้นการวาดภาพ K-map
แต่เน้นการจัดกลุ่มด้วยจำนวนเลข 1
การจับคู่ term
การทำเครื่องหมาย term ที่ถูกใช้รวมแล้ว
การหา prime implicant
และการเลือก prime implicant ที่ครอบคลุม minterm ของฟังก์ชันครบถ้วนโดยใช้จำนวนน้อยที่สุด

## 5.2 ศัพท์เฉพาะของ Quine-McCluskey

ก่อนลงขั้นตอน ต้องแยกศัพท์สำคัญให้ชัด เพราะคำตอบของวิธีตารางไม่ได้เกิดจากพจน์เดียว
แต่เกิดจากการคัดเลือกพจน์หลายระดับ

| ศัพท์ | ความหมายในวิธีตาราง | ตัวอย่าง |
|---|---|---|
| Minterm | แถวอินพุตที่ฟังก์ชันมีค่า 1 | `m5 = 0101` สำหรับ `A=0,B=1,C=0,D=1` |
| Don't care | แถวอินพุตที่อนุญาตให้ output เป็น 0 หรือ 1 เพื่อช่วยลดรูป | BCD invalid code 1010-1111 |
| Implicant | พจน์ที่ครอบคลุม minterm อย่างน้อยหนึ่งตัวของฟังก์ชัน | `0-01` ครอบคลุม `0001` และ `0101` |
| Prime implicant (PI) | implicant ที่รวมต่อให้ใหญ่ขึ้นไม่ได้แล้ว | `-00-` ครอบคลุม 0,1,8,9 |
| Essential prime implicant (EPI) | PI ที่ครอบคลุม minterm บางตัวเพียง PI เดียว | ถ้า minterm 9 มีเฉพาะ `-00-` ครอบคลุม `-00-` เป็น EPI |
| Prime implicant chart | ตารางแสดงว่า PI แต่ละตัวครอบคลุม minterm ใด | แถวเป็น PI คอลัมน์เป็น minterm |
| Cover | ชุด PI ที่เลือกแล้วครอบคลุม minterm ที่ต้องเป็น 1 ครบทุกตัว | `{B'C', CD', A'BD}` |

การอ่าน pattern แบบมี dash ต้องอ่านทีละตัวแปรจากซ้ายไปขวา
ถ้าใช้ตัวแปร `A,B,C,D` pattern `-00-` แปลว่า `A` ไม่สนใจ, `B=0`, `C=0`, `D` ไม่สนใจ
ดังนั้นพจน์คือ `B'C'`

```text
pattern: - 0 0 -
ตัวแปร:  A B C D
พจน์:      B' C'
```

อีกตัวอย่าง pattern `01-1` แปลว่า `A=0`, `B=1`, `C` ไม่สนใจ, `D=1`
ดังนั้นพจน์คือ `A'BD`

```text
pattern: 0 1 - 1
ตัวแปร:  A B C D
พจน์:    A'B  D
```

จุดที่นักศึกษามักสับสนคือ prime implicant ไม่จำเป็นต้องถูกเลือกทุกตัว
PI คือพจน์ที่มีสิทธิ์ถูกเลือก
แต่คำตอบสุดท้ายต้องเลือกเพียงบาง PI ที่ครอบคลุม minterm ของฟังก์ชันครบ
และถ้าเลือกได้หลายชุด ต้องเลือกชุดที่มีต้นทุนต่ำที่สุดตามเกณฑ์ที่โจทย์กำหนด

## 5.3 หลักการรวมพจน์: ต่างกันหนึ่งบิตจึงรวมได้

Quine-McCluskey อาศัยกฎพื้นฐานของพีชคณิตบูลีน
ถ้าพจน์สองพจน์เหมือนกันทุกตัวแปรยกเว้นตัวแปรเดียว
ตัวแปรที่ต่างกันจะหายไปเมื่อรวมด้วย OR

```text
A'B'C'D' + A'B'C'D
= A'B'C'(D' + D)
= A'B'C'
```

ในรูปเลขฐานสอง:

```text
0000 + 0001 = 000-
```

เพราะ `0000` และ `0001` ต่างกันเฉพาะบิต `D`
ดังนั้น `D` ถูกแทนด้วย `-`

ตัวอย่างที่รวมไม่ได้:

```text
0000 กับ 0011 ต่างกัน 2 บิต คือ C และ D
0000 กับ 1010 ต่างกัน 2 บิต คือ A และ C
0-01 กับ 1-10 ต่างกัน 3 ตำแหน่งที่พิจารณาจริง คือ A, C, D
```

การรวมในรอบถัดไปต้องระวังว่า dash ต้องอยู่ในตำแหน่งเดียวกันจึงจะรวมต่อได้
เช่น `000-` รวมกับ `100-` ได้ เพราะต่างกันเฉพาะ `A`
และ dash อยู่ที่ตำแหน่ง `D` เหมือนกัน

```text
000- + 100- = -00-
```

แต่ `000-` รวมกับ `00-0` ไม่ได้
ถึงแม้จะดูเหมือนต่างกันหนึ่งตำแหน่ง
เพราะ dash อยู่คนละตำแหน่ง และจริง ๆ แล้วครอบคลุมชุด minterm คนละรูปทรงทางตรรกะ

```text
000- ครอบคลุม 0000, 0001
00-0 ครอบคลุม 0000, 0010
ถ้ารวมผิดเป็น 00-- จะครอบคลุม 0011 เพิ่มเข้ามา
ซึ่งอาจไม่ใช่ minterm ของฟังก์ชัน
```

กฎนี้สำคัญมากเมื่อนำไปเขียนโปรแกรม
โปรแกรมจะตรวจ pattern ทีละตำแหน่ง
ถ้ามีตำแหน่งที่เป็น `-` ไม่ตรงกัน จะไม่อนุญาตให้รวม
และถ้ามีตำแหน่งที่เป็น 0/1 ต่างกันมากกว่าหนึ่งตำแหน่ง ก็ไม่อนุญาตให้รวมเช่นกัน

## 5.4 ขั้นตอนวิธี Quine-McCluskey แบบเต็ม

ขั้นตอนต่อไปนี้ใช้กับฟังก์ชันแบบ Sum of Minterms
เป้าหมายคือหา minimal SOP หรือผลบวกของพจน์ AND ที่ลดรูปแล้ว

### 5.4.1 เขียน minterm เป็นเลขฐานสอง

กำหนดจำนวนตัวแปรก่อน
ถ้ามี 4 ตัวแปร `A,B,C,D` ต้องเขียน minterm ทุกตัวเป็นเลขฐานสอง 4 บิต
ถ้ามี 5 ตัวแปร `A,B,C,D,E` ต้องเขียนเป็น 5 บิต
บิตซ้ายสุดแทนตัวแปรตัวแรกตามลำดับที่โจทย์ให้

```text
m0  = 0000
m1  = 0001
m2  = 0010
m5  = 0101
m14 = 1110
```

ห้ามเขียน `m5 = 101` ในโจทย์ 4 ตัวแปร
เพราะจะทำให้ตำแหน่งตัวแปรคลาดเคลื่อน
ต้องเขียนเป็น `0101` เสมอ

### 5.4.2 จัดกลุ่มตามจำนวนเลข 1

นำ minterm ที่เขียนเป็นฐานสองแล้วจัดเข้ากลุ่มตามจำนวนบิตที่เป็น 1
เช่น `0000` อยู่กลุ่ม 0
`0001`, `0010`, `1000` อยู่กลุ่ม 1
และ `0111` อยู่กลุ่ม 3

เหตุผลที่จัดแบบนี้คือพจน์ที่จะรวมกันได้ต้องต่างกันหนึ่งบิต
ดังนั้นจำนวนเลข 1 ของสองพจน์ต้องต่างกัน 1 ด้วย
เราจึงเปรียบเทียบเฉพาะกลุ่มติดกัน เช่น กลุ่ม 0 กับ 1, กลุ่ม 1 กับ 2, กลุ่ม 2 กับ 3
ไม่ต้องเปรียบเทียบกลุ่ม 0 กับ 3 เพราะต่างกันอย่างน้อย 3 บิตแน่นอน

### 5.4.3 จับคู่กลุ่มติดกันและแทนบิตที่ต่างด้วย dash

เปรียบเทียบทุกพจน์ในกลุ่มหนึ่งกับทุกพจน์ในกลุ่มถัดไป
ถ้าต่างกันเพียงหนึ่งตำแหน่ง ให้สร้างพจน์ใหม่โดยแทนตำแหน่งนั้นด้วย `-`
และทำเครื่องหมายพจน์เดิมว่า "ถูกใช้รวมแล้ว"

```text
0000 กับ 0001 ต่างกันที่ D -> 000-
0000 กับ 0010 ต่างกันที่ C -> 00-0
0001 กับ 0101 ต่างกันที่ B -> 0-01
```

เครื่องหมายถูกมีความหมายเฉพาะในรอบนั้น
ถ้าพจน์เดิมถูกใช้รวมอย่างน้อยหนึ่งครั้ง แปลว่าพจน์เดิมยังไม่ใช่ prime implicant
เพราะยังขยายให้ใหญ่ขึ้นได้
พจน์ใดที่ไม่ถูกใช้รวมเลยในรอบนั้น จะกลายเป็น candidate ของ prime implicant

### 5.4.4 ทำซ้ำรอบการรวม

นำพจน์ที่มี dash แล้วมาจัดกลุ่มใหม่ตามจำนวนเลข 1 ที่ยังเหลืออยู่
จากนั้นรวมเฉพาะพจน์ที่ dash อยู่ตำแหน่งเดียวกันและต่างกันหนึ่งบิตจริง
ทำซ้ำจนไม่มีพจน์ใดรวมต่อได้

ตัวอย่าง:

```text
000- กับ 100- -> -00-
00-0 กับ 10-0 -> -0-0
0-10 กับ 1-10 -> --10
```

ถ้ารอบใดไม่มีการรวมเพิ่ม
พจน์ทั้งหมดที่เหลือในรอบนั้นเป็น prime implicant

### 5.4.5 สร้าง prime implicant chart

หลังจากหา PI ได้ทั้งหมดแล้ว
สร้างตารางที่แถวเป็น PI และคอลัมน์เป็น minterm ที่ฟังก์ชันต้องมีค่า 1
ถ้า PI ใดครอบคลุม minterm ใด ให้ทำเครื่องหมาย `x`

```text
          minterm ของ F
PI        0  1  2  5  6  7  8  9 10 14
-----------------------------------------
-00-      x  x                 x  x
```

ตารางนี้เป็นหัวใจของขั้นเลือกคำตอบ
เพราะแสดงให้เห็นว่าพจน์ใดครอบคลุมแถวใดบ้าง
และ minterm ใดถูกครอบคลุมโดยพจน์เดียวเท่านั้น

### 5.4.6 หา essential prime implicant

ดูทีละคอลัมน์ของ PI chart
ถ้าคอลัมน์ใดมีเครื่องหมาย `x` เพียงช่องเดียว
PI ในช่องนั้นเป็น essential prime implicant
ต้องเลือกเข้าคำตอบสุดท้ายทันที
เพราะไม่มี PI อื่นครอบคลุม minterm นั้นได้

หลังเลือก EPI แล้ว
ให้ขีดทับ minterm ทั้งหมดที่ EPI นั้นครอบคลุม
จากนั้นพิจารณาเฉพาะ minterm ที่ยังไม่ถูกครอบคลุม

### 5.4.7 เลือกชุดครอบคลุมที่เหลือให้น้อยที่สุด

ถ้ายังมี minterm เหลือหลังเลือก EPI
ต้องเลือก PI เพิ่ม
เกณฑ์พื้นฐานคือเลือกให้จำนวนพจน์น้อยที่สุด
ถ้าจำนวนพจน์เท่ากัน ให้เลือกชุดที่มีจำนวน literal น้อยกว่า
ถ้ายังเท่ากันอีก ในงานต่อวงจรจริงอาจเลือกชุดที่ใช้ IC 74xx น้อยกว่า
หรือมีระดับเกต (logic level) น้อยกว่า

ขั้นนี้เป็นจุดที่บางโจทย์มีคำตอบ minimal ได้มากกว่าหนึ่งแบบ
ในระดับบทนี้จะเลือกโดยดูจาก PI chart โดยตรง
ส่วนกรณีที่ซับซ้อนมากจะใช้ Petrick's method ซึ่งเกริ่นไว้ท้ายบท

## 5.5 ตัวอย่างเต็ม: ลดรูปฟังก์ชัน 4 ตัวแปรด้วยทุกตาราง

ให้

```text
F(A,B,C,D) = Σm(0,1,2,5,6,7,8,9,10,14)
```

ต้องหา minimal SOP ด้วยวิธี Quine-McCluskey
ลำดับตัวแปรคือ `A B C D`
ดังนั้น minterm ทุกตัวต้องเขียนเป็นเลขฐานสอง 4 บิต

### 5.5.1 ตาราง minterm และเลขฐานสอง

```text
+---------+------+------+------+------+-----------+
| minterm |  A   |  B   |  C   |  D   | binary    |
+---------+------+------+------+------+-----------+
|   m0    |  0   |  0   |  0   |  0   | 0000      |
|   m1    |  0   |  0   |  0   |  1   | 0001      |
|   m2    |  0   |  0   |  1   |  0   | 0010      |
|   m5    |  0   |  1   |  0   |  1   | 0101      |
|   m6    |  0   |  1   |  1   |  0   | 0110      |
|   m7    |  0   |  1   |  1   |  1   | 0111      |
|   m8    |  1   |  0   |  0   |  0   | 1000      |
|   m9    |  1   |  0   |  0   |  1   | 1001      |
|   m10   |  1   |  0   |  1   |  0   | 1010      |
|   m14   |  1   |  1   |  1   |  0   | 1110      |
+---------+------+------+------+------+-----------+
```

### 5.5.2 รอบที่ 0: จัดกลุ่มตามจำนวนเลข 1

```text
+---------+-------------+--------------------+
| จำนวน 1 | minterm     | binary             |
+---------+-------------+--------------------+
|    0    | m0          | 0000               |
|    1    | m1          | 0001               |
|         | m2          | 0010               |
|         | m8          | 1000               |
|    2    | m5          | 0101               |
|         | m6          | 0110               |
|         | m9          | 1001               |
|         | m10         | 1010               |
|    3    | m7          | 0111               |
|         | m14         | 1110               |
+---------+-------------+--------------------+
```

เฉพาะกลุ่มที่อยู่ติดกันเท่านั้นที่ต้องเปรียบเทียบ
กลุ่ม 0 เทียบกับกลุ่ม 1
กลุ่ม 1 เทียบกับกลุ่ม 2
และกลุ่ม 2 เทียบกับกลุ่ม 3

### 5.5.3 รอบรวมที่ 1: รวม minterm เป็นพจน์ที่มี dash หนึ่งตำแหน่ง

```text
+------------+------------+----------+----------------+-------+

+------------+------------+----------+----------------+-------+
| 0000 (m0)  | 0001 (m1)  | 000-     | 0,1            |  ✓    |
| 0000 (m0)  | 0010 (m2)  | 00-0     | 0,2            |  ✓    |
| 0000 (m0)  | 1000 (m8)  | -000     | 0,8            |  ✓    |
| 0001 (m1)  | 0101 (m5)  | 0-01     | 1,5            |  ✓    |
| 0001 (m1)  | 1001 (m9)  | -001     | 1,9            |  ✓    |
| 0010 (m2)  | 0110 (m6)  | 0-10     | 2,6            |  ✓    |
| 0010 (m2)  | 1010 (m10) | -010     | 2,10           |  ✓    |

| 1000 (m8)  | 1010 (m10) | 10-0     | 8,10           |  ✓    |
| 0101 (m5)  | 0111 (m7)  | 01-1     | 5,7            |  ✓    |
```

ในรอบนี้ minterm เดิมทุกตัวถูกใช้รวมอย่างน้อยหนึ่งครั้ง
จึงยังไม่มี minterm เดี่ยวใดเป็น prime implicant

### 5.5.4 จัดกลุ่มพจน์จากรอบที่ 1

จำนวนเลข 1 ให้นับเฉพาะตำแหน่งที่เป็น `1`
ไม่ต้องนับ dash

```text
+---------+----------+----------------+

+---------+----------+----------------+

|         | 00-0     | 0,2            |
|         | -000     | 0,8            |
|    1    | 0-01     | 1,5            |
|         | -001     | 1,9            |
|         | 0-10     | 2,6            |
|         | -010     | 2,10           |
|         | 100-     | 8,9            |
|         | 10-0     | 8,10           |
|    2    | 01-1     | 5,7            |
|         | 011-     | 6,7            |
|         | -110     | 6,14           |
|         | 1-10     | 10,14          |
+---------+----------+----------------+
```

### 5.5.5 รอบรวมที่ 2: รวมพจน์ที่มี dash ตำแหน่งเดียวกัน

```text
+-----------+-----------+----------+------------------+-------+
| term ที่ 1 | term ที่ 2 | ผลรวม    | ครอบคลุม          | ใช้?  |
+-----------+-----------+----------+------------------+-------+
| 000-      | 100-      | -00-     | 0,1,8,9          |  ✓    |
| 00-0      | 10-0      | -0-0     | 0,2,8,10         |  ✓    |
| -000      | -001      | -00-     | 0,1,8,9          |  ✓    |
| -000      | -010      | -0-0     | 0,2,8,10         |  ✓    |
| 0-10      | 1-10      | --10     | 2,6,10,14        |  ✓    |
| -010      | -110      | --10     | 2,6,10,14        |  ✓    |
+-----------+-----------+----------+------------------+-------+
```

บางผลรวมเกิดซ้ำ เช่น `-00-` และ `--10`
ให้เก็บเพียงหนึ่งรายการ
แต่ต้องเข้าใจว่ามันอาจเกิดจากเส้นทางการรวมหลายทาง
การเกิดซ้ำไม่ได้ทำให้ต้องใส่พจน์ซ้ำในคำตอบ

พจน์จากรอบที่ 1 ที่ไม่ได้ถูกใช้ในรอบที่ 2 จะเป็น prime implicant:

```text
0-01   ครอบคลุม 1,5
01-1   ครอบคลุม 5,7
011-   ครอบคลุม 6,7
```

### 5.5.6 รอบรวมที่ 3: ตรวจว่ารวมต่อได้หรือไม่

พจน์ที่ได้จากรอบที่ 2 คือ

```text
-00-   ครอบคลุม 0,1,8,9
-0-0   ครอบคลุม 0,2,8,10
--10   ครอบคลุม 2,6,10,14
```

ลองตรวจการรวมต่อ:

```text
-00- กับ -0-0  รวมไม่ได้ เพราะ dash อยู่คนละตำแหน่งและจะเพิ่ม minterm ที่ไม่ควรครอบคลุม
-00- กับ --10  รวมไม่ได้ เพราะตำแหน่งที่เป็น 0/1 ต่างกันมากกว่าหนึ่งเงื่อนไข
-0-0 กับ --10  รวมไม่ได้ เพราะ dash ไม่อยู่ชุดตำแหน่งเดียวกัน
```

ดังนั้นพจน์ทั้งสามนี้เป็น prime implicant ด้วย

### 5.5.7 สรุป prime implicant ทั้งหมด

```text
+------+----------+------------------+--------------------------+
| ลำดับ | pattern  | พจน์บูลีน        | minterm ที่ครอบคลุม       |
+------+----------+------------------+--------------------------+
| P1   | 0-01     | A'C'D            | 1,5                      |
| P2   | 01-1     | A'BD             | 5,7                      |
| P3   | 011-     | A'BC             | 6,7                      |
| P4   | -00-     | B'C'             | 0,1,8,9                  |
| P5   | -0-0     | B'D'             | 0,2,8,10                 |
| P6   | --10     | CD'              | 2,6,10,14                |
+------+----------+------------------+--------------------------+
```

วิธีแปลง pattern เป็นพจน์:

```text
0-01 -> A=0, B=-, C=0, D=1 -> A'C'D
01-1 -> A=0, B=1, C=-, D=1 -> A'BD
011- -> A=0, B=1, C=1, D=- -> A'BC
-00- -> A=-, B=0, C=0, D=- -> B'C'
-0-0 -> A=-, B=0, C=-, D=0 -> B'D'
--10 -> A=-, B=-, C=1, D=0 -> CD'
```

### 5.5.8 สร้าง prime implicant chart

ใน chart ให้ใส่เฉพาะ minterm ที่เป็น 1 ของฟังก์ชัน
ถ้ามี don't care จะไม่ใส่เป็นคอลัมน์ที่ต้องครอบคลุมในคำตอบสุดท้าย
ตัวอย่างนี้ไม่มี don't care

```text
| P1   | 0-01     |   | x |   | x |   |   |   |   |    |    |
| P2   | 01-1     |   |   |   | x |   | x |   |   |    |    |
| P3   | 011-     |   |   |   |   | x | x |   |   |    |    |
| P4   | -00-     | x | x |   |   |   |   | x | x |    |    |
| P5   | -0-0     | x |   | x |   |   |   | x |   | x  |    |
| P6   | --10     |   |   | x |   | x |   |   |   | x  | x  |
+------+----------+---+---+---+---+---+---+---+---+----+----+
```

### 5.5.9 หา essential prime implicant

ตรวจทีละคอลัมน์:

```text
minterm 0  ถูกครอบคลุมโดย P4, P5
minterm 1  ถูกครอบคลุมโดย P1, P4
minterm 2  ถูกครอบคลุมโดย P5, P6
minterm 5  ถูกครอบคลุมโดย P1, P2
minterm 6  ถูกครอบคลุมโดย P3, P6
minterm 7  ถูกครอบคลุมโดย P2, P3
minterm 8  ถูกครอบคลุมโดย P4, P5
minterm 9  ถูกครอบคลุมโดย P4 เท่านั้น
minterm 10 ถูกครอบคลุมโดย P5, P6
minterm 14 ถูกครอบคลุมโดย P6 เท่านั้น
```

ดังนั้น

```text
P4 = -00- = B'C' เป็น essential prime implicant เพราะครอบคลุม m9 เพียงตัวเดียว
P6 = --10 = CD'  เป็น essential prime implicant เพราะครอบคลุม m14 เพียงตัวเดียว
```

เลือก P4 และ P6 ก่อน

```text
P4 ครอบคลุม 0,1,8,9
P6 ครอบคลุม 2,6,10,14
```

minterm ที่ถูกครอบคลุมแล้ว:

```text
0,1,2,6,8,9,10,14
```

minterm ที่ยังเหลือ:

```text
5,7
```

### 5.5.10 เลือก PI เพิ่มเพื่อครอบคลุม minterm ที่เหลือ

จาก PI chart:

```text
minterm 5 ครอบคลุมได้โดย P1 หรือ P2
minterm 7 ครอบคลุมได้โดย P2 หรือ P3
```

PI ที่ครอบคลุมทั้ง 5 และ 7 พร้อมกันคือ P2

```text
P2 = 01-1 = A'BD
```

ดังนั้นชุดครอบคลุมสุดท้ายคือ

```text
P4, P6, P2
```

แปลงเป็นสมการ:

```text
F(A,B,C,D) = B'C' + CD' + A'BD
```

```text
|   0     | 0000   |   1   |   0    |    0     |
|   1     | 0001   |   1   |   0    |    0     |
|   2     | 0010   |   0   |   1    |    0     |
|   5     | 0101   |   0   |   0    |    1     |
|   6     | 0110   |   0   |   1    |    0     |
|   7     | 0111   |   0   |   0    |    1     |
```

และไม่มีพจน์ใดในคำตอบครอบคลุม minterm ที่ไม่อยู่ในฟังก์ชันโดยไม่ได้รับอนุญาต

### 5.5.12 แผนภาพวงจรจากคำตอบ minimal SOP

สมการ

```text
F = B'C' + CD' + A'BD
```

สามารถต่อเป็นวงจร AND-OR ได้ดังนี้

```text
      B ──┬──[NOT]── B' ──┐
          │               │
      C ──┼──[NOT]── C' ──┼──[AND]── term1 = B'C' ──┐
          │               │                         │
          │               └─────────────────────────┤
      C ─────────────────────[AND]── term2 = CD' ───┼──[OR]── F
      D ──[NOT]── D' ─────────┘                     │
                                                    │
      A ──[NOT]── A' ──┐                            │
      B ───────────────┼──[AND]── term3 = A'BD ─────┘
      D ───────────────┘
```

ถ้าต่อด้วยไอซีตระกูล 74xx แบบพื้นฐาน
ต้องมี inverter สำหรับ `A'`, `B'`, `C'`, `D'`
มี AND gate สำหรับพจน์สองตัวแปรสองพจน์
มี AND gate สามอินพุตหนึ่งพจน์
และมี OR gate รวมสามพจน์
ถ้าใช้เฉพาะ 74LS08 ซึ่งเป็น AND 2-input จะต้องสร้าง AND 3-input จาก AND 2-input สองตัว
และถ้าใช้ 74LS32 ซึ่งเป็น OR 2-input จะต้องรวมสามพจน์ด้วย OR สองชั้น

```text
term3 = A'BD

A' ──┐
     ├─[AND]─ X ─┐
B  ──┘          ├─[AND]─ term3
D  ─────────────┘
```

จำนวน gate ที่ใช้จริงจึงขึ้นกับชนิดไอซีที่อนุญาต
ไม่ใช่ขึ้นกับจำนวนพจน์ในสมการเพียงอย่างเดียว

## 5.6 การจัดการ don't care ในวิธีตาราง

ในวิธี Quine-McCluskey ค่า don't care ใช้เพื่อช่วยรวมพจน์ให้ใหญ่ขึ้น
แต่ไม่จำเป็นต้องครอบคลุมในคำตอบสุดท้าย
หลักสำคัญมี 2 ข้อ:

1. ตอนสร้างรอบการรวม ให้รวม minterm ที่เป็น 1 และ don't care เข้าไปด้วย
2. ตอนสร้าง PI chart ให้คอลัมน์บังคับมีเฉพาะ minterm ที่เป็น 1 เท่านั้น

สาเหตุคือ don't care เป็นสถานะอินพุตที่ระบบไม่ใช้งานจริง
หรือเป็นสถานะที่ output จะเป็น 0 หรือ 1 ก็ไม่กระทบ specification
ตัวอย่างที่ชัดคือวงจรสำหรับรหัส BCD
อินพุต 4 บิตที่แทนเลข 0-9 เท่านั้น
สถานะ 1010-1111 ไม่ใช่รหัส BCD ที่ถูกต้อง
จึงมักกำหนดเป็น don't care เพื่อช่วยลดรูปวงจร

### 5.6.1 ตัวอย่าง don't care แบบย่อ

ให้

```text
F(A,B,C,D) = Σm(1,3,7,11,15) + d(0,2,5)
```

แปลว่า minterm ที่ต้องเป็น 1 คือ 1,3,7,11,15
และ don't care คือ 0,2,5

เขียนทุกตัวรวมกันเพื่อใช้ในขั้นรวม:

```text
+------+--------+--------------+
| ชนิด | เลข    | binary       |
+------+--------+--------------+
| d    | 0      | 0000         |
| 1    | 1      | 0001         |
| d    | 2      | 0010         |
| 1    | 3      | 0011         |
| d    | 5      | 0101         |
| 1    | 7      | 0111         |
| 1    | 11     | 1011         |
| 1    | 15     | 1111         |
+------+--------+--------------+
```

จัดกลุ่มตามจำนวนเลข 1:

```text
+---------+---------------------------+
| จำนวน 1 | term                      |
+---------+---------------------------+
|    0    | 0000(d0)                  |
|    1    | 0001(m1), 0010(d2)        |
|    2    | 0011(m3), 0101(d5)        |
|    3    | 0111(m7), 1011(m11)       |
|    4    | 1111(m15)                 |
+---------+---------------------------+
```

รวมรอบแรก:

```text
+-----------+-----------+----------+----------------+
| term 1    | term 2    | ผลรวม    | ครอบคลุม        |
+-----------+-----------+----------+----------------+
| 0000(d0)  | 0001(m1)  | 000-     | d0,m1          |
| 0000(d0)  | 0010(d2)  | 00-0     | d0,d2          |
| 0001(m1)  | 0011(m3)  | 00-1     | m1,m3          |
| 0001(m1)  | 0101(d5)  | 0-01     | m1,d5          |
| 0010(d2)  | 0011(m3)  | 001-     | d2,m3          |
| 0011(m3)  | 0111(m7)  | 0-11     | m3,m7          |
| 0011(m3)  | 1011(m11) | -011     | m3,m11         |
| 0101(d5)  | 0111(m7)  | 01-1     | d5,m7          |
| 0111(m7)  | 1111(m15) | -111     | m7,m15         |
| 1011(m11) | 1111(m15) | 1-11     | m11,m15        |
+-----------+-----------+----------+----------------+
```

รวมต่อบางส่วน:

```text
000- กับ 001- -> 00--    ครอบคลุม d0,m1,d2,m3
00-1 กับ 01-1 -> 0--1    ครอบคลุม m1,m3,d5,m7
0-11 กับ 1-11 -> --11    ครอบคลุม m3,m7,m11,m15
-011 กับ -111 -> --11    ครอบคลุม m3,m7,m11,m15
```

PI chart ต้องใส่เฉพาะ minterm 1,3,7,11,15:

```text
+----------+---+---+---+----+----+
| PI       | 1 | 3 | 7 | 11 | 15 |
+----------+---+---+---+----+----+
| 00--     | x | x |   |    |    |
| 0--1     | x | x | x |    |    |
| --11     |   | x | x | x  | x  |
+----------+---+---+---+----+----+
```

จาก chart:

```text
minterm 11 มีเฉพาะ --11
minterm 15 มีเฉพาะ --11
ดังนั้น --11 เป็น essential PI
```

เมื่อเลือก `--11` จะครอบคลุม 3,7,11,15
เหลือ minterm 1
เลือก `0--1` เพื่อครอบคลุม 1

คำตอบ:

```text
--11 = CD
0--1 = A'D
F = CD + A'D
```

สังเกตว่า don't care ช่วยให้พจน์ใหญ่ขึ้น
แต่ไม่ถูกบังคับให้ปรากฏใน chart
เช่น don't care 5 ถูกใช้เพื่อสร้าง `0--1`
แต่ไม่ต้องมีคอลัมน์ 5 ในตารางครอบคลุมสุดท้าย

## 5.7 ความสัมพันธ์กับ K-map และข้อจำกัดของแต่ละวิธี

Quine-McCluskey และ K-map ลดรูปด้วยหลักการเดียวกัน
คือการรวม minterm ที่อยู่ติดกันทางตรรกะเพื่อกำจัดตัวแปรที่เปลี่ยนค่า
ต่างกันที่วิธีแสดงข้อมูล

K-map แสดงความติดกันด้วยตำแหน่งบนแผนที่
ผู้เรียนมองกลุ่มด้วยสายตา
ส่วน Quine-McCluskey แสดงความติดกันด้วยจำนวนบิตที่ต่างกัน
ผู้เรียนตรวจด้วยตารางและ pattern

```text
K-map:
  เห็นกลุ่มเป็นรูปสี่เหลี่ยมบน Gray-code map

Quine-McCluskey:
  เห็นกลุ่มเป็น pattern ที่มี dash เช่น -00-, --10
```

| ประเด็น | K-map | Quine-McCluskey |
|---|---|---|
| จำนวนตัวแปรที่เหมาะ | 2-4 ตัวแปร, 5 ตัวแปรพอทำได้แต่เริ่มยาก | 4 ตัวแปรขึ้นไป และขยายเป็นหลายตัวแปรได้เป็นระบบ |
| วิธีคิดหลัก | มองช่องติดกันและจับกลุ่ม | เปรียบเทียบบิตและรวมด้วย dash |
| จุดแข็ง | เห็นภาพเร็ว เหมาะกับการสอน intuition | ตรวจย้อนกลับได้ดี เขียนเป็นอัลกอริทึมได้ |
| จุดอ่อน | หลุดกลุ่มง่ายเมื่อช่องมากหรือหลายแผ่น | ตารางยาวมากเมื่อ minterm จำนวนมาก |
| การจัดการ don't care | ใช้ X ช่วยจับกลุ่มบนแผนที่ | ใส่ d ในขั้นรวม แต่ไม่บังคับใน PI chart |
| การพิสูจน์คำตอบ | อาศัยการเห็นกลุ่มครบทุกช่อง 1 | ใช้ PI chart ตรวจว่า minterm ครบทุกตัว |
| ความเหมาะกับโปรแกรม | ไม่สะดวกเพราะขึ้นกับรูปภาพ | เหมาะกว่าเพราะเป็นขั้นตอนเชิงตาราง |

สำหรับงานออกแบบวงจรขนาดเล็ก
K-map มักเร็วกว่าและช่วยให้เข้าใจความหมายของกลุ่มได้ดี
สำหรับงานที่ต้องส่งขั้นตอนละเอียดหรือฟังก์ชันมีตัวแปรมาก
Quine-McCluskey ให้ความเป็นระบบกว่า
แต่ถ้าฟังก์ชันมีตัวแปรและ minterm จำนวนมากมาก
วิธีตารางเองก็อาจขยายตัวจนมี PI จำนวนมาก
เครื่องมือ logic synthesis จริงจึงมักใช้ heuristic algorithm เช่น Espresso แทนการหาคำตอบ exact ทุกกรณี

## 5.8 Petrick's method สำหรับกรณีเลือกชุดไม่ unique

หลังเลือก essential PI แล้ว บางโจทย์ยังเหลือ minterm ที่สามารถครอบคลุมได้หลายชุด
ถ้ามอง chart ด้วยตาอาจเลือกได้
แต่ถ้า chart ใหญ่และมีคำตอบหลายแบบ การเลือกด้วยตาอาจพลาด
**Petrick's method** เป็นวิธีแปลงปัญหาเลือก PI ให้เป็นพีชคณิตของชุดครอบคลุม

แนวคิดคือสำหรับ minterm แต่ละตัว
เขียนผลบวกของ PI ที่ครอบคลุม minterm นั้น
จากนั้นนำทุกวงเล็บมาคูณกัน
ผลคูณที่ได้คือชุด PI ที่ครอบคลุม minterm เหลือทั้งหมด

ตัวอย่างหลังเลือก EPI แล้ว เหลือ minterm 5 และ 7 จากตัวอย่างหลัก

```text
minterm 5 ครอบคลุมโดย P1 หรือ P2 -> (P1 + P2)
minterm 7 ครอบคลุมโดย P2 หรือ P3 -> (P2 + P3)
```

ดังนั้นสมการ Petrick คือ

```text
P = (P1 + P2)(P2 + P3)
```

กระจาย:

```text
P = P1P2 + P1P3 + P2P2 + P2P3
P = P1P2 + P1P3 + P2 + P2P3
```

ใช้กฎดูดกลืน (absorption):

```text
P2 + P1P2 = P2
P2 + P2P3 = P2
```

เหลือ

```text
P = P2 + P1P3
```

แปลว่าเลือก `P2` เพียงตัวเดียวก็พอ
หรือเลือก `P1` กับ `P3` สองตัวก็ได้
ชุดที่มีพจน์น้อยกว่าคือ `P2`
จึงตรงกับการเลือก `A'BD` ในตัวอย่างหลัก

ถ้ามีหลายชุดที่มีจำนวน PI เท่ากัน
ต้องเปรียบเทียบจำนวน literal รวม
เช่นชุดหนึ่งมีพจน์สองตัวแต่รวม 6 literal
อีกชุดมีพจน์สองตัวแต่รวม 5 literal
โดยทั่วไปเลือกชุดที่มี 5 literal
เว้นแต่โจทย์กำหนดเกณฑ์ฮาร์ดแวร์อื่น เช่น ใช้ NAND ล้วน หรือใช้ IC ที่มีอยู่จำกัด

## 5.9 ตัวอย่างเลือกชุดที่มีคำตอบ minimal มากกว่าหนึ่งแบบ

ให้หลังจากหา PI แล้วเหลือ chart เฉพาะ minterm ที่ยังไม่ถูกครอบคลุมดังนี้

```text
+------+----+----+----+----+
| PI   | 3  | 5  | 9  | 11 |
+------+----+----+----+----+
| P1   | x  | x  |    |    |
| P2   | x  |    | x  |    |
| P3   |    | x  |    | x  |
| P4   |    |    | x  | x  |
+------+----+----+----+----+
```

ไม่มีคอลัมน์ใดมี x เพียงตัวเดียว
ดังนั้นไม่มี EPI ในส่วนที่เหลือ

เขียน Petrick:

```text
minterm 3  -> (P1 + P2)
minterm 5  -> (P1 + P3)
minterm 9  -> (P2 + P4)
minterm 11 -> (P3 + P4)

P = (P1 + P2)(P1 + P3)(P2 + P4)(P3 + P4)
```

เลือกด้วยการมอง chart จะเห็นชุดครอบคลุมขนาด 2 ได้สองแบบ:

```text
ชุดที่ 1: P1 + P4
P1 ครอบคลุม 3,5
P4 ครอบคลุม 9,11

ชุดที่ 2: P2 + P3
P2 ครอบคลุม 3,9
P3 ครอบคลุม 5,11
```

ถ้า PI ทั้งสี่มีจำนวน literal เท่ากัน
สองชุดนี้เป็นคำตอบ minimal เท่ากัน
ในรายงานต้องเขียนให้ชัดว่า minimal SOP มีได้มากกว่าหนึ่งคำตอบ
และระบุเกณฑ์ที่ใช้เลือกคำตอบที่นำไปวาดวงจร

ตัวอย่างการตัดสินใจด้านฮาร์ดแวร์:

```text
ถ้า P1 และ P4 ใช้สัญญาณ complement ที่มีอยู่แล้วจากวงจรอื่น
แต่ P2 และ P3 ต้องเพิ่ม inverter ใหม่
อาจเลือกชุด P1 + P4 แม้จำนวน literal เท่ากัน
```

นี่คือเหตุผลที่คำว่า minimal ในคณิตศาสตร์กับ minimal ในฮาร์ดแวร์จริงอาจไม่เหมือนกันทุกครั้ง
คณิตศาสตร์นับพจน์และ literal
แต่วงจรจริงนับชนิด gate, จำนวน IC, fan-out, delay และสายที่ต้องลากบน breadboard ด้วย

## 5.10 ข้อผิดพลาดเฉพาะวิธีตารางที่ต้องระวัง

ข้อผิดพลาดที่ 1 คือเขียนเลขฐานสองไม่ครบจำนวนบิต
ถ้าโจทย์มี 4 ตัวแปร ต้องเขียน `m3` เป็น `0011`
ไม่ใช่ `11`
เพราะตำแหน่ง `A` และ `B` จะหายไป

ข้อผิดพลาดที่ 2 คือรวมพจน์ที่ต่างกันมากกว่าหนึ่งบิต
เช่น `0000` กับ `0011` รวมไม่ได้
แม้อยู่ใกล้กันในเลขฐานสิบ
เพราะต่างกันทั้ง `C` และ `D`

ข้อผิดพลาดที่ 3 คือรวมพจน์ที่มี dash คนละตำแหน่ง
เช่น `000-` กับ `00-0`
ถ้ารวมผิดจะได้ `00--`
ซึ่งครอบคลุม `0011` เพิ่มเข้ามาโดยไม่มีเหตุผล

ข้อผิดพลาดที่ 4 คือใส่ don't care เป็นคอลัมน์บังคับใน PI chart
don't care ใช้ช่วยรวมได้
แต่ไม่ต้องถูกครอบคลุมในคำตอบสุดท้าย
ถ้าเอา don't care ไปบังคับครอบคลุม อาจทำให้เลือกพจน์มากเกินจำเป็น

ข้อผิดพลาดที่ 5 คือคิดว่า essential prime implicant ทุกตัวคือคำตอบทั้งหมด
จริง ๆ แล้ว EPI เป็นเพียงพจน์ที่ต้องเลือกแน่
หลังเลือก EPI แล้วยังอาจมี minterm เหลือ
ต้องเลือก PI เพิ่มจน chart ครอบคลุมครบ

ข้อผิดพลาดที่ 6 คือไม่ลบผลรวมซ้ำ
ถ้าได้ `-00-` จากสองเส้นทาง
ให้เก็บเป็น PI เดียว
ไม่ต้องใส่สองแถวใน PI chart

ข้อผิดพลาดที่ 7 คือหยุดรวมเร็วเกินไป
ถ้าได้พจน์ที่มี dash หนึ่งตำแหน่งแล้ว
ยังต้องลองรวมต่อเพื่อให้ dash เพิ่มเป็นสองตำแหน่งหรือมากกว่า
พจน์ที่ใหญ่กว่าจะมี literal น้อยกว่าและมักทำให้วงจรง่ายกว่า

## 5.11 ลำดับการทำโจทย์ในกระดาษคำตอบ

เมื่อต้องแสดงวิธีทำในข้อสอบหรือรายงาน
ควรจัดคำตอบให้ผู้อ่านตรวจตามได้ดังนี้

1. เขียนโจทย์และลำดับตัวแปร เช่น `A,B,C,D`
2. เขียน minterm เป็นเลขฐานสองครบจำนวนบิต
3. จัดกลุ่มตามจำนวนเลข 1
4. แสดงตารางรวมรอบที่ 1 พร้อมผลรวมและ minterm ที่ครอบคลุม
5. แสดงตารางรวมรอบถัดไปจนรวมต่อไม่ได้
6. ระบุ PI ทั้งหมดพร้อมพจน์บูลีน
7. สร้าง PI chart โดยใส่เฉพาะ minterm ที่ต้องเป็น 1
8. วงหรือระบุ EPI และ minterm ที่ถูกครอบคลุม
9. เลือก PI เพิ่มด้วยเหตุผลจาก chart หรือ Petrick's method
10. เขียนคำตอบ minimal SOP และถ้าต้องการให้วาดวงจรจากสมการสุดท้าย

รูปแบบนี้ช่วยลดการเสียคะแนนจากคำตอบที่ได้ผลลัพธ์ถูกแต่ไม่มีหลักฐาน
โดยเฉพาะโจทย์ที่มีคำตอบ minimal ได้หลายแบบ
ผู้ตรวจต้องเห็นว่าผู้เรียนเลือกชุดครอบคลุมด้วยเหตุผล ไม่ใช่เดาสมการจาก K-map ในใจ

## 5.12 การตรวจสอบผลลัพธ์หลังลดรูป

หลังได้สมการ minimal SOP แล้ว ต้องตรวจอย่างน้อยสองระดับ

ระดับแรกคือการตรวจ minterm ที่ต้องเป็น 1
แทนค่าอินพุตทุก minterm ที่โจทย์กำหนด
แล้วตรวจว่าอย่างน้อยหนึ่งพจน์ในสมการให้ค่า 1

ระดับที่สองคือการตรวจ minterm ที่ไม่ควรเป็น 1
ถ้าไม่มี don't care ต้องตรวจว่าไม่มีพจน์ใดครอบคลุม minterm ที่ไม่ได้อยู่ในโจทย์
ถ้ามี don't care พจน์สามารถครอบคลุม don't care ได้
แต่ห้ามครอบคลุม minterm ที่ต้องเป็น 0

สำหรับตัวอย่างหลัก
พจน์ `B'C'` ครอบคลุมเฉพาะ `0000,0001,1000,1001`
ซึ่งตรงกับ `0,1,8,9`
พจน์ `CD'` ครอบคลุม `0010,0110,1010,1110`
ซึ่งตรงกับ `2,6,10,14`
พจน์ `A'BD` ครอบคลุม `0101,0111`
ซึ่งตรงกับ `5,7`
รวมแล้วได้ชุด minterm เดิมครบถ้วน

```text
B'C'  -> 0,1,8,9
CD'   -> 2,6,10,14
A'BD  -> 5,7
รวม   -> 0,1,2,5,6,7,8,9,10,14
```

เมื่อนำไปจำลองใน Logisim Evolution
ควรสร้าง input pin 4 ตัวชื่อ `A,B,C,D`
สร้าง NOT gate สำหรับตัวแปรที่ต้องกลับค่า
สร้าง AND gate สำหรับแต่ละพจน์
และ OR gate รวม output
จากนั้นใช้ poke tool เปลี่ยนอินพุตตาม minterm 0-15
แล้วบันทึกว่า output เป็น 1 เฉพาะ minterm ที่โจทย์กำหนด

## 5.13 สรุปสาระของบท

Quine-McCluskey เป็นวิธีลดรูปฟังก์ชันบูลีนที่เปลี่ยนการจับกลุ่มจากภาพบน K-map
ให้เป็นขั้นตอนเชิงตาราง
การจัดกลุ่มตามจำนวนเลข 1 ทำให้รู้ว่าพจน์ใดมีโอกาสรวมกัน
การแทนบิตที่ต่างด้วย dash ทำให้ตัวแปรที่เปลี่ยนค่าถูกตัดออก
การทำเครื่องหมายพจน์ที่ถูกใช้รวมช่วยแยก prime implicant
และ prime implicant chart ใช้เลือกชุดพจน์ที่ครอบคลุม minterm ของฟังก์ชันครบถ้วน

ในตัวอย่างหลัก
ฟังก์ชัน

```text
F(A,B,C,D) = Σm(0,1,2,5,6,7,8,9,10,14)
```

ลดรูปได้เป็น

```text
F = B'C' + CD' + A'BD
```

พจน์ `B'C'` และ `CD'` เป็น essential prime implicant
ส่วน `A'BD` ถูกเลือกเพิ่มเพราะครอบคลุม minterm 5 และ 7 พร้อมกัน

การจัดการ don't care ต้องจำให้แม่นว่า
don't care เข้าไปช่วยในขั้นรวมได้
แต่ไม่ใช่คอลัมน์ที่ต้องบังคับครอบคลุมใน PI chart
ถ้าชุดครอบคลุมหลังเลือก EPI ไม่ชัดเจน
Petrick's method เป็นเครื่องมือเชิงพีชคณิตสำหรับเลือกชุด PI ที่เหลืออย่างเป็นระบบ

## แบบฝึกหัดท้ายบท

1. ใช้วิธี Quine-McCluskey ลดรูป `F(A,B,C,D)=Σm(1,3,4,5,9,11,12,13)` โดยแสดงตารางจัดกลุ่ม รอบรวม PI chart และคำตอบ minimal SOP
2. ใช้วิธี Quine-McCluskey ลดรูป `F(A,B,C,D)=Σm(0,2,3,6,7,8,10,11,14,15)` แล้วระบุว่า PI ใดเป็น essential prime implicant
3. ลดรูป `F(A,B,C,D)=Σm(2,3,7,10,11,15)+d(0,1,5)` โดยอธิบายว่า don't care ตัวใดถูกใช้ช่วยรวม และตัวใดไม่จำเป็นต้องใช้
4. สำหรับฟังก์ชัน `F(A,B,C,D,E)=Σm(0,1,3,7,8,9,11,15,16,17,19,23)` ให้เขียนเฉพาะตารางจัดกลุ่มตามจำนวนเลข 1 และรอบรวมที่ 1
5. จาก PI chart ต่อไปนี้ จงหา EPI และเลือกชุดครอบคลุมที่เหลือให้น้อยที่สุด

```text
+------+---+---+---+---+---+
| P1   | x | x |   |   |   |

| P3   |   |   | x |   | x |
| P4   | x |   |   | x |   |
| P5   |   |   |   | x | x |
+------+---+---+---+---+---+
```

7. อธิบายความแตกต่างระหว่าง pattern `-0-0` กับ `--10` ว่าตัดตัวแปรใดออก และครอบคลุม minterm ใดบ้างในระบบตัวแปร `A,B,C,D`
8. เปรียบเทียบการแก้โจทย์ 5 ตัวแปรด้วย K-map และ Quine-McCluskey โดยยกตัวอย่างข้อผิดพลาดที่มีโอกาสเกิดใน K-map อย่างน้อย 2 ข้อ

10. ออกแบบวงจร AND-OR จากคำตอบในข้อ 1 โดยนับจำนวน NOT, AND, OR gate ที่ต้องใช้ หากใช้เฉพาะเกต 2-input
