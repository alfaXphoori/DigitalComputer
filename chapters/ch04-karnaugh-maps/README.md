# Chapter 4: แผนผังคาร์โนห์

## Karnaugh Maps (K-Maps)

---

## 4.1 บทนำ

**Karnaugh Map (K-Map)** คือเครื่องมือกราฟิกสำหรับลดรูปสมการบูลีนอย่างเป็นระบบ ให้ได้สมการที่สั้นที่สุด → สร้างวงจรด้วยเกตน้อยที่สุด

- ใช้ได้กับ **2–4 ตัวแปร** (สูงสุด 5–6 แต่ไม่สะดวก)
- ลำดับตัวแปรใช้ **Gray Code** (เปลี่ยนทีละ 1 บิต เท่านั้น)
- เซลล์ที่อยู่ติดกัน (รวมถึงขอบ) แตกต่างกัน **เพียง 1 บิต**

> 💡 ข้อดีเหนือ Boolean Algebra: ผลลัพธ์ที่ได้ **ย่อสุดเสมอ** สำหรับ 2-level circuit

### โครงสร้าง K-Map ตามจำนวนตัวแปร

| จำนวนตัวแปร | จำนวนเซลล์ | ขนาด K-Map |
|:---:|:---:|:---:|
| 2 | 4 | 2×2 |
| 3 | 8 | 2×4 |
| 4 | 16 | 4×4 |
| 5 | 32 | 4×8 |

## 4.2 K-Map 2 ตัวแปร (4 เซลล์)

$$F(A,B) = \sum m(1, 2, 3)$$

**ขั้นที่ 1: สร้าง K-Map และใส่ค่า**

```
  ┌──────┬───────┬───────┐
  │ A\B  │  B=0  │  B=1  │
  ├──────┼───────┼───────┤
  │  0   │   0   │   1   │  ← m0=0, m1=1
  ├──────┼───────┼───────┤
  │  1   │   1   │   1   │  ← m2=1, m3=1
  └──────┴───────┴───────┘
```

**ขั้นที่ 2: จัดกลุ่ม**

- **กลุ่ม 1** (ขนาด 2): m1, m3 — คอลัมน์ B=1 ทั้งหมด
- **กลุ่ม 2** (ขนาด 2): m2, m3 — แถว A=1 ทั้งหมด

**ขั้นที่ 3: อ่านตัวแปรจากแต่ละกลุ่ม**

- กลุ่ม 1 (m1, m3): A เปลี่ยน (0→1) ตัดออก; B=1 คงที่ → **$B$**
- กลุ่ม 2 (m2, m3): B เปลี่ยน (0→1) ตัดออก; A=1 คงที่ → **$A$**

$$\boxed{F = A + B}$$

## 4.3 K-Map 3 ตัวแปร (8 เซลล์)

### ตำแหน่ง Minterm

```
  ┌───────┬──────┬──────┬──────┬──────┐
  │ A \ BC│  00  │  01  │  11  │  10  │
  ├───────┼──────┼──────┼──────┼──────┤
  │   0   │  m0  │  m1  │  m3  │  m2  │
  ├───────┼──────┼──────┼──────┼──────┤
  │   1   │  m4  │  m5  │  m7  │  m6  │
  └───────┴──────┴──────┴──────┴──────┘
```

> ⚠️ **ลำดับคอลัมน์เป็น Gray Code:** 00, 01, **11**, 10 — ไม่ใช่ 00, 01, 10, 11!

### ตัวอย่างที่ 1: $F(A,B,C) = \sum m(0, 2, 5, 7)$

**ขั้นที่ 1: ใส่ค่าลงตาราง**

```
  ┌───────┬──────┬──────┬──────┬──────┐
  │ A \ BC│  00  │  01  │  11  │  10  │
  ├───────┼──────┼──────┼──────┼──────┤
  │   0   │  1   │  0   │  0   │  1   │
  ├───────┼──────┼──────┼──────┼──────┤
  │   1   │  0   │  1   │  1   │  0   │
  └───────┴──────┴──────┴──────┴──────┘
```

**ขั้นที่ 2: จัดกลุ่ม**

- **กลุ่ม 1** (ขนาด 2): m0, m2 — คอลัมน์ BC=00 และ BC=10 (แถว A=0)
- **กลุ่ม 2** (ขนาด 2): m5, m7 — คอลัมน์ BC=01 และ BC=11 (แถว A=1)

**ขั้นที่ 3: อ่านตัวแปร**

| กลุ่ม | เซลล์ | A | B | C | ตัวแปรที่คงที่ |
|:---:|:---:|:---:|:---:|:---:|:---:|
| กลุ่ม 1 | m0, m2 | 0 คงที่ | เปลี่ยน | 0 คงที่ | $\overline{A}\,\overline{C}$ |
| กลุ่ม 2 | m5, m7 | 1 คงที่ | เปลี่ยน | 1 คงที่ | $AC$ |

$$\boxed{F = \overline{A}\,\overline{C} + AC}$$

### ตัวอย่างที่ 2: $F(A,B,C) = \sum m(0, 1, 2, 3, 7)$

**K-Map:**

```
  ┌───────┬──────┬──────┬──────┬──────┐
  │ A \ BC│  00  │  01  │  11  │  10  │
  ├───────┼──────┼──────┼──────┼──────┤
  │   0   │  1   │  1   │  1   │  1   │
  ├───────┼──────┼──────┼──────┼──────┤
  │   1   │  0   │  0   │  1   │  0   │
  └───────┴──────┴──────┴──────┴──────┘
```

**จัดกลุ่ม:**

- **กลุ่ม 1** (ขนาด 4): m0, m1, m2, m3 — แถว A=0 ทั้งหมด → A=0 คงที่ → **$\overline{A}$**
- **กลุ่ม 2** (ขนาด 2): m3, m7 — คอลัมน์ BC=11 → B=1, C=1 คงที่ → **$BC$**

$$\boxed{F = \overline{A} + BC}$$

## 4.4 K-Map 4 ตัวแปร (16 เซลล์) ⭐

### ตำแหน่ง Minterm

```
  ┌─────────┬──────┬──────┬──────┬──────┐
  │ AB \ CD │  00  │  01  │  11  │  10  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   00    │  m0  │  m1  │  m3  │  m2  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   01    │  m4  │  m5  │  m7  │  m6  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   11    │  m12 │  m13 │  m15 │  m14 │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   10    │  m8  │  m9  │  m11 │  m10 │
  └─────────┴──────┴──────┴──────┴──────┘
```

### ตัวอย่างที่ 1: $F(A,B,C,D) = \sum m(0, 1, 2, 5, 8, 9, 10)$

**ขั้นที่ 1: ใส่ค่า**

```
  ┌─────────┬──────┬──────┬──────┬──────┐
  │ AB \ CD │  00  │  01  │  11  │  10  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   00    │  1   │  1   │  0   │  1   │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   01    │  0   │  1   │  0   │  0   │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   11    │  0   │  0   │  0   │  0   │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   10    │  1   │  1   │  0   │  1   │
  └─────────┴──────┴──────┴──────┴──────┘
```

**ขั้นที่ 2: จัดกลุ่ม**

- **กลุ่ม 1** (ขนาด 4): m0, m2, m8, m10 — 4 มุมซ้าย/ขวา บน/ล่าง
  - A: 0,0,1,1 เปลี่ยน; B: 0,0,0,0 คงที่=0; C: 0,1,0,1 เปลี่ยน; D: 0,0,0,0 คงที่=0
  - → **$\overline{B}\,\overline{D}$**

- **กลุ่ม 2** (ขนาด 4): m0, m1, m8, m9 — คอลัมน์ CD=00,01 แถว AB=00,10
  - A: 0,0,1,1 เปลี่ยน; B: 0,0,0,0 คงที่=0; C: 0,0,0,0 คงที่=0; D: 0,1,0,1 เปลี่ยน
  - → **$\overline{B}\,\overline{C}$**

- **กลุ่ม 3** (ขนาด 2): m1, m5 — คอลัมน์ CD=01 แถว AB=00,01
  - A: 0,0 คงที่=0; B: 0,1 เปลี่ยน; C: 0,0 คงที่=0; D: 1,1 คงที่=1
  - → **$\overline{A}\,\overline{C}\,D$**

> 💡 m1 ถูกนับซ้ำในกลุ่ม 1, 2 และ 3 ได้ — ช่วยขยายกลุ่มให้ใหญ่ขึ้น

$$\boxed{F = \overline{B}\,\overline{D} + \overline{B}\,\overline{C} + \overline{A}\,\overline{C}\,D}$$

### ตัวอย่างที่ 2: $F(A,B,C,D) = \sum m(0, 1, 3, 5, 7, 8, 9, 11, 15)$

**K-Map:**

```
  ┌─────────┬──────┬──────┬──────┬──────┐
  │ AB \ CD │  00  │  01  │  11  │  10  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   00    │  1   │  1   │  1   │  0   │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   01    │  0   │  1   │  1   │  0   │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   11    │  0   │  0   │  1   │  0   │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   10    │  1   │  1   │  1   │  0   │
  └─────────┴──────┴──────┴──────┴──────┘
```

**จัดกลุ่ม:**

- **กลุ่ม 1** (ขนาด 4): m1, m3, m9, m11 — คอลัมน์ CD=01,11 แถว AB=00,10
  - A เปลี่ยน, B=0, C เปลี่ยน, D=1 คงที่ → **$\overline{B}\,D$**
- **กลุ่ม 2** (ขนาด 4): m3, m7, m11, m15 — คอลัมน์ CD=11 ทั้ง 4 แถว
  - C=1, D=1 คงที่ → **$CD$**
- **กลุ่ม 3** (ขนาด 2): m0, m8 — คอลัมน์ CD=00 แถว AB=00,10
  - A เปลี่ยน, B=0, C=0, D=0 คงที่ → **$\overline{B}\,\overline{C}\,\overline{D}$**

$$\boxed{F = \overline{B}\,D + CD + \overline{B}\,\overline{C}\,\overline{D}}$$

## 4.5 วิธีอ่านตัวแปรจากกลุ่ม ⭐

นี่คือทักษะสำคัญที่สุดในการใช้ K-Map

### หลักการ:

> **ตัวแปรที่ "เปลี่ยนค่า" ภายในกลุ่ม → ตัดออก**
> **ตัวแปรที่ "คงที่" ภายในกลุ่ม → เก็บไว้** (ถ้า =1 เขียนปกติ, ถ้า =0 เขียน complement)

### ขั้นตอนอ่านตัวแปร (K-Map 4 ตัวแปร):

```
1. ดูว่ากลุ่มครอบคลุมแถว AB ไหนบ้าง
   - A เปลี่ยน (มีทั้ง 0 และ 1) → ตัด A ออก
   - A คงที่=1 → เก็บ A        A คงที่=0 → เก็บ Ā
   - B เปลี่ยน → ตัด B ออก    B=1 → เก็บ B  B=0 → เก็บ B̄

2. ดูว่ากลุ่มครอบคลุมคอลัมน์ CD ไหนบ้าง
   - C เปลี่ยน → ตัด C ออก    C=1 → เก็บ C  C=0 → เก็บ C̄
   - D เปลี่ยน → ตัด D ออก    D=1 → เก็บ D  D=0 → เก็บ D̄

3. ตัวแปรที่เหลือ → นำมา AND กัน
```

### ตัวอย่างอ่านตัวแปร: กลุ่ม m0, m1, m8, m9 (ขนาด 4)

```
  ┌─────────┬──────┬──────┬──────┬──────┐
  │ AB \ CD │  00  │  01  │  11  │  10  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   00    │ [1]  │ [1]  │      │      │   ← A=0, B=0
  ├─────────┼──────┼──────┼──────┼──────┤
  │   01    │      │      │      │      │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   11    │      │      │      │      │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   10    │ [1]  │ [1]  │      │      │   ← A=1, B=0
  └─────────┴──────┴──────┴──────┴──────┘
```

| ตัวแปร | ค่าในกลุ่ม | สรุป |
|:---:|:---:|:---:|
| A | 0, 0, 1, 1 | เปลี่ยน → **ตัดออก** |
| B | 0, 0, 0, 0 | คงที่=0 → **$\overline{B}$** |
| C | 0, 0, 0, 0 | คงที่=0 → **$\overline{C}$** |
| D | 0, 1, 0, 1 | เปลี่ยน → **ตัดออก** |

→ **$\overline{B}\,\overline{C}$**

## 4.6 กฎการจัดกลุ่ม (Grouping Rules) ⭐

| ข้อ | กฎ |
|:---:|:---|
| 1 | ขนาดกลุ่มต้องเป็น **$2^n$** เท่านั้น: 1, 2, 4, 8, 16 |
| 2 | รูปร่างต้องเป็น **สี่เหลี่ยมผืนผ้า** (รวมถึง wrap-around) |
| 3 | กลุ่ม **ยิ่งใหญ่ยิ่งดี** (ลดตัวแปรได้มาก) |
| 4 | ทุกช่อง `1` ต้องอยู่ใน **อย่างน้อย 1 กลุ่ม** |
| 5 | ช่อง `1` สามารถ **ซ้อนทับกลุ่ม** ได้ |
| 6 | จำนวนกลุ่มรวมให้ **น้อยที่สุด** |

### กลุ่มขนาด n → ลดได้กี่ตัวแปร?

| ขนาดกลุ่ม | ตัวแปรที่ตัดออก | ตัวแปรที่เหลือ (4-var) |
|:---:|:---:|:---:|
| 1 ช่อง | 0 | 4 ตัวแปร |
| 2 ช่อง | 1 | 3 ตัวแปร |
| 4 ช่อง | 2 | 2 ตัวแปร |
| 8 ช่อง | 3 | 1 ตัวแปร |
| 16 ช่อง | 4 | 0 ตัวแปร (= 1) |

### กลุ่มที่ถูกต้อง vs ผิด

```
ถูกต้อง ✓:                    ผิด ✗:

┌──┬──┬──┬──┐                ┌──┬──┬──┬──┐
│1 │1 │  │  │  ← ขนาด 2      │1 │1 │1 │  │  ← ขนาด 3 (ไม่ใช่ 2ⁿ)
├──┼──┼──┼──┤                └──┴──┴──┴──┘
│  │  │  │  │
└──┴──┴──┴──┘

┌──┬──┬──┬──┐                ┌──┬──┬──┬──┐
│1 │  │  │1 │  ← wrap ขนาด2  │1 │  │1 │  │  ← ไม่เป็นสี่เหลี่ยม
├──┼──┼──┼──┤                └──┴──┴──┴──┘
│  │  │  │  │
├──┼──┼──┼──┤
│  │  │  │  │
├──┼──┼──┼──┤
│1 │  │  │1 │  ← 4 มุม ขนาด 4 ✓
└──┴──┴──┴──┘
```

## 4.7 Wrap-Around (ขอบเชื่อมกัน)

ใน K-Map — **ขอบซ้าย-ขวา** และ **บน-ล่าง** เชื่อมถึงกันเหมือนผิวทรงกลม

```
  ┌─────────┬──────┬──────┬──────┬──────┐
  │ AB \ CD │  00  │  01  │  11  │  10  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   00    │  m0  │  m1  │  m3  │  m2  │  ← เชื่อมกับแถว AB=10
  ├─────────┼──────┼──────┼──────┼──────┤
  │   01    │  m4  │  m5  │  m7  │  m6  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   11    │  m12 │  m13 │  m15 │  m14 │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   10    │  m8  │  m9  │  m11 │  m10 │  ← เชื่อมกับแถว AB=00
  └─────────┴──────┴──────┴──────┴──────┘
               ↕                      ↕
          CD=00 เชื่อมกับ CD=10 (ซ้าย-ขวา)
```

### ตัวอย่าง Wrap-Around ที่พบบ่อย:

**1. สี่มุม (4-corner group): m0, m2, m8, m10**

```
  ┌─────────┬──────┬──────┬──────┬──────┐
  │ AB \ CD │  00  │  01  │  11  │  10  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   00    │ [1]  │      │      │ [1]  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   01    │      │      │      │      │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   11    │      │      │      │      │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   10    │ [1]  │      │      │ [1]  │
  └─────────┴──────┴──────┴──────┴──────┘
```

B=0, D=0 คงที่ → **$\overline{B}\,\overline{D}$**

**2. แถวบน-ล่าง (top-bottom wrap): m0, m1, m8, m9**

```
  ┌─────────┬──────┬──────┬──────┬──────┐
  │ AB \ CD │  00  │  01  │  11  │  10  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   00    │ [1]  │ [1]  │      │      │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   01    │      │      │      │      │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   11    │      │      │      │      │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   10    │ [1]  │ [1]  │      │      │
  └─────────┴──────┴──────┴──────┴──────┘
```

B=0, C=0 คงที่ → **$\overline{B}\,\overline{C}$**

**3. ขอบซ้าย-ขวา (left-right wrap): m0, m2, m4, m6**

```
  ┌─────────┬──────┬──────┬──────┬──────┐
  │ AB \ CD │  00  │  01  │  11  │  10  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   00    │ [1]  │      │      │ [1]  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   01    │ [1]  │      │      │ [1]  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   11    │      │      │      │      │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   10    │      │      │      │      │
  └─────────┴──────┴──────┴──────┴──────┘
```

A=0, D=0 คงที่ → **$\overline{A}\,\overline{D}$**

## 4.8 Don't Care Conditions (สภาวะไม่แคร์)

เมื่อบาง input combination **ไม่มีโอกาสเกิดขึ้นจริง** หรือ **ไม่สนใจเอาต์พุต** กำหนดเป็น **`X`**

$$F(A,B,C,D) = \sum m(1, 3, 7, 11, 15) + \sum d(0, 2, 5)$$

### กฎการใช้ Don't Care:

1. ช่อง `X` เลือกเป็น `0` หรือ `1` ได้ตามสะดวก
2. เลือกเป็น `1` เมื่อทำให้กลุ่มใหญ่ขึ้น → ลดรูปได้มากขึ้น
3. เลือกเป็น `0` เมื่อไม่ช่วยอะไร
4. **ห้าม** สร้างกลุ่มที่มีแต่ `X` อย่างเดียว

### ตัวอย่าง Don't Care จริง: BCD Decoder (เลขคู่)

เลขฐาน 10 มีค่า 0–9 เท่านั้น → ค่า 10–15 เป็น don't care ทั้งหมด

$$F(A,B,C,D) = \sum m(0, 2, 4, 6, 8) + \sum d(10, 11, 12, 13, 14, 15)$$

**K-Map พร้อม Don't Care:**

```
  ┌─────────┬──────┬──────┬──────┬──────┐
  │ AB \ CD │  00  │  01  │  11  │  10  │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   00    │  1   │  0   │  0   │  1   │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   01    │  1   │  0   │  0   │  1   │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   11    │  X   │  X   │  X   │  X   │
  ├─────────┼──────┼──────┼──────┼──────┤
  │   10    │  1   │  X   │  X   │  0   │
  └─────────┴──────┴──────┴──────┴──────┘
```

**จัดกลุ่ม (ใช้ X เป็น 1 เพื่อขยายกลุ่ม):**

- ทุกเลขคู่มี D=0 เสมอ ดังนั้นสังเกต column CD=00 และ CD=10 ล้วนเป็น 1 (หรือ X)
- กลุ่มขนาด 8: คอลัมน์ CD=00, CD=10 ทั้ง 4 แถว (ใช้ X ใน AB=11 ขยาย)
  - C=0, D=0 คงที่ → **$\overline{D}$** (เพราะ D=0 ในทุกช่อง)

**ผลลัพธ์:**

$$\boxed{F = \overline{D}}$$

(เลขคู่ทุกตัวใน BCD มี D=0 เสมอ)

## 4.9 POS จาก K-Map

### ขั้นตอน:

1. จัดกลุ่มช่อง **`0`** แทน (ไม่ใช่ `1`)
2. เขียน **complement** ของตัวแปรในกลุ่ม 0 → ได้ maxterm
3. นำ maxterm มาคูณกัน → ได้ POS

> ⚠️ กฎ: ตัวแปรคงที่=0 ใน K-Map (กลุ่ม 0) → เขียน $X$ ปกติ (ไม่ complement), คงที่=1 → เขียน $\overline{X}$

### ตัวอย่าง: $F(A,B,C) = \sum m(1, 2, 3, 5) = \prod M(0, 4, 6, 7)$

**K-Map (มองช่อง 0):**

```
  ┌───────┬──────┬──────┬──────┬──────┐
  │ A \ BC│  00  │  01  │  11  │  10  │
  ├───────┼──────┼──────┼──────┼──────┤
  │   0   │ [0]  │  1   │  1   │  1   │
  ├───────┼──────┼──────┼──────┼──────┤
  │   1   │ [0]  │  1   │ [0]  │ [0]  │
  └───────┴──────┴──────┴──────┴──────┘
```

**จัดกลุ่มช่อง 0:**

- **กลุ่ม 1**: m0, m4 — คอลัมน์ BC=00 ทั้ง 2 แถว; B=0, C=0 คงที่ → maxterm = $(B + C)$
- **กลุ่ม 2**: m6, m7 — A=1, B=1 คงที่; C เปลี่ยน → maxterm = $(\overline{A} + \overline{B})$

$$\boxed{F = (B + C)(\overline{A} + \overline{B})}$$

## 4.10 ขั้นตอนสรุปการใช้ K-Map

```
1. สร้างตาราง K-Map ตามจำนวนตัวแปร (ลำดับ Gray Code)
          ↓
2. ใส่ค่า 1, 0, X ลงในเซลล์ตามฟังก์ชัน
          ↓
3. จัดกลุ่ม 1 (SOP) หรือ 0 (POS)
   ┌─ กลุ่มใหญ่สุด (เริ่มจากกลุ่มใหญ่ก่อน)
   ├─ จำนวนกลุ่มน้อยสุด
   ├─ รูปสี่เหลี่ยม, ขนาด 2ⁿ
   ├─ ขอบเชื่อมกัน (wrap-around)
   ├─ ซ้อนทับได้ แต่ทุกช่อง 1 ต้องอยู่ในกลุ่ม
   └─ Don't care ช่วยขยายกลุ่ม
          ↓
4. อ่านตัวแปรจากแต่ละกลุ่ม
   ├─ ตัวแปรที่เปลี่ยนค่า → ตัดออก
   └─ ตัวแปรที่คงที่ → เก็บไว้ (=1 ปกติ, =0 complement)
          ↓
5. เขียนสมการ SOP (OR ทุกกลุ่ม) หรือ POS (AND ทุกกลุ่ม)
```

## 4.11 เปรียบเทียบวิธีลดรูป

| เกณฑ์ | Boolean Algebra | K-Map |
|:---|:---:|:---:|
| จำนวนตัวแปร | ไม่จำกัด | 2–6 |
| ความเป็นระบบ | ต้องใช้ประสบการณ์ | เป็นขั้นตอนชัดเจน |
| ผลลัพธ์ดีที่สุด? | ไม่แน่ | ใช่ (สำหรับ 2-level) |
| Don't care | ใช้ยาก | ใช้ง่าย |
| เหมาะสำหรับ | ลดรูปทั่วไป | ลดรูป SOP/POS |

## 4.12 K-Map แบบ 5 ตัวแปร

เมื่อมี 5 ตัวแปร (A, B, C, D, E) จะมีเซลล์ทั้งหมด 32 เซลล์
- วิธีสร้างคือการใช้ K-Map 4 ตัวแปร **2 ตาราง** วางซ้อนกัน (ตารางหนึ่งสำหรับ A=0 อีกตารางสำหรับ A=1)
- การจัดกลุ่มสามารถทำได้ในตารางเดียวกันตามปกติ และสามารถ **จับคู่ทะลุทะลวง (3D)** ระหว่างเซลล์ตำแหน่งเดียวกันในทั้งสองตารางได้

## 4.13 คำศัพท์สำคัญ (Implicants)

ในการลดรูปด้วย K-Map และวิธี Quine-McCluskey (Tabular Method):
- **Implicant:** กลุ่มใดๆ ก็ตามที่มีแต่เลข 1 (หรือ X)
- **Prime Implicant (PI):** กลุ่มที่ **ใหญ่ที่สุด** เท่าที่จะเป็นไปได้ ไม่สามารถขยายรวมกับช่องอื่นได้อีก
- **Essential Prime Implicant (EPI):** Prime Implicant ที่ครอบคลุมช่อง 1 **อย่างน้อย 1 ช่องที่ไม่ถูกครอบคลุมโดยกลุ่มอื่นเลย** (จำเป็นต้องมีกลุ่มนี้ในสมการคำตอบเสมอ)

## 4.14 การกำจัด Static Hazard ด้วย K-Map

ในวงจรจริง การกระโดดข้ามจาก Prime Implicant กลุ่มหนึ่งไปยังอีกกลุ่มหนึ่งอาจเกิด Static Hazard ได้
**วิธีแก้:** ให้เพิ่มการจับกลุ่มคร่อมระหว่าง Prime Implicant ทั้งสอง (เพิ่ม Redundant Group) เพื่อให้ครอบคลุมการเปลี่ยนสถานะ

## แบบฝึกหัดท้ายบท

1. ลดรูป $F(A,B,C) = \sum m(0, 1, 2, 3, 7)$ โดยใช้ K-Map (SOP)
2. ลดรูป $F(A,B,C,D) = \sum m(0, 1, 3, 5, 7, 8, 9, 11, 15)$ (SOP)
3. ลดรูป $F(A,B,C,D) = \sum m(1, 5, 6, 7, 11, 12, 13, 15) + \sum d(3, 9, 10)$
4. หา POS ของ $F(A,B,C) = \sum m(1, 3, 5, 7)$
5. ออกแบบวงจรจากผลลัพธ์ K-Map ข้อ 2 โดยใช้เกต NAND เท่านั้น
6. ลดรูป $F(A,B,C,D) = \sum m(0, 2, 4, 6, 8, 10, 12, 14)$ และระบุว่าสมการที่ได้คืออะไร
7. กำหนด $F(A,B,C,D) = \sum m(0, 4, 8, 12) + \sum d(1, 3, 5, 7, 9, 11, 13, 15)$ จงลดรูป
8. หา SOP และ POS จาก K-Map ของ $F(A,B,C) = \sum m(0, 3, 5, 6)$ แล้วเปรียบเทียบจำนวนเกต
