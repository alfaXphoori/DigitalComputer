## ทฤษฎีย่อเฉพาะวงจรลงรหัส

Encoder เป็นวงจรเชิงผสม (combinational circuit) ที่รับสัญญาณ input หลายเส้น แล้วสร้างรหัส binary ที่บอกว่าเส้นใด active อยู่ ตัวอย่างในใบงานนี้คือ encoder 8-to-3 ซึ่งมี input `I0` ถึง `I7` และ output 3 บิตคือ `B2 B1 B0`

แนวคิดสำคัญของ encoder ธรรมดาคือ input ต้องเป็นแบบ one-hot กล่าวคือมีเพียงหนึ่งเส้นเท่านั้นที่เป็น `1` ในแต่ละเวลา ถ้า `I5 = 1` และเส้นอื่นเป็น `0` output ต้องเป็น `101` เพราะเลข 5 ในฐานสองคือ `101`

| Input ที่ active | เลขฐานสิบ | รหัสฐานสอง `B2 B1 B0` |
|---:|---:|:---:|
| `I0` | 0 | `000` |
| `I1` | 1 | `001` |
| `I2` | 2 | `010` |
| `I3` | 3 | `011` |
| `I4` | 4 | `100` |
| `I5` | 5 | `101` |
| `I6` | 6 | `110` |
| `I7` | 7 | `111` |

สำหรับ encoder 8-to-3 แบบ active-high สมการจากตาราง one-hot คือ

```text
B2 = I4 + I5 + I6 + I7
B1 = I2 + I3 + I6 + I7
B0 = I1 + I3 + I5 + I7
V  = I0 + I1 + I2 + I3 + I4 + I5 + I6 + I7
```

เครื่องหมาย `+` ในสมการนี้หมายถึง OR ไม่ใช่การบวกเลขฐานสิบ ส่วน `V` คือ valid bit ซึ่งบอกว่ามี input อย่างน้อยหนึ่งเส้น active ถ้าไม่ใช้ `V` จะเกิดความกำกวมทันทีระหว่าง "ไม่มี input active" กับ "`I0` active" เพราะทั้งสองกรณีให้ output binary เป็น `000`

```text
I0 ───────────────────────────────────────┐
I1 ───────────────┐                       │
I2 ───────┐       │                       │
I3 ───────┼───────┼──────┐                │
I4 ──┐    │       │      │                │
I5 ──┼────┼───────┘      │                │
I6 ──┼────┘              │                │
I7 ──┘                   │                │
     │                   │                │
     ▼                   ▼                ▼
  OR for B2           OR for B1        OR for B0
  I4,I5,I6,I7         I2,I3,I6,I7      I1,I3,I5,I7
```

## ปัญหาของ Encoder ธรรมดาเมื่อ Input ซ้อนกัน

วงจร OR ด้านบนไม่ได้เลือก "เส้นที่ชนะ" แต่เพียงรวมบิตตามสมการ ถ้ามี input active มากกว่าหนึ่งเส้น output ที่ได้อาจกลายเป็นรหัสของเส้นที่ไม่ได้ active เลย

ตัวอย่างที่ 1: กด `I1` และ `I2` พร้อมกัน

```text
I2 I1 active

B2 = I4+I5+I6+I7 = 0
B1 = I2+I3+I6+I7 = 1
B0 = I1+I3+I5+I7 = 1

Output = 011
```

ผลลัพธ์ `011` หมายถึง `I3` ทั้งที่ `I3` ไม่ได้ active ปัญหานี้ไม่ได้เกิดจากต่อวงจรผิด แต่เกิดจากเงื่อนไขของ encoder ธรรมดาว่าต้องป้อน input แบบ one-hot เท่านั้น

ตัวอย่างที่ 2: กด `I3` และ `I4` พร้อมกัน

```text
B2 = I4+I5+I6+I7 = 1
B1 = I2+I3+I6+I7 = 1
B0 = I1+I3+I5+I7 = 1

Output = 111
```

ผลลัพธ์ `111` หมายถึง `I7` ทั้งที่ active จริงคือ `I3` และ `I4` เท่านั้น วงจรจึงไม่เหมาะกับปุ่มกดหลายปุ่มหรือคำขอ interrupt หลายแหล่ง ถ้ายังไม่ได้เพิ่มวงจรจัดลำดับความสำคัญ

Priority encoder แก้ปัญหานี้โดยกำหนดว่าเมื่อมี input หลายเส้น active พร้อมกัน ให้เส้นที่มี priority สูงสุดเป็นผู้ชนะ ในใบงานนี้ใช้ลำดับ `D3` สูงกว่า `D2`, `D2` สูงกว่า `D1`, และ `D1` สูงกว่า `D0`

```text
Priority order: D3 > D2 > D1 > D0

D3 D2 D1 D0 = 1 1 0 1  → output = 11, valid = 1  เพราะ D3 ชนะ
D3 D2 D1 D0 = 0 1 1 1  → output = 10, valid = 1  เพราะ D2 ชนะ
D3 D2 D1 D0 = 0 0 0 0  → output = 00, valid = 0  เพราะไม่มี input active
```

สมการของ priority encoder 4-to-2 แบบ active-high ที่ใช้ในใบงานนี้คือ

```text
P1 = D3 + D2
P0 = D3 + (not D2 · D1)
V  = D3 + D2 + D1 + D0
```

จุดที่ต้องสังเกตคือ `D0` ไม่ปรากฏใน `P1` และ `P0` เพราะเมื่อ `D0` active เพียงเส้นเดียว output เป็น `00` อยู่แล้ว แต่ `D0` ต้องปรากฏใน `V` เพื่อแยกกรณี `D0 = 1` ออกจากกรณีไม่มี input active

## ตัวอย่างออกแบบก่อนลงมือปฏิบัติ

### ตัวอย่างที่ 1: เขียนสมการ encoder 8-to-3 จากตาราง one-hot

พิจารณา output `B2` ของรหัส binary 3 บิต

| เส้น active | รหัส `B2 B1 B0` | `B2` |
|---:|:---:|:---:|
| `I0` | `000` | 0 |
| `I1` | `001` | 0 |
| `I2` | `010` | 0 |
| `I3` | `011` | 0 |
| `I4` | `100` | 1 |
| `I5` | `101` | 1 |
| `I6` | `110` | 1 |
| `I7` | `111` | 1 |

ดังนั้น `B2` ต้องเป็น 1 เมื่อ `I4`, `I5`, `I6`, หรือ `I7` active

```text
B2 = I4 + I5 + I6 + I7
```

ทำเช่นเดียวกันกับ `B1` พบว่า `B1 = 1` เมื่อเลขที่ถูก encode มีบิตกลางเป็น 1 คือ 2, 3, 6, 7

```text
B1 = I2 + I3 + I6 + I7
```

และ `B0 = 1` เมื่อเลขที่ถูก encode เป็นเลขคี่ 1, 3, 5, 7

```text
B0 = I1 + I3 + I5 + I7
```

### ตัวอย่างที่ 2: วิเคราะห์ output ผิดจาก input หลายเส้น

ให้ encoder 8-to-3 ธรรมดามี `I2 = 1`, `I5 = 1`, และเส้นอื่นเป็น 0

```text
B2 = I4+I5+I6+I7 = 1
B1 = I2+I3+I6+I7 = 1
B0 = I1+I3+I5+I7 = 1
Output = 111
```

ถ้าอ่าน output อย่างเดียวจะเข้าใจว่า `I7` active ทั้งที่ไม่มี `I7` ใน input เลย ข้อสรุปสำหรับการทดลองคือ output ของ encoder ธรรมดาใช้ได้เฉพาะเมื่อยืนยันได้ว่า input เป็น one-hot เท่านั้น

### ตัวอย่างที่ 3: พิสูจน์สมการ priority encoder 4-to-2

ใช้ลำดับความสำคัญ `D3 > D2 > D1 > D0`

| เงื่อนไข input | เส้นที่ชนะ | `P1 P0` | `V` |
|---|---:|:---:|:---:|
| `0000` | ไม่มี | `00` | 0 |
| `0001` | `D0` | `00` | 1 |
| `001x` | `D1` | `01` | 1 |
| `01xx` | `D2` | `10` | 1 |
| `1xxx` | `D3` | `11` | 1 |

สำหรับ `P1` เอาต์พุตต้องเป็น 1 เมื่อผู้ชนะคือ `D2` หรือ `D3`

```text
P1 = D3 + D2
```

สำหรับ `P0` เอาต์พุตต้องเป็น 1 เมื่อผู้ชนะคือ `D1` หรือ `D3` แต่ `D1` จะชนะได้เฉพาะเมื่อ `D2 = 0` และ `D3 = 0` ในสมการที่ย่อแล้วสามารถเขียนเป็น

```text
P0 = D3 + (not D2 · D1)
```

ถ้า `D3 = 1` สมการให้ `P0 = 1` ทันที ถ้า `D2 = 1` และ `D3 = 0` สมการให้ `P0 = 0` ซึ่งตรงกับรหัส `10` ของ `D2` ถ้า `D1 = 1` โดยไม่มี `D2` และ `D3` สมการให้ `P0 = 1` ซึ่งตรงกับรหัส `01`

## ส่วนที่ 2: สร้าง Encoder 8-to-3 จาก OR Gate (35 นาที)

### 2.1 วาง OR gate สำหรับ `B2`

1. ไปที่ **Gates > OR Gate**
2. วาง OR gate ตัวแรกกลางหน้าจอและตั้งชื่อใกล้เกตว่า `OR_B2`
3. ตั้งค่า `Number of Inputs = 4`
4. ต่อ `I4`, `I5`, `I6`, `I7` เข้า OR gate ตัวนี้
5. ต่อ output ของ `OR_B2` ไปยัง output pin `B2`

```text
I4 ─┐
I5 ─┼── OR_B2 ── B2
I6 ─┤
I7 ─┘
```

### 2.2 วาง OR gate สำหรับ `B1`

1. วาง OR gate ตัวที่สองและตั้งชื่อว่า `OR_B1`
2. ตั้งค่า `Number of Inputs = 4`
3. ต่อ `I2`, `I3`, `I6`, `I7` เข้า OR gate ตัวนี้
4. ต่อ output ของ `OR_B1` ไปยัง output pin `B1`

```text
I2 ─┐
I3 ─┼── OR_B1 ── B1
I6 ─┤
I7 ─┘
```

### 2.3 วาง OR gate สำหรับ `B0`

1. วาง OR gate ตัวที่สามและตั้งชื่อว่า `OR_B0`
2. ตั้งค่า `Number of Inputs = 4`
3. ต่อ `I1`, `I3`, `I5`, `I7` เข้า OR gate ตัวนี้
4. ต่อ output ของ `OR_B0` ไปยัง output pin `B0`

```text
I1 ─┐
I3 ─┼── OR_B0 ── B0
I5 ─┤
I7 ─┘
```

### 2.4 วาง OR gate สำหรับ `V`

1. วาง OR gate ตัวที่สี่และตั้งชื่อว่า `OR_VALID`
2. ตั้งค่า `Number of Inputs = 8`
3. ต่อ `I0` ถึง `I7` เข้า OR gate ตัวนี้
4. ต่อ output ของ `OR_VALID` ไปยัง output pin `V`

```text
I0 ─┐
I1 ─┤
I2 ─┤
I3 ─┤
I4 ─┼── OR_VALID ── V
I5 ─┤
I6 ─┤
I7 ─┘
```

### 2.5 ตรวจวงจรก่อนทดสอบ

- คลิก input ทีละตัวแล้วตรวจว่าสายเปลี่ยนสีถึง OR gate ที่เกี่ยวข้อง
- ถ้า output แสดงค่าผิดปกติ ให้ตรวจว่ามีสายชนกันโดยไม่ตั้งใจหรือไม่
- `I0` ต้องต่อเข้า `V` เท่านั้น ไม่ควรต่อเข้า `B2`, `B1`, หรือ `B0`
- `I7` ต้องต่อเข้า `B2`, `B1`, `B0`, และ `V`
- บันทึกภาพวงจร `encoder_8to3_or` ก่อนเริ่มกรอกตาราง

## ส่วนที่ 4: ทดลอง Input Active พร้อมกันหลายเส้น (25 นาที)

ในส่วนนี้ยังใช้วงจร `encoder_8to3_or` เดิม แต่เปลี่ยนรูปแบบ input ให้มีมากกว่าหนึ่งเส้น active จุดประสงค์ไม่ใช่ให้วงจรผ่านทุกแถว แต่ให้เห็นข้อจำกัดของ encoder ธรรมดา

### ตารางสังเกตความผิดพลาดของ encoder ธรรมดา

| กรณี | Input ที่ตั้งเป็น 1 | ค่า OR ที่วงจรให้ `B2B1B0` | ถ้าอ่านเป็นเลขจะกลายเป็น | ปัญหาที่ต้องอธิบาย |
|---:|---|:---:|---:|---|
| 1 | `I1`, `I2` | | | output อาจชี้ไปยังเส้นที่ไม่ได้ active |
| 2 | `I3`, `I4` | | | OR รวมบิตจนเหมือนรหัสสูงกว่า |
| 3 | `I2`, `I5` | | | output ไม่ได้บอกว่ามี input สองแหล่ง |
| 4 | `I0`, `I7` | | | output ดูเหมือน `I7` แต่ไม่รู้ว่า `I0` ก็กดอยู่ |
| 5 | `I4`, `I5`, `I6` | | | output เป็นผลรวมเชิงบิต ไม่ใช่การเลือกผู้ชนะ |
| 6 | `I0`, `I1`, `I2`, `I3` | | | valid บอกแค่ว่ามี input แต่ไม่บอกว่ามีกี่เส้น |

ตัวอย่างผังการเกิด output หลอก

```text
กดจริง:        I2 = 1, I5 = 1

              I5 ───────┐
                         ├── B2 = 1
              I2 ───────┼── B1 = 1
              I5 ───────┘
                         └── B0 = 1

ผลที่อ่านได้: B2B1B0 = 111
ความหมายผิด: เหมือน I7 active ทั้งที่ I7 = 0
```

ให้นักศึกษาตอบในรายงานสั้น ๆ หลังตารางว่า "แถวใดทำให้เห็นปัญหาชัดที่สุด" โดยอ้างค่าจริงจาก `Actual` ไม่ใช่ตอบจากความรู้สึก

## ส่วนที่ 5: สร้าง Priority Encoder 4-to-2 พร้อม Valid Bit (45 นาที)

สร้างวงจรใหม่ในไฟล์เดิมโดยคลิก **Project > Add Circuit** แล้วตั้งชื่อว่า `priority_4to2`

### 5.1 กำหนดสัญญาณ

| ชื่อสัญญาณ | ชนิด | ความหมาย |
|---|---|---|
| `D3` | input | คำขอลำดับสูงสุด |
| `D2` | input | คำขอลำดับรองจาก `D3` |
| `D1` | input | คำขอลำดับรองจาก `D2` |
| `D0` | input | คำขอลำดับต่ำสุด |
| `P1` | output | บิตสูงของรหัสผู้ชนะ |
| `P0` | output | บิตต่ำของรหัสผู้ชนะ |
| `V` | output | มี input อย่างน้อยหนึ่งเส้น active |

### 5.2 ต่อวงจรสำหรับ `P1`

1. วาง OR gate 2-input หนึ่งตัว
2. ต่อ `D3` และ `D2` เข้า OR gate
3. ต่อ output ไปยัง `P1`

```text
D3 ─┐
    ├── OR ── P1
D2 ─┘
```

### 5.3 ต่อวงจรสำหรับ `P0`

1. วาง NOT gate หนึ่งตัวเพื่อสร้าง `not D2`
2. วาง AND gate หนึ่งตัวเพื่อสร้าง `not D2 · D1`
3. วาง OR gate หนึ่งตัวเพื่อรวม `D3` กับผลจาก AND gate
4. ต่อ output ของ OR gate ไปยัง `P0`

```text
D2 ── NOT ──┐
            ├── AND ──┐
D1 ─────────┘         ├── OR ── P0
D3 ───────────────────┘
```

เหตุผลที่ไม่ต้องใส่ `not D3` ในพจน์ของ `D1` คือถ้า `D3 = 1` เอาต์พุต `P0` ต้องเป็น 1 อยู่แล้วเพราะผู้ชนะคือ `D3` และรหัสของ `D3` คือ `11`

### 5.4 ต่อวงจรสำหรับ `V`

1. วาง OR gate 4-input หนึ่งตัว
2. ต่อ `D3`, `D2`, `D1`, `D0` เข้า OR gate
3. ต่อ output ไปยัง `V`

```text
D3 ─┐
D2 ─┤
D1 ─┼── OR ── V
D0 ─┘
```

### 5.5 ผังรวมที่ควรได้

```text
                      ┌────────────── P1 = D3 + D2
D3 ───────────────────┤
                      │
D2 ───────┬───────────┘
          │
          └── NOT ──┐
                    ├── AND ──┐
D1 ─────────────────┘         ├── OR ── P0 = D3 + (not D2 · D1)
D3 ───────────────────────────┘

D3 ─┐
D2 ─┤
D1 ─┼── OR ── V
D0 ─┘
```

### 5.6 ตาราง priority แบบย่อด้วย don't care

| เงื่อนไข input `D3 D2 D1 D0` | ผู้ชนะ | Expected `P1P0` | Expected `V` |
|---|---:|:---:|:---:|
| `0000` | ไม่มี | `00` | 0 |
| `0001` | `D0` | `00` | 1 |
| `001x` | `D1` | `01` | 1 |
| `01xx` | `D2` | `10` | 1 |
| `1xxx` | `D3` | `11` | 1 |

สัญลักษณ์ `x` หมายถึงค่า 0 หรือ 1 ก็ได้ เพราะถูก input ที่มี priority สูงกว่ากลบไปแล้ว เช่น `01xx` หมายถึงทุกกรณีที่ `D3=0` และ `D2=1` ไม่ว่า `D1` กับ `D0` จะเป็นอะไร ผู้ชนะยังคงเป็น `D2`

### 5.7 ตารางบันทึกผล priority encoder

| กรณี | `D3` | `D2` | `D1` | `D0` | ผู้ชนะตาม priority | Expected `P1P0` | Expected `V` | Actual `P1P0` | Actual `V` | ผ่าน/ไม่ผ่าน |
|---:|:---:|:---:|:---:|:---:|---:|:---:|:---:|:---:|:---:|:---:|
| 1 | 0 | 0 | 0 | 0 | ไม่มี | `00` | 0 | | | |
| 2 | 0 | 0 | 0 | 1 | `D0` | `00` | 1 | | | |
| 3 | 0 | 0 | 1 | 0 | `D1` | `01` | 1 | | | |
| 4 | 0 | 0 | 1 | 1 | `D1` | `01` | 1 | | | |
| 5 | 0 | 1 | 0 | 0 | `D2` | `10` | 1 | | | |
| 6 | 0 | 1 | 0 | 1 | `D2` | `10` | 1 | | | |
| 7 | 0 | 1 | 1 | 0 | `D2` | `10` | 1 | | | |
| 8 | 0 | 1 | 1 | 1 | `D2` | `10` | 1 | | | |
| 9 | 1 | 0 | 0 | 0 | `D3` | `11` | 1 | | | |
| 10 | 1 | 0 | 0 | 1 | `D3` | `11` | 1 | | | |
| 11 | 1 | 0 | 1 | 0 | `D3` | `11` | 1 | | | |

## ส่วนที่ 6: เปรียบเทียบ Encoder ธรรมดากับ Priority Encoder (15 นาที)

ตั้ง input เหมือนกันในสองวงจร แล้วบันทึกว่า output ต่างกันอย่างไร ให้ใช้ตารางนี้กับวงจร `encoder_8to3_or` และ `priority_4to2`

สำหรับการเปรียบเทียบ ให้ map input 4 เส้นล่างของ encoder ธรรมดาเป็น `D0 = I0`, `D1 = I1`, `D2 = I2`, `D3 = I3`

| กรณี | Input active | Encoder ธรรมดา output | Priority encoder output | ข้อสรุป |
|---:|---|:---:|:---:|---|
| 1 | `I0` / `D0` | | | ทั้งสองควรแสดงรหัส 0 แต่ต้องดู `V` |
| 2 | `I1`, `I2` / `D1`, `D2` | | | encoder ธรรมดาอาจให้ `011`; priority ต้องให้ `10` |
| 3 | `I0`, `I3` / `D0`, `D3` | | | priority ต้องเลือก `D3` |
| 4 | `I1`, `I2`, `I3` / `D1`, `D2`, `D3` | | | priority ต้องเลือก `D3` |
| 5 | ไม่มี input active | | | output binary ต้องอ่านร่วมกับ valid bit |

ในรายงานให้เขียนคำอธิบาย 5-8 บรรทัดว่า ถ้าใช้วงจรเหล่านี้กับปุ่มกด 4 ปุ่มโดยไม่มีวงจร debounce ผลการกดเกือบพร้อมกันจะตีความอย่างไร และเหตุใด priority encoder ยังไม่ใช่วงจรแก้ปัญหาสัญญาณเด้งของปุ่มกด

## ส่วนที่ 7: ตัวเลือกต่อไอซี 74148 บน Breadboard (30 นาที)

ส่วนนี้ทำเมื่อมีอุปกรณ์จริงและเวลาพอ ไอซี 74148 หรือ 74LS148 เป็น priority encoder 8-to-3 ที่ input และ output เป็น active-low หมายความว่า input ที่ถูกกดจะเป็น `0` ไม่ใช่ `1` และ output ที่อ่านได้ต้องกลับตรรกะก่อนเทียบกับรหัส active-high

### 7.1 ขาสำคัญของ 74148 แบบ DIP-16

### 7.2 วิธีต่อเพื่อทดสอบ input active-low

1. เสียบไอซีข้ามร่องกลาง breadboard โดยให้รอยบากอยู่ด้านบน
2. ต่อขา 16 เข้ากับ `+5 V`
3. ต่อขา 8 เข้ากับ `GND`
4. ต่อขา 5 (`EI`) ลง `GND` เพื่อ enable ไอซี
5. ต่อ input `0` ถึง `7` ผ่านตัวต้านทาน pull-up 10 kΩ ไปที่ `+5 V`
6. ต่อปุ่มกดหรือ jumper จาก input แต่ละเส้นลง `GND` เมื่อต้องการให้ input นั้น active
7. ต่อ LED พร้อมตัวต้านทาน 330 Ω จาก `+5 V` ไปยังขา output `A2`, `A1`, `A0`, `GS` ถ้า output เป็น LOW LED จะติด
8. ทดสอบโดยดึง input `3` ลง `GND` เพียงเส้นเดียว แล้วอ่าน output active-low
9. ทดสอบโดยดึง input `3` และ `6` ลง `GND` พร้อมกัน แล้วตรวจว่า input `6` ชนะ
10. ห้ามปล่อย input ลอย เพราะ TTL input ที่ลอยอาจอ่านเป็น HIGH อย่างไม่เสถียรและทำให้ผลทดลองไม่น่าเชื่อถือ

```text
              │

74148 input ──┼──── jumper/switch ─── GND
              │

          active   = LOW
```

## ส่วนที่ 8: ประยุกต์กับปุ่มกดและ Interrupt Priority (15 นาที)

ระบบไมโครคอนโทรลเลอร์มักมีคำขอ interrupt หลายแหล่ง เช่น ปุ่มหยุดฉุกเฉิน ตัวจับเวลา การรับข้อมูลอนุกรม และเซนเซอร์เตือนภัย ถ้าทุกแหล่งส่งคำขอพร้อมกัน ระบบต้องเลือกเหตุการณ์ที่สำคัญที่สุดก่อน ไม่ใช่รวมรหัสด้วย OR แบบ encoder ธรรมดา

ตัวอย่างระบบปุ่มกด 4 ปุ่ม

```text
KEY_STOP   ── D3  priority สูงสุด  ┐
KEY_ALARM  ── D2                  │
KEY_MENU   ── D1                  ├── Priority Encoder ── code P1P0 ── CPU
KEY_SELECT ── D0  priority ต่ำสุด ┘                         │
                                                             └── V = interrupt request
```

การตีความในระบบนี้คือ

| ปุ่มที่กดพร้อมกัน | ผู้ชนะ | รหัสส่งเข้า CPU | เหตุผล |
|---|---:|:---:|---|
| `KEY_SELECT` | `D0` | `00` | มีปุ่มเดียวและ `V = 1` |
| `KEY_MENU`, `KEY_SELECT` | `D1` | `01` | เมนูสำคัญกว่าเลือก |
| `KEY_ALARM`, `KEY_MENU` | `D2` | `10` | alarm สำคัญกว่าเมนู |
| `KEY_STOP`, `KEY_ALARM`, `KEY_SELECT` | `D3` | `11` | stop ฉุกเฉินสูงสุด |

คำเตือนเชิงออกแบบ: priority encoder ไม่ได้จำเหตุการณ์ที่ถูกกลบ ถ้า `D3` และ `D1` active พร้อมกัน output จะบอกเฉพาะ `D3` หากระบบต้องกลับไปจัดการ `D1` ภายหลัง จะต้องมี latch, register, interrupt pending bit หรือซอฟต์แวร์อ่านสถานะ input เพิ่มเติม

## คำถามท้ายใบงาน

1. เพราะเหตุใด encoder 8-to-3 ธรรมดาจึงต้องการ input แบบ one-hot หากไม่ใช่ one-hot จะเกิดปัญหาอย่างไร จงยกตัวอย่างจากตาราง multi-active ของกลุ่มตนเอง
2. กรณีไม่มี input active กับกรณี `I0` active ให้ output binary เป็น `000` เหมือนกัน เหตุใดจึงต้องมี valid bit และถ้าไม่มี valid bit ระบบปลายทางจะตีความผิดอย่างไร
3. Decoder และ encoder เป็นวงจรที่ดูเหมือนตรงข้ามกัน แต่ไม่ได้แทนกันได้ทุกกรณี จงอธิบายด้วยตัวอย่าง 3-to-8 decoder และ 8-to-3 encoder
4. ใน priority encoder 4-to-2 ที่ใช้ลำดับ `D3 > D2 > D1 > D0` ทำไมสมการ `P0 = D3 + (not D2 · D1)` จึงไม่ต้องมี `D0`
5. ถ้าใช้ priority encoder กับปุ่มกดจริงแล้วกด `D2` ค้างไว้ จากนั้นกด `D3` เพิ่ม ระบบจะอ่าน output เปลี่ยนอย่างไร และเมื่อปล่อย `D3` แต่ยังค้าง `D2` อยู่ output ควรเป็นอะไร
6. ไอซี 74148 ใช้ input และ output แบบ active-low ต่างจากวงจร Logisim active-high ที่สร้างในส่วนที่ 5 อย่างไร หากต้องต่อ LED เพื่อแสดง output ควรตีความ LED ติดและดับอย่างไร
7. Priority encoder บอกผู้ชนะเพียงหนึ่งเส้น ถ้าระบบ interrupt ต้องไม่สูญเสียคำขอที่ priority ต่ำกว่า ควรเพิ่มวงจรหรือแนวคิดใดต่อจาก encoder
8. เปรียบเทียบจำนวน gate ระหว่าง encoder 8-to-3 ธรรมดากับ priority encoder 4-to-2 ที่สร้างในคาบนี้ วงจรใดต่อยากกว่า เพราะเหตุใด

## ข้อควรระวังเฉพาะแลปนี้

- ห้ามใช้ built-in priority encoder ของ Logisim แทนวงจร gate ในส่วนที่ 5 เพราะต้องการตรวจสมการ priority
- ห้ามสรุปว่า encoder 8-to-3 ธรรมดา "เสีย" เมื่อ input หลายเส้น active เพราะพฤติกรรมนั้นเป็นผลจากเงื่อนไข input ไม่เป็น one-hot
- เมื่อเห็น output `000` ต้องอ่าน `V` คู่กันเสมอ
- ถ้าสาย `I7` ไม่ต่อเข้าครบทั้ง `B2`, `B1`, `B0`, `V` ตารางแถว `I7` จะผิดทันที
- ถ้าสาย `D0` ต่อเข้า `P0` ใน priority encoder 4-to-2 ให้ตรวจใหม่ เพราะ `D0` ควรมีผลต่อ `V` เท่านั้น
- ในวงจร 74148 ห้ามต่อ input active-low เข้ากับสวิตช์แบบปล่อยลอย ต้องมี pull-up หรือกำหนดระดับตรรกะให้แน่นอน
- ปิดแหล่งจ่ายก่อนเปลี่ยนสายบน breadboard และตรวจขา 16 กับขา 8 ก่อนจ่ายไฟทุกครั้ง
