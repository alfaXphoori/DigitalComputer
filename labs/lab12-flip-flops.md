## ทฤษฎีย่อเฉพาะการทดลองนี้

วงจรเชิงลำดับ (sequential circuit) มี output ที่ขึ้นกับทั้ง input ปัจจุบันและสถานะเดิม ดังนั้นสัญญาณ `Q` ของ latch หรือ flip-flop ไม่ได้เปลี่ยนทุกครั้งที่ input เปลี่ยนเสมอไป แต่เปลี่ยนตามกติกาของอุปกรณ์จำสถานะนั้น

ในแลปนี้ให้ใช้สัญลักษณ์ต่อไปนี้ให้สม่ำเสมอ

| สัญลักษณ์ | ความหมาย | วิธีอ่านในการทดลอง |
|---|---|---|
| `Q` | สถานะปัจจุบัน (present state) | ค่า LED/probe ก่อนกด tick ถัดไป |
| `Q(next)` หรือ `Q+` | สถานะถัดไป (next state) | ค่า LED/probe หลัง clock edge หรือหลัง latch รับคำสั่ง |
| `CLK` | clock | สัญญาณจังหวะที่ทำให้ flip-flop รับข้อมูล |
| `D` | data input | ค่าที่ D flip-flop จะคัดลอกเข้า `Q` ที่ขอบ clock |
| `S`, `R` | set, reset | คำสั่งบังคับ latch ให้เป็น 1 หรือ 0 |
| `J`, `K` | input ของ JK flip-flop | 00 hold, 01 reset, 10 set, 11 toggle |
| `T` | toggle input | 0 hold, 1 toggle |
| `PRE`, `CLR` | preset, clear | ขา asynchronous สำหรับบังคับ `Q` โดยไม่รอ clock |

### SR latch จาก NOR gate

SR latch แบบ NOR ใช้ NOR gate 2 ตัวต่อ feedback ไขว้กัน ถ้าใช้ active-high input จะมีพฤติกรรมดังนี้

| S | R | Q(next) | ความหมาย |
|:---:|:---:|:---:|---|
| 0 | 0 | Q | Hold - จำค่าเดิม |
| 1 | 0 | 1 | Set - บังคับให้ `Q = 1` |
| 0 | 1 | 0 | Reset - บังคับให้ `Q = 0` |
| 1 | 1 | ไม่อนุญาต | Forbidden - output ทั้งคู่ถูกบังคับผิดธรรมชาติของคู่ `Q/Q'` |

ภาพวงจรที่ต้องสร้างใน Logisim ใช้ NOR gate สองตัวต่อกลับกันดังนี้

```text
              +---------+
      S ------| NOR     |------ Q
              |         |        |
          +---|         |        |
          |   +---------+        |
          |                      |
          |                      v
          |   +---------+        |
          +---|         |<-------+
              | NOR     |
      R ------|         |------ Q'
              +---------+
```

สำหรับ NOR latch ห้ามใช้ `S=1` และ `R=1` เป็นสถานะใช้งาน เพราะเมื่อปล่อยกลับเป็น `S=0, R=0` ผลสุดท้ายอาจขึ้นกับ delay ของ gate และลำดับการปล่อยสวิตช์ ทำให้คาดการณ์ไม่ได้

### Edge-triggered flip-flop

Flip-flop แบบ edge-triggered รับ input เฉพาะตอนเกิดขอบ clock เช่น ขอบขาขึ้น (rising edge) จาก 0 เป็น 1 ถ้า input เปลี่ยนระหว่าง clock คงที่ `Q` จะไม่เปลี่ยนจนกว่าจะถึง edge ถัดไป

```text
CLK : ___|‾‾‾|___|‾‾‾|___|‾‾‾|___
edge:    ↑       ↑       ↑
D   : 0_____1_______0_1_______1___
Q   : 0_____1_______0_________1___
```

อ่าน timing diagram จากซ้ายไปขวา โดยให้ค่า `Q` เปลี่ยนเฉพาะตำแหน่งลูกศรของ clock เท่านั้น กรณีที่ `D` เปลี่ยนก่อนหรือหลัง edge ต้องดูว่าค่า `D` ณ ขอบ clock เป็นอะไร

### Characteristic equation ที่ต้องใช้เทียบผล

| ชนิด | กฎของ `Q(next)` | ความหมายเชิงทดลอง |
|---|---|---|
| D flip-flop | `Q(next) = D` | คัดลอกค่า `D` เข้า `Q` ที่ clock edge |
| JK flip-flop | `Q(next) = JQ' + K'Q` | รวม hold, reset, set และ toggle ในอุปกรณ์เดียว |
| T flip-flop | `Q(next) = T XOR Q` | ถ้า `T=1` จะกลับค่าทุก clock edge |

## ส่วนที่ 1: สร้าง SR latch จาก NOR gate ใน Logisim (35 นาที)

### ขั้นตอนการวางวงจร

1. เปิด Logisim Evolution แล้วเลือก **File > New**
2. บันทึกไฟล์เป็น `lab12-flip-flops.circ`
3. คลิกขวาที่ชื่อ project แล้วเลือก **Add Circuit...** ตั้งชื่อ `01_SR_Latch`
4. เลือก **Gates > NOR Gate** วาง NOR gate 2 ตัวเรียงบน-ล่าง
5. ตั้ง attribute ของ NOR gate ทั้งสองตัวเป็น `Number of Inputs = 2`
6. เลือก **Wiring > Pin** วาง input pin 2 ตัวด้านซ้าย ตั้งชื่อ `S` และ `R`
7. ตั้ง attribute ของ input pin เป็น `Output? = No`, `Data Bits = 1`
8. เลือก **Wiring > Probe** หรือ output pin วางด้านขวา 2 ตัว ตั้งชื่อ `Q` และ `Q_bar`
9. ต่อ `S` เข้าขา input หนึ่งของ NOR gate ตัวบน
10. ต่อ `R` เข้าขา input หนึ่งของ NOR gate ตัวล่าง
11. ต่อ output ของ NOR gate ตัวบนไปยัง probe `Q`
12. ต่อ output ของ NOR gate ตัวล่างไปยัง probe `Q_bar`
13. ต่อ feedback จาก `Q` กลับไปยัง input ที่เหลือของ NOR gate ตัวล่าง
14. ต่อ feedback จาก `Q_bar` กลับไปยัง input ที่เหลือของ NOR gate ตัวบน
15. ใช้ **Poke Tool** เปลี่ยนค่า `S` และ `R` ตามตารางทดลอง

### ตำแหน่งสัญญาณที่ควรได้

```text
S input        NOR บน                         Q probe
  o------------\                              o
                )o---------------------------+
Q_bar feedback /                             |
                                             |
                                             |
Q feedback    \                              |
                )o---------------------------+---- Q_bar probe
R input       /
  o----------/
```

### ลำดับการทดลอง SR latch

ให้เริ่มจาก `S=0, R=1` เพื่อ reset ให้ `Q=0` ก่อน แล้วจึงทดลองตามลำดับด้านล่าง ห้ามข้ามลำดับ เพราะต้องเห็นผลของการจำค่าเดิม

| ลำดับ | S | R | คำสั่งที่ต้องทำ | Q คาดหมาย | Q_bar คาดหมาย | Q ที่วัดได้ | Q_bar ที่วัดได้ | หมายเหตุ |
|---:|:---:|:---:|---|:---:|:---:|:---:|:---:|---|
| 1 | 0 | 1 | Reset เริ่มต้น | 0 | 1 | | | |
| 2 | 0 | 0 | ปล่อยกลับ Hold | 0 | 1 | | | ต้องจำค่า 0 |
| 3 | 1 | 0 | Set | 1 | 0 | | | |
| 4 | 0 | 0 | ปล่อยกลับ Hold | 1 | 0 | | | ต้องจำค่า 1 |
| 5 | 0 | 1 | Reset อีกครั้ง | 0 | 1 | | | |
| 6 | 1 | 1 | Forbidden | 0 | 0 | | | สำหรับ NOR latch output มักเป็น 0 ทั้งคู่ |
| 7 | 0 | 0 | ปล่อยจาก forbidden | ไม่แน่นอน | ไม่แน่นอน | | | บันทึกผลจริง |
| 8 | 1 | 0 | Set เพื่อกู้สถานะ | 1 | 0 | | | |

### สิ่งที่ต้องสังเกต

- แถวที่ `S=0, R=0` ไม่ใช่คำสั่งให้ output เป็น 0 แต่คือคำสั่งให้จำค่าเดิม
- แถว forbidden ทำให้ `Q` และ `Q_bar` ไม่เป็นค่าตรงข้ามกัน จึงไม่ควรนำไปใช้แทนบิตข้อมูล
- หลังออกจาก forbidden ผลอาจต่างกันระหว่างกลุ่มหรือระหว่างการจำลอง ขึ้นกับลำดับ delay ภายใน simulation

## ส่วนที่ 2: สร้าง D flip-flop แบบ edge-triggered (35 นาที)

### ขั้นตอนใน Logisim

1. สร้าง circuit ใหม่ชื่อ `02_D_FF`
2. เลือก **Memory > D Flip-Flop** วางไว้กลางพื้นที่ทำงาน
3. ถ้า component มี attribute `Trigger` ให้ตั้งเป็น `Rising Edge`
4. เลือก **Wiring > Pin** วาง input pin ชื่อ `D`
5. เลือก **Wiring > Clock** วาง clock component ชื่อ `CLK`
6. ต่อ `D` เข้าขา D ของ flip-flop
7. ต่อ `CLK` เข้าขา clock ของ flip-flop
8. วาง probe หรือ output pin ชื่อ `Q_D` ที่ output `Q`
9. วาง probe หรือ output pin ชื่อ `Q_D_bar` ที่ output complement ถ้ามี
10. เปิดเมนู **Simulate** แล้วตรวจว่า `Ticks Enabled` ยังไม่ถูกเปิด เพื่อให้เดิน clock ด้วยมือก่อน
11. ใช้ **Simulate > Tick Once** หรือปุ่มลัด tick ทีละครั้ง แล้วบันทึกค่า `Q_D`

### วงจรที่คาดหวัง

```text
       +----------------+
D ---->| D          Q   |---- Q_D
CLK -->| >              |
       | D Flip-Flop    |---- Q_D_bar
       +----------------+
```

### ตัวอย่างที่ 1: อ่านค่า D เฉพาะที่ขอบ clock

กำหนดเริ่มต้น `Q=0` แล้วทดลองค่าตามตารางนี้ ก่อนกด tick ให้ตั้งค่า `D` ตามแถวปัจจุบัน จากนั้นกด tick หนึ่งครั้ง แล้วอ่าน `Q(after edge)`

| Tick | D ก่อนขอบ clock | Q ก่อน tick | Q(after edge) คาดหมาย | Q(after edge) ที่วัดได้ | เหตุผล |
|---:|:---:|:---:|:---:|:---:|---|
| 1 | 0 | 0 | 0 | | `Q(next)=D=0` |
| 2 | 1 | 0 | 1 | | คัดลอก 1 เข้า Q |
| 3 | 1 | 1 | 1 | | D ยังเป็น 1 |
| 4 | 0 | 1 | 0 | | คัดลอก 0 เข้า Q |
| 5 | 0 | 0 | 0 | | D ยังเป็น 0 |
| 6 | 1 | 0 | 1 | | คัดลอก 1 เข้า Q |

ให้เพิ่มการสังเกต 1 ครั้งโดยเปลี่ยน `D` ระหว่างที่ยังไม่กด tick แล้วดูว่า `Q_D` เปลี่ยนทันทีหรือไม่ ผลที่ถูกต้องคือ `Q_D` ไม่เปลี่ยนจนกว่าจะเกิด clock edge

### ตัวอย่างที่ 2: คำนวณ JK จากสมการ characteristic

ต้องแสดงการคำนวณในรายงานอย่างน้อย 2 แถว ตัวอย่างต่อไปนี้ใช้สมการ

`Q(next) = JQ' + K'Q`

กรณีที่ 1: `J=1, K=1, Q=0`

```text
Q'      = 1
K'      = 0
Q(next) = JQ' + K'Q
        = (1)(1) + (0)(0)
        = 1 + 0
        = 1
```

ดังนั้นเมื่อ `J=K=1` และ `Q=0` หลัง clock edge ต้องได้ `Q(next)=1`

กรณีที่ 2: `J=0, K=1, Q=1`

```text
Q'      = 0
K'      = 0
Q(next) = JQ' + K'Q
        = (0)(0) + (0)(1)
        = 0 + 0
        = 0
```

ดังนั้นแถวนี้เป็น reset เพราะ `Q` เปลี่ยนจาก 1 เป็น 0

ถ้า `Q` ไม่ toggle ทุก tick ให้ตรวจว่า clock ต่อเข้าขา clock จริง ไม่ได้ต่อเข้าขา enable หรือขา asynchronous โดยผิดตำแหน่ง

## ส่วนที่ 4: สร้าง T flip-flop จาก JK flip-flop (20 นาที)

ในทางปฏิบัติ T flip-flop สร้างจาก JK flip-flop ได้โดยผูก `J` และ `K` เข้าด้วยกันเป็นสัญญาณ `T`

```text
T ----+---- J
      |
      +---- K

เมื่อ T = 0  -> J=0, K=0 -> Hold
เมื่อ T = 1  -> J=1, K=1 -> Toggle
```

### ขั้นตอนใน Logisim

1. สร้าง circuit ใหม่ชื่อ `04_T_From_JK`
2. วาง JK Flip-Flop 1 ตัว
3. วาง input pin ชื่อ `T`
4. ต่อสายจาก `T` แยกเป็นสองเส้นเข้าขา `J` และ `K`
5. วาง clock component ชื่อ `CLK_T`
6. ต่อ output `Q` ไปยัง probe ชื่อ `Q_T`
7. ทดลองตามตาราง โดยเดิน clock หนึ่ง tick ต่อหนึ่งแถว

### ตารางทดสอบ T flip-flop

| Tick | T | Q ก่อน tick | Q(next) คาดหมาย | Q(next) ที่วัดได้ | ความหมาย |
|---:|:---:|:---:|:---:|:---:|---|
| 1 | 0 | 0 | 0 | | Hold |
| 2 | 1 | 0 | 1 | | Toggle |
| 3 | 1 | 1 | 0 | | Toggle |
| 4 | 0 | 0 | 0 | | Hold |
| 5 | 1 | 0 | 1 | | Toggle |
| 6 | 0 | 1 | 1 | | Hold |

### ตัวอย่างที่ 3: ตรวจ T ด้วยสมการ XOR

ถ้า `T=1` และ `Q=1`

```text
Q(next) = T XOR Q
        = 1 XOR 1
        = 0
```

ถ้า `T=0` และ `Q=1`

```text
Q(next) = T XOR Q
        = 0 XOR 1
        = 1
```

ดังนั้น `T=0` คือการจำค่าเดิม ส่วน `T=1` คือการกลับค่าทุก clock edge ซึ่งเป็นพื้นฐานของ counter แบบนับไบนารี

---

## ส่วนที่ 5: ทดลอง asynchronous Preset/Clear (25 นาที)

Preset และ Clear เป็นขาที่มีผลทันทีโดยไม่ต้องรอ clock จึงเรียกว่า asynchronous input ใน flip-flop จริงชื่อขาอาจเป็น active-low เช่น `PRE'` และ `CLR'` แต่ใน Logisim ต้องตรวจ attribute หรือสัญลักษณ์ของ component ว่าขานั้น active-high หรือ active-low

### ขั้นตอนใน Logisim

1. สร้าง circuit ใหม่ชื่อ `05_Async_PreClr`
2. วาง D Flip-Flop 1 ตัว หรือ JK Flip-Flop 1 ตัวที่มีขา preset/clear
3. ถ้าขา preset/clear ไม่แสดง ให้ดู attribute ของ component และเปิดขา asynchronous control ตามที่โปรแกรมรองรับ
4. วาง input pin ชื่อ `PRE` และ `CLR`
5. ต่อ `PRE` เข้าขา preset และ `CLR` เข้าขา clear
6. ต่อ `D` หรือ `J/K` และ clock ตามปกติ
7. วาง probe `Q_ASYNC`
8. ตั้งค่า input หลักให้คงที่ก่อน เช่น `D=0`
9. เปลี่ยน `PRE` แล้วสังเกตว่า `Q` เปลี่ยนโดยไม่ต้องกด tick หรือไม่
10. เปลี่ยน `CLR` แล้วสังเกตว่า `Q` เปลี่ยนโดยไม่ต้องกด tick หรือไม่

### ตารางทดลอง Preset/Clear

ให้ปรับคำว่า active เป็นค่าจริงตาม component ที่ใช้ เช่น active-high หรือ active-low

| ลำดับ | CLK | D/J/K | PRE | CLR | Q ก่อนเปลี่ยน | Q หลังเปลี่ยน | ต้องกด tick หรือไม่ | ข้อสังเกต |
|---:|---|---|:---:|:---:|:---:|:---:|---|---|
| 1 | คงที่ | D=0 | inactive | inactive | 0 | 0 | ไม่ | สถานะเริ่มต้น |
| 2 | คงที่ | D=0 | active | inactive | 0 | 1 | ไม่ | Preset บังคับ Q=1 |
| 3 | คงที่ | D=1 | inactive | inactive | 1 | 1 | ไม่ | ปล่อย preset แล้ว Q ควรคงเดิม |
| 4 | คงที่ | D=1 | inactive | active | 1 | 0 | ไม่ | Clear บังคับ Q=0 |
| 5 | ↑ | D=1 | inactive | inactive | 0 | 1 | ใช่ | กลับมาทำงานตาม clock |
| 6 | คงที่ | D=1 | active | active | | | ไม่ควรใช้ | บันทึกผลและระบุว่าเป็นสถานะห้ามใช้ |

### ข้อควรระวังของ asynchronous input

- ในระบบจริงไม่ควรปล่อย `PRE` หรือ `CLR` ลอย เพราะ noise อาจบังคับ state โดยไม่ตั้งใจ
- ถ้า `PRE` และ `CLR` active พร้อมกัน มักเป็นสถานะห้ามใช้หรือไม่รับประกันผล
- เมื่อปล่อย asynchronous reset ใกล้ clock edge อาจเกิดปัญหา timing ในวงจรใหญ่ จึงควรออกแบบ reset release ให้เหมาะสม

## ส่วนที่ 6: ตัวเลือกการต่อไอซี 7474 และ 7476 บนเบรดบอร์ด (30 นาที)

ส่วนนี้ทำเมื่อมีอุปกรณ์จริงและได้รับอนุญาตจากผู้สอน ให้ปิดแหล่งจ่ายทุกครั้งก่อนเปลี่ยนสาย และตรวจ datasheet ของไอซีรุ่นที่ใช้จริงก่อนจ่ายไฟ

### 7474: Dual D positive-edge-triggered flip-flop

ไอซี 7474 มี D flip-flop 2 ตัว ขา preset และ clear โดยทั่วไปเป็น active-low สังเกตสัญลักษณ์ขีดบนชื่อขา เช่น `/PRE` และ `/CLR`

| ขา 7474 | หน้าที่โดยทั่วไป | การต่อในการทดลอง D FF ตัวที่ 1 |
|---:|---|---|
| 14 | VCC | +5 V |
| 7 | GND | 0 V |
| 2 | 1D | สวิตช์ D ผ่าน pull-up/pull-down |
| 3 | 1CLK | ปุ่ม clock พร้อมวงจร debounce ถ้ามี |
| 5 | 1Q | LED ผ่าน resistor 330 Ω |
| 6 | 1Q' | LED ผ่าน resistor 330 Ω หรือเว้นไว้ |
| 1 | 1CLR | ต่อ HIGH เมื่อไม่ใช้งาน clear |
| 4 | 1PRE | ต่อ HIGH เมื่อไม่ใช้งาน preset |

วงจรต่อปุ่ม clock ขั้นต่ำควรมี pull-down หรือ pull-up ให้สถานะนิ่ง และควรใส่ debounce ถ้าต้องการผลนับที่เสถียร

```text
+5V ----[10k]----+---- CLK (7474 pin 3)
                 |
              push button
                 |
GND -------------+

Q (pin 5) ----[330]---->|---- GND
```

ถ้ากดปุ่มหนึ่งครั้งแล้ว `Q` เปลี่ยนมากกว่าหนึ่งครั้ง สาเหตุที่เป็นไปได้คือ switch bounce ไม่ใช่ logic ของ D flip-flop ผิด

### 7476: Dual JK flip-flop

ไอซี 7476 เป็น JK flip-flop ที่มี preset/clear เช่นกัน pinout อาจต่างตามผู้ผลิต ต้องเปิด datasheet ของเบอร์ที่มีในห้องปฏิบัติการก่อนต่อจริง

| สัญญาณ | การต่อเพื่อทดสอบ toggle |
|---|---|
| `J` | ต่อ HIGH |

| `/CLR` | ต่อ HIGH เมื่อไม่ใช้งาน |
| `Q` | ต่อ LED ผ่าน resistor |
| VCC/GND | ต่อ +5 V และ 0 V ตาม datasheet |

กด clock ครั้งที่ 1: Q 0 -> 1
กด clock ครั้งที่ 2: Q 1 -> 0

## แนวทางวิเคราะห์ผลในรายงาน

ให้เขียนวิเคราะห์แบบอ้างอิงผลวัดจริงอย่างน้อย 1 ย่อหน้าต่อหัวข้อต่อไปนี้

1. แถว `S=0, R=0` ของ SR latch แสดงการจำค่าเดิมอย่างไร โดยยกหลักฐานจากแถวก่อนหน้าในตาราง
2. แถว forbidden ของ SR latch ทำให้ `Q` และ `Q_bar` ผิดเงื่อนไข complement อย่างไร
3. D flip-flop ต่างจากสายตรง `D -> Q` อย่างไร เมื่อเปลี่ยน `D` แต่ยังไม่เกิด clock edge
4. JK flip-flop แก้ปัญหาสถานะต้องห้ามของ SR ได้อย่างไรในกรณี `J=K=1`
5. asynchronous Preset/Clear มีลำดับความสำคัญเหนือ clock อย่างไรจากผลทดลอง

---

## คำถามท้ายใบงาน

1. SR latch และ D flip-flop ต่างกันอย่างไรในแง่ level-sensitive และ edge-triggered ให้ยกตัวอย่างจากผลทดลองของตนเอง
2. ทำไม `S=1, R=1` ของ NOR SR latch จึงเป็นสถานะต้องห้าม และหลังปล่อยกลับ `S=0, R=0` จึงไม่ควรใช้ผลลัพธ์เป็นข้อมูลสำคัญ
3. ถ้า D flip-flop เป็น rising-edge triggered แล้ว `D` เปลี่ยนจาก 0 เป็น 1 หลังขอบ clock ไปเล็กน้อย `Q` ควรเปลี่ยนทันทีหรือรอถึงขอบถัดไป เพราะเหตุใด
4. ใช้สมการ `Q(next)=JQ' + K'Q` พิสูจน์ว่า JK flip-flop เมื่อ `J=K=1` จะ toggle ทั้งกรณี `Q=0` และ `Q=1`
5. T flip-flop ที่ได้จากการผูก `J=K=T` เหมาะกับการสร้าง counter อย่างไร
6. Preset/Clear แบบ asynchronous ต่างจากการป้อน `D=1` หรือ `D=0` แล้วรอ clock อย่างไร
7. ถ้าต่อ 7474 จริงแล้วกดปุ่ม clock หนึ่งครั้งแต่ LED เปลี่ยนหลายครั้ง ปัญหาเกิดจากอะไรได้บ้าง และควรแก้อย่างไร
8. ในการออกแบบวงจรลำดับขนาดใหญ่ เหตุใดจึงต้องระวัง setup time, hold time และการปล่อย reset ใกล้ clock edge
