# 🔬 Lab 15: โปรเจกต์ย่อย: ออกแบบวงจรดิจิทัลด้วย FSM

**รายวิชา:** ตรรกศาสตร์ของดิจิตอลคอมพิวเตอร์ (Digital Computer Logic)  
**หลักสูตร:** วิศวกรรมคอมพิวเตอร์ ชั้นปีที่ 1  
**เวลาปฏิบัติการ:** 3 ชั่วโมง  
**เครื่องมือ:** Logisim Evolution, ไอซี 74xx ตามหัวข้อที่เลือก, breadboard, 7-segment display, LEDs, push button  

---

## วัตถุประสงค์

- ออกแบบวงจรลำดับ (Sequential Circuit) จากข้อกำหนดของระบบจริงขนาดเล็กได้
- แปลงพฤติกรรมของระบบเป็น state diagram, state table และสมการ next-state/output ได้
- ใช้ flip-flop, counter, register, decoder และวงจรแสดงผลร่วมกันในโปรเจกต์เดียว
- สร้างวงจรใน Logisim Evolution ให้ทดสอบซ้ำได้ด้วย clock, button, probe และ display
- จัดทำรายงานวิศวกรรมที่อธิบายเหตุผลของการเลือก state, encoding, test case และข้อจำกัดของวงจร
- นำเสนอวงจรพร้อมสาธิตการทำงานและตอบคำถามเรื่อง timing, reset, invalid state และ debouncing ได้

## ภาพรวมโปรเจกต์

Lab 15 เป็นงานสรุปของหัวข้อวงจรลำดับจากบทที่ 9-10 โดยให้นักศึกษาออกแบบวงจรดิจิทัลหนึ่งระบบที่มีสถานะภายใน ไม่ใช่เพียงวงจร combinational ที่ input เปลี่ยนแล้ว output เปลี่ยนทันที ระบบที่เลือกต้องมีอย่างน้อยหนึ่งส่วนที่ใช้ clock และมีการเปลี่ยนสถานะตามลำดับเวลา เช่น counter, finite state machine (FSM), shift register หรือการควบคุม 7-segment ตามสถานะ

ผลลัพธ์ขั้นต่ำของโปรเจกต์ต้องมี 3 ส่วนต่อไปนี้

ถ้ากลุ่มเลือกต่อวงจรด้วยไอซี 74xx บน breadboard ให้ส่งภาพถ่ายวงจรจริงเพิ่มเติมอย่างน้อย 3 ภาพ ได้แก่ ภาพรวม breadboard, ภาพขา IC ที่ต่อ VCC/GND และภาพ output ขณะทดสอบกรณีสำคัญหนึ่งกรณี

## ตัวเลือกหัวข้อโปรเจกต์

ให้เลือก 1 หัวข้อจาก 4 หัวข้อต่อไปนี้ ถ้าต้องการเสนอหัวข้ออื่นต้องให้ผู้สอนอนุมัติก่อนเริ่มออกแบบ และหัวข้อใหม่ต้องมี FSM หรือ counter ที่ทดสอบสถานะได้ชัดเจน

### หัวข้อ A: นาฬิกาดิจิทัลนับวินาที/นาที

ออกแบบวงจรนับเวลาอย่างน้อยระดับวินาทีและนาที โดยแสดงเลขผ่าน 7-segment

ข้อกำหนดขั้นต่ำ:

- มีตัวนับหลักหน่วยวินาที 0-9
- มีตัวนับหลักสิบวินาที 0-5
- มีตัวนับหลักหน่วยนาที 0-9 หรืออย่างน้อย LED แสดง carry ไปนาที
- มี reset ให้กลับเป็น `00:00`
- มี enable/pause อย่างน้อย 1 สัญญาณ
- แสดงผลบน 7-segment อย่างน้อย 2 หลัก หรือใช้ probe แสดง BCD ถ้ายังต่อ display ไม่ครบ

โครงบล็อกที่แนะนำ:

```text
1 Hz Clock ── Enable ──► MOD-10 seconds unit ── carry ──► MOD-6 seconds ten ── carry ──► MOD-10 minutes unit
     │                         │                              │                              │
   Reset ──────────────────────┴──────────────────────────────┴──────────────────────────────┘
                               │                              │                              │
                            BCD[3:0]                       BCD[3:0]                       BCD[3:0]
                               │                              │                              │
                         BCD to 7-seg                  BCD to 7-seg                  BCD to 7-seg
```

การทดสอบที่ต้องมี:

- reset จากสถานะใด ๆ แล้วได้ `00:00`
- นับจาก `00:08` ถึง `00:12`
- นับจาก `00:58` ถึง `01:02`
- pause ค้างค่าเดิมอย่างน้อย 3 clock
- resume แล้วนับต่อจากค่าที่ค้างไว้

### หัวข้อ B: ตู้ขายของอัตโนมัติแบบ FSM

ออกแบบ FSM สำหรับตู้ขายของที่รับเหรียญและจ่ายสินค้าเมื่อยอดเงินถึงราคาที่กำหนด

ข้อกำหนดขั้นต่ำ:

- ราคาสินค้า 15 บาท หรือ 20 บาท โดยเลือกหนึ่งราคาและระบุในรายงาน
- รับเหรียญ 5 บาทและ 10 บาท ด้วย input 2 เส้น เช่น `N5`, `N10`
- มี output `DISPENSE` เมื่อเงินถึงราคาสินค้า
- มี output `CHANGE5` ถ้าต้องทอน 5 บาท
- มี reset/cancel เพื่อกลับสถานะเริ่มต้น
- ต้องระบุวิธีจัดการกรณี input เหรียญ 5 และ 10 เป็น 1 พร้อมกัน

ตัวอย่าง state สำหรับราคาสินค้า 15 บาท:

```text
S0  = ยอดสะสม 0 บาท
S5  = ยอดสะสม 5 บาท
S10 = ยอดสะสม 10 บาท

N5  คือใส่เหรียญ 5 บาท
N10 คือใส่เหรียญ 10 บาท
```

โครง state diagram:

```text
          N5                 N5 / DISPENSE
     ┌──────────► S5 ─────────────────────► S0
     │             │
     │             │ N10 / DISPENSE
     │             ▼
    S0 ── N10 ──► S10
     ▲             │
     │             │ N10 / DISPENSE + CHANGE5
     └─────────────┘
```

การทดสอบที่ต้องมี:

- 5 + 5 + 5 แล้วจ่ายสินค้า
- 10 + 5 แล้วจ่ายสินค้า
- 5 + 10 แล้วจ่ายสินค้า
- 10 + 10 แล้วจ่ายสินค้าและทอน 5
- cancel หลังใส่ 5 หรือ 10 แล้วกลับ S0
- input ผิดรูปแบบ `N5=1, N10=1` แล้วระบบตอบสนองตามที่ระบุ

### หัวข้อ C: ลูกเต๋าอิเล็กทรอนิกส์

ออกแบบวงจรสุ่มเชิงดิจิทัลอย่างง่ายโดยใช้ counter นับ 1-6 อย่างรวดเร็ว และหยุดแสดงค่าเมื่อปล่อยหรือกดปุ่มตาม spec ที่กลุ่มกำหนด

ข้อกำหนดขั้นต่ำ:

- มี counter วนค่า 1, 2, 3, 4, 5, 6 แล้วกลับ 1
- มีปุ่ม `ROLL` หรือ `HOLD` เพื่อควบคุมการนับและหยุดแสดงผล
- แสดงผลด้วย 7-segment เป็นเลข 1-6
- reset แล้วเริ่มที่ค่า 1
- state 0 และ 7 ต้องไม่ค้างถ้าใช้ counter 3 บิต ต้องถูกบังคับกลับเข้า sequence ที่ถูกต้อง

โครง timing ที่คาดหวัง:

```text
Clock : ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐
        │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │
        └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘
ROLL  : ────────████████████████────────────
Count :    1   1   2   3   4   5   6   6   6
Display:   1   1   2   3   4   5   6   6   6
```

การทดสอบที่ต้องมี:

- reset แล้วแสดง 1
- กด ROLL ต่อเนื่อง 8 clock แล้ว sequence ต้องเป็น 1-2-3-4-5-6-1-2
- ปล่อย ROLL แล้วค่าต้องค้าง
- บังคับ state เป็น 000 หรือ 111 แล้ว clock ถัดไปต้องกลับสู่ 001
- ทดสอบ segment สำหรับเลข 1-6 ครบทุกเลข

### หัวข้อ D: Sequence Detector ตรวจจับลำดับ `1011`

ออกแบบ FSM ตรวจจับบิตอินพุตต่อเนื่อง แล้วให้ output เป็น 1 เมื่อพบลำดับ `1011`

ข้อกำหนดขั้นต่ำ:

- รับ input serial 1 บิตชื่อ `X`
- มี output ชื่อ `Z`
- ตรวจจับแบบ overlapping ได้ เช่น input `1011011` ต้องตรวจพบสองครั้ง
- ใช้ D flip-flop หรือ JK flip-flop อย่างน้อย 2 ตัว
- มี reset กลับสถานะเริ่มต้น
- แสดง present state ด้วย LED หรือ probe

โครง state ที่แนะนำ:

```text
S0 = ยังไม่พบ prefix ที่ตรงกับ 1011
S1 = พบ 1
S2 = พบ 10
S3 = พบ 101
```

การทดสอบที่ต้องมี:

- input `1011` แล้ว `Z=1` ที่บิตสุดท้าย
- input `1011011` แล้ว `Z=1` สองครั้ง
- input `0000` แล้วไม่เกิด output
- input `1111` แล้วไม่เกิด output ยกเว้น state คงที่ตาม diagram
- reset กลางลำดับแล้วเริ่มตรวจจับใหม่

| คำถามของ spec | คำตอบของกลุ่ม |
|---|---|
| ระบบที่เลือกคือหัวข้อใด | |
| input ภายนอกมีชื่ออะไรบ้าง | |
| output ภายนอกมีชื่ออะไรบ้าง | |
| ใช้ clock แบบ manual หรือ generator | |
| reset เป็น active-high หรือ active-low | |
| เมื่อเกิด input ผิดรูปแบบต้องทำอย่างไร | |
| จำนวน state หรือจำนวนบิต counter คือเท่าใด | |
| แสดงผลด้วย LED, probe หรือ 7-segment | |

ตัวอย่าง spec ของ sequence detector:

| รายการ | ค่า |

| input | `X`, `CLK`, `RESET` |
| output | `Z`, `Q1`, `Q0` |

| output style | Mealy output, `Z=1` ใน clock ที่อ่านบิตสุดท้าย |

### ช่วงที่ 2: วาด state diagram หรือ block diagram

ถ้าเลือกหัวข้อที่เป็น FSM เช่น ตู้ขายของหรือ sequence detector ต้องวาด state diagram ที่ระบุทั้ง input และ output บนลูกศร ถ้าเลือกหัวข้อที่เป็น counter/display เช่น นาฬิกาหรือลูกเต๋า ต้องวาด block diagram พร้อมเงื่อนไขการเกิด carry/reset ของแต่ละบล็อก

ตัวอย่าง block diagram สำหรับลูกเต๋า:

```text
                ┌────────────┐
Clock ─────────►│ 3-bit MOD6 │── Q2 Q1 Q0 ──► 1-to-6 decoder ──► 7-segment
ROLL  ─────────►│ counter    │
Reset ─────────►│ reset to 1 │── invalid detector ──► force 001
                └────────────┘
```

ตัวอย่าง block diagram สำหรับตู้ขายของ:

```text
N5 ───────┐
          ▼
       ┌───────────┐      D inputs      ┌──────────────┐      Q state      ┌──────────────┐
N10 ──►│ Next-state├───────────────────►│ State FFs    ├──────────────────►│ Output logic │── DISPENSE
       │ logic     │                    │ Q1,Q0        │                   │              │── CHANGE5
RST ──►└───────────┘                    └──────────────┘                   └──────────────┘
```

### ช่วงที่ 3: สร้าง state table หรือ excitation table

FSM ที่ใช้ D flip-flop ต้องมีตาราง present state, input, next state และ output เพราะสมการของ D flip-flop คือ `D = Q(next)` โดยตรง ส่วน FSM ที่ใช้ JK flip-flop ต้องเพิ่ม excitation ของ `J` และ `K`

ตาราง excitation ของ JK flip-flop:

| Q(t) | Q(t+1) | J | K |
|:---:|:---:|:---:|:---:|
| 0 | 0 | 0 | X |
| 0 | 1 | 1 | X |
| 1 | 0 | X | 1 |
| 1 | 1 | X | 0 |

สำหรับ counter แบบนาฬิกาหรือลูกเต๋า ให้ระบุ transition ของค่าที่แสดง ไม่ใช่เขียนเพียงว่า "นับขึ้น" เช่น ลูกเต๋าต้องมี `001 → 010 → 011 → 100 → 101 → 110 → 001` และต้องระบุการแก้ state `000` กับ `111`

### ช่วงที่ 4: ลดรูปสมการหรือเลือกบล็อกมาตรฐาน

ถ้าออกแบบระดับ gate ให้ลดรูปสมการด้วย Karnaugh map หรือ Boolean algebra แล้วแสดงสมการสุดท้ายในรายงาน ถ้าใช้บล็อกมาตรฐานของ Logisim เช่น Counter, Comparator, Decoder หรือ Register ให้ระบุ parameter ของบล็อก เช่น bit width, max value, trigger และ reset behavior

ตัวอย่าง parameter ที่ต้องบันทึก:

| บล็อก | Parameter ที่ต้องระบุ |
|---|---|
| Counter | bit width, maximum count, enable, clear/load |
| D Flip-Flop | trigger edge, reset/preset, initial value |
| 7-segment decoder | active-high หรือ active-low segment |
| Comparator | จำนวนบิตและเงื่อนไขเทียบเท่า |
| Button | ใช้เป็น pulse, level, หรือ manual clock |

### ช่วงที่ 5: สร้างวงจรใน Logisim Evolution

ขั้นตอน Logisim ต่อไปนี้เป็นขั้นต่ำที่ทุกกลุ่มต้องทำ แม้หัวข้อแต่ละกลุ่มจะแตกต่างกัน

1. เปิด Logisim Evolution แล้วสร้างไฟล์ `lab15-mini-project-groupXX.circ`
2. สร้างวงจรย่อยชื่อ `top`
3. สร้างวงจรย่อยตามหน้าที่ เช่น `fsm_next_state`, `state_register`, `display_decoder`, `test_inputs`
4. วาง input pin เฉพาะสัญญาณภายนอกด้านซ้าย เช่น `CLK`, `RESET`, `X`, `N5`, `N10`, `ROLL`
5. วาง output pin หรือ probe ด้านขวา เช่น `Z`, `DISPENSE`, `CHANGE5`, `SEG_A` ถึง `SEG_G`
6. ถ้าใช้ flip-flop ให้ตั้งชื่อ output เป็น `Q1`, `Q0` หรือชื่อ state bit ที่อ่านง่าย
7. ต่อ `RESET` ไปยัง clear/reset ของ flip-flop หรือ counter ทุกตัวที่เกี่ยวข้อง
8. ใช้ splitter เฉพาะเมื่อจำเป็น และตั้ง bit order ให้ตรงกับรายงาน
9. ใช้ label บนสายสำคัญ เช่น `carry_sec_unit`, `state_S10`, `invalid_000_111`
10. ทดสอบด้วย clock manual ก่อนเปิด clock generator
11. หลังวงจรผ่าน test case หลักแล้ว จึงปรับความถี่ clock generator ให้เหมาะกับการสาธิต
12. บันทึกไฟล์และถ่ายภาพวงจรทั้งภาพรวมและภาพซูมส่วน FSM/counter

ข้อห้ามในการทำ Logisim:

- ห้ามใช้ subcircuit สำเร็จรูปที่ซ่อน logic ทั้งหมดโดยไม่อธิบาย เช่น นำไฟล์วงจรจากภายนอกมาใช้โดยไม่แสดง state table
- ห้ามปล่อย input pin ที่เกี่ยวข้องกับ state ให้ลอยโดยไม่มีค่าเริ่มต้น
- ห้ามใช้ clock คนละแหล่งใน state register หลายตัวโดยไม่อธิบายการ synchronize
- ห้ามใช้ output `Z` หรือ `DISPENSE` ต่อจากปุ่ม input โดยตรงโดยไม่ผ่าน state/counter logic

## ตัวอย่างออกแบบที่ 1: FSM ตรวจจับ `1011`

ตัวอย่างนี้เป็นแนวทางการเขียนรายงานและการแปลง FSM เป็นสมการ ไม่ใช่คำตอบสำเร็จรูปสำหรับทุกหัวข้อ กลุ่มที่เลือกหัวข้อ D ต้องขยายส่วนทดสอบและวงจรให้ครบตาม rubric

กำหนดให้เป็น Mealy FSM เพราะ output `Z` เกิดจาก present state และ input ปัจจุบัน ทำให้ตรวจจับบิตสุดท้ายแล้วสร้าง output ได้ในจังหวะเดียวกับ input นั้น

นิยาม state:

| State | ความหมาย | Encoding Q1Q0 |
|---|---|:---:|
| S0 | ยังไม่พบ prefix | 00 |
| S1 | พบ `1` | 01 |
| S2 | พบ `10` | 10 |
| S3 | พบ `101` | 11 |

State diagram แบบ overlapping:

```text
                 X=1
          ┌─────────────┐
          │             ▼
        ┌────┐ X=1   ┌────┐ X=0   ┌────┐ X=1   ┌────┐
 reset ►│ S0 │──────►│ S1 │──────►│ S2 │──────►│ S3 │
        └────┘       └────┘       └────┘       └────┘
          ▲  ▲          │            │            │
          │  │ X=0      │ X=1        │ X=0        │ X=0
          │  └──────────┘            └────────────┘
          │
          └──────────── X=0 จาก S0

จาก S3:
X=1 ให้ Z=1 และไป S1 เพราะ suffix `1` อาจเป็น prefix ของรอบถัดไป
X=0 ให้ Z=0 และไป S2 เพราะ suffix `10` ตรงกับ prefix สองตัวแรก
```

State table:

| Present State | Q1 | Q0 | X | Next State | Q1+ | Q0+ | Z |
|---|:---:|:---:|:---:|---|:---:|:---:|:---:|
| S0 | 0 | 0 | 0 | S0 | 0 | 0 | 0 |
| S0 | 0 | 0 | 1 | S1 | 0 | 1 | 0 |
| S1 | 0 | 1 | 0 | S2 | 1 | 0 | 0 |
| S1 | 0 | 1 | 1 | S1 | 0 | 1 | 0 |
| S2 | 1 | 0 | 0 | S0 | 0 | 0 | 0 |
| S2 | 1 | 0 | 1 | S3 | 1 | 1 | 0 |
| S3 | 1 | 1 | 0 | S2 | 1 | 0 | 0 |
| S3 | 1 | 1 | 1 | S1 | 0 | 1 | 1 |

หา D input ของ flip-flop:

```text
D1 = Q1+ = 1 ที่แถว 3, 6, 7
D0 = Q0+ = 1 ที่แถว 2, 4, 6, 8
Z  = 1 เฉพาะ S3 และ X=1
```

K-map ของ `D1` โดยใช้แถวเป็น `Q1Q0` และคอลัมน์เป็น `X`:

```text
        X
       0  1
Q1Q0 +------
 00  | 0  0
 01  | 1  0
 11  | 1  0
 10  | 0  1
```

กลุ่มที่จับได้:

```text
กลุ่มแนวตั้งที่ X=0, Q0=1  ให้ Q0·X'
ช่องเดี่ยว Q1Q0=10, X=1 ให้ Q1·Q0'·X

D1 = Q0·X' + Q1·Q0'·X
```

K-map ของ `D0`:

```text
        X
       0  1
Q1Q0 +------
 00  | 0  1
 01  | 0  1
 11  | 0  1
 10  | 0  1
```

จึงได้

```text
D0 = X
```

สมการ output:

```text
Z = Q1·Q0·X
```

วงจรระดับบล็อก:

```text
                 ┌─────────────────────────────┐
X ──────────────►│ next-state logic             │── D1 ──► DFF ── Q1 ──┐
Q1 ─────────────►│ D1 = Q0X' + Q1Q0'X           │                      │
Q0 ─────────────►│ D0 = X                       │── D0 ──► DFF ── Q0 ──┼──► output logic Z=Q1Q0X
CLK ─────────────────────────────────────────────────────► CLK          │
RST ─────────────────────────────────────────────────────► CLR          │
                 └─────────────────────────────┘                       │
```

ตัวอย่าง timing สำหรับ input `1011011`:

```text
Clock edge :  1   2   3   4   5   6   7
X          :  1   0   1   1   0   1   1
State before: S0  S1  S2  S3  S1  S2  S3
State after : S1  S2  S3  S1  S2  S3  S1
Z          :  0   0   0   1   0   0   1
```

ข้อสังเกตสำคัญคือหลังพบ `1011` แล้ว state ไป `S1` ไม่ใช่ `S0` เพราะบิตสุดท้ายเป็น `1` และอาจเป็นจุดเริ่มต้นของ pattern รอบใหม่

## ตัวอย่างออกแบบที่ 2: Counter ลูกเต๋า 1-6

ลูกเต๋าใช้ 3-bit state เพราะต้องแทนเลข 1-6 ได้ แต่ state `000` และ `111` ไม่ใช่เลขหน้าลูกเต๋าที่ใช้แสดงผล กลุ่มต้องออกแบบให้ invalid state ไม่ค้าง

กำหนด state:

| ค่าแสดงผล | Q2Q1Q0 |
|---:|:---:|
| 1 | 001 |
| 2 | 010 |
| 3 | 011 |
| 4 | 100 |
| 5 | 101 |
| 6 | 110 |
| invalid | 000, 111 |

Transition เมื่อ `ROLL=1`:

```text
001 → 010 → 011 → 100 → 101 → 110 → 001
000 → 001
111 → 001
```

เมื่อ `ROLL=0` ให้ state ค้างค่าเดิม

ตาราง next-state แบบย่อ:

| Present Q2Q1Q0 | ROLL | Next Q2+Q1+Q0+ | ความหมาย |
|:---:|:---:|:---:|---|
| 000 | 1 | 001 | แก้ invalid |
| 001 | 1 | 010 | 1 ไป 2 |
| 010 | 1 | 011 | 2 ไป 3 |
| 011 | 1 | 100 | 3 ไป 4 |
| 100 | 1 | 101 | 4 ไป 5 |
| 101 | 1 | 110 | 5 ไป 6 |
| 110 | 1 | 001 | 6 กลับ 1 |
| 111 | 1 | 001 | แก้ invalid |
| any | 0 | present | ค้างค่า |

ใน Logisim สามารถทำได้สองแนวทาง:

1. ใช้ Counter 3 บิตพร้อม comparator ตรวจ `110` แล้ว load `001` ใน clock ถัดไป
2. ใช้ D flip-flop 3 ตัวแล้วสร้าง next-state logic จากตาราง

แนวทางใช้ Counter ง่ายต่อการสาธิต แต่รายงานยังต้องระบุว่าทำไมต้องตรวจค่า 6 และ invalid state

โครงวงจร:

```text
ROLL ───────────────► Enable ของ Counter
RESET ──────────────► Load value 001
Q2Q1Q0 ──► Comparator == 110 ──► next clock load 001
Q2Q1Q0 ──► Comparator == 000 ─┐
Q2Q1Q0 ──► Comparator == 111 ─┴──► invalid_load_001
Q2Q1Q0 ──► 7-segment decoder for 1..6
```

ตารางทดสอบเฉพาะลูกเต๋า:

| TC | ROLL | Present | Expected Next | Expected Display | Actual | ผ่าน/ไม่ผ่าน |
|---:|:---:|:---:|:---:|:---:|---|:---:|
| D1 | 1 | 001 | 010 | 2 | | |
| D2 | 1 | 010 | 011 | 3 | | |
| D3 | 1 | 011 | 100 | 4 | | |
| D4 | 1 | 100 | 101 | 5 | | |
| D5 | 1 | 101 | 110 | 6 | | |
| D6 | 1 | 110 | 001 | 1 | | |
| D7 | 0 | 100 | 100 | 4 | | |
| D8 | 1 | 000 | 001 | 1 | | |
| D9 | 1 | 111 | 001 | 1 | | |

## ตัวอย่างออกแบบที่ 3: นาฬิกา BCD วินาที 00-59

นาฬิกา 00-59 ต้องใช้ counter สองหลักที่มี modulus ต่างกัน หลักหน่วยวินาทีเป็น MOD-10 ส่วนหลักสิบวินาทีเป็น MOD-6 ความผิดพลาดที่พบบ่อยคือให้หลักสิบวินาทีนับถึง 9 ทำให้เกิดค่า 68 หรือ 79 ซึ่งไม่ใช่เวลา

กำหนดสัญญาณ:

| สัญญาณ | ความหมาย |
|---|---|
| `SEC_U[3:0]` | BCD หลักหน่วยวินาที 0-9 |
| `SEC_T[3:0]` | BCD หลักสิบวินาที 0-5 |
| `CARRY_U` | เป็น 1 เมื่อ `SEC_U=9` และ clock ถัดไปต้องเพิ่มหลักสิบ |
| `CARRY_T` | เป็น 1 เมื่อ `SEC_T=5` และ `SEC_U=9` เพื่อส่งไปหลักนาที |
| `PAUSE` | ค้างค่าทุก counter |
| `RESET` | กลับเป็น 00 |

เงื่อนไขการนับ:

```text
ถ้า RESET=1:
    SEC_T = 0, SEC_U = 0
ถ้า PAUSE=1:
    SEC_T และ SEC_U ค้างค่าเดิม
ถ้า SEC_U < 9:
    SEC_U = SEC_U + 1
ถ้า SEC_U = 9:
    SEC_U = 0 และ SEC_T เพิ่ม 1
ถ้า SEC_U = 9 และ SEC_T = 5:
    SEC_U = 0 และ SEC_T = 0 และส่ง carry ไปนาที
```

ตัวอย่าง timing จาก 00:58 ถึง 01:02:

```text
Clock edge :   1     2     3     4     5
Before     : 00:58 00:59 01:00 01:01 01:02
SEC_T      :   5     5     0     0     0
SEC_U      :   8     9     0     1     2
CARRY_U    :   0     1     0     0     0
CARRY_T    :   0     1     0     0     0
```

กรณีทดสอบสำคัญ:

| TC | ค่าเริ่มต้น | Clock | PAUSE | Expected |
|---:|---|:---:|:---:|---|
| C1 | 00:00 | 1 | 0 | 00:01 |
| C2 | 00:08 | 1 | 0 | 00:09 |
| C3 | 00:09 | 1 | 0 | 00:10 |
| C4 | 00:58 | 1 | 0 | 00:59 |
| C5 | 00:59 | 1 | 0 | 01:00 หรือ carry ไปนาที |
| C6 | 00:27 | 3 | 1 | 00:27 |
| C7 | 00:45 | reset | X | 00:00 |

## แนวทางต่อวงจรด้วยไอซี 74xx

การต่อจริงบน breadboard เป็นส่วนเสริม แต่กลุ่มที่เลือกทำจะได้คะแนนในส่วนความสมบูรณ์ของการสาธิต หากต่อจริงให้เลือกวงจรขนาดที่เหมาะกับเวลา เช่น sequence detector หรือ counter ลูกเต๋า ไม่ควรพยายามต่อนาฬิกาหลายหลักถ้ายังไม่มั่นใจเรื่องสายและการจ่ายไฟ

ไอซีที่ใช้ได้ตามหัวข้อ:

| หัวข้อ | ไอซีที่เหมาะสม | หน้าที่ |
|---|---|---|
| Sequence detector | 7474, 7408, 7432, 7404 | D flip-flop, AND, OR, NOT |
| Vending machine FSM | 7474, 7408, 7432, 7486, 7404 | state register และ combinational logic |
| Dice counter | 7490/7493 หรือ 7474, 7447, 7408, 7404 | counter และ BCD/7-segment |
| Digital clock | 7490, 7447, 7408, 7404 | decade counter, decoder, reset logic |

ตัวอย่าง pinout ที่ควรตรวจจาก datasheet ก่อนต่อจริง:

| IC | ขาที่ต้องระวัง | หมายเหตุ |
|---|---|---|
| 7404 | VCC=14, GND=7 | input ที่ไม่ใช้ไม่ควรปล่อยลอย |
| 7408 | VCC=14, GND=7 | AND gate 4 ชุด |
| 7432 | VCC=14, GND=7 | OR gate 4 ชุด |
| 7474 | VCC=14, GND=7 | มี preset/clear แบบ active-low ต้องผูกให้ถูก |
| 7447 | VCC=16, GND=8 | ขา output เหมาะกับ common-anode 7-segment |

ขั้นตอน breadboard สำหรับ sequence detector ด้วย 7474:

1. เสียบ 7474 คร่อมร่องกลาง breadboard
2. ต่อ VCC และ GND ของ 7474 ก่อนต่อสัญญาณอื่น
3. ต่อ `PRE` ของ flip-flop ทั้งสองเป็น logic 1 ถ้าไม่ได้ใช้ preset
4. ต่อ `CLR` ทั้งสองเข้ากับวงจร reset active-low หรือผูกผ่านสวิตช์ให้กดแล้ว clear
5. สร้าง `X'` ด้วย 7404
6. สร้างพจน์ `Q0·X'` และ `Q1·Q0'·X` ด้วย 7408
7. รวมสองพจน์เป็น `D1` ด้วย 7432
8. ต่อ `D0 = X` เข้าขา D ของ flip-flop ตัวที่สอง
9. ต่อ clock เดียวกันเข้ากับ flip-flop ทั้งสอง
10. ต่อ LED ผ่านตัวต้านทาน 220-1k ohm ที่ `Q1`, `Q0` และ `Z`
11. ทดสอบ clock ทีละขอบและบันทึก state ตามตาราง

ข้อควรระวังเฉพาะการใช้ปุ่มจริง:

- ปุ่มกดมีการ bounce จึงอาจทำให้ clock เกิดหลายครั้งจากการกดครั้งเดียว
- ถ้าต้องการสาธิตบน breadboard ให้ใช้ปุ่มสำหรับ input และใช้ clock ช้าจาก function generator หรือวงจร debounced clock
- ขา clear/preset ของ 7474 เป็น active-low ในหลายรุ่น ถ้าผูกผิด flip-flop จะไม่เปลี่ยน state
- 7-segment common-anode และ common-cathode ใช้ logic ตรงข้ามกัน ต้องระบุชนิดในรายงาน

8. ตาราง test case พร้อมผลจริง
9. วิเคราะห์ปัญหาที่พบ เช่น output กระพริบผิดจังหวะ reset ไม่ครอบคลุม หรือ 7-segment active-low
10. สรุปสิ่งที่ได้เรียนรู้และข้อจำกัดของวงจร

รายงานที่ดีต้องทำให้ผู้อ่านสามารถสร้างวงจรตามได้โดยไม่ต้องถามสมาชิกกลุ่มซ้ำว่า "state นี้มาจากอะไร" หรือ "สายนี้ต่อไปไหน"

---

## แนวทางนำเสนอ 5-8 นาที

ลำดับการนำเสนอที่แนะนำ:

1. แนะนำปัญหาและ spec ที่เลือก ใช้เวลาไม่เกิน 1 นาที
2. อธิบาย state diagram หรือ block diagram ใช้เวลา 1-2 นาที
3. แสดง state table หรือ counter transition ที่สำคัญ ใช้เวลา 1 นาที
4. เปิด Logisim และสาธิต test case อย่างน้อย 3 กรณี ใช้เวลา 2-3 นาที
5. อธิบายปัญหาที่พบและวิธีแก้ ใช้เวลา 1 นาที
6. ตอบคำถามผู้สอนและเพื่อนร่วมชั้น

ระหว่างสาธิตให้ใช้ clock ช้าและอ่านค่า state/output ทีละจังหวะ ถ้าวงจรเป็น sequence detector ให้เตรียม input sequence ไว้ล่วงหน้า ถ้าวงจรเป็นนาฬิกาหรือลูกเต๋าให้เตรียมสถานะก่อน boundary เช่น 59 หรือ 6 เพื่อไม่เสียเวลารอนับหลายรอบ

---

## คำถามท้ายโปรเจกต์และข้อสะท้อน

ให้ตอบเป็นรายบุคคลในรายงานหรือภาคผนวก โดยคำตอบต้องอ้างอิงวงจรของกลุ่ม ไม่ตอบแบบทั่วไป

1. State หรือ counter value ใดในวงจรของกลุ่มที่มีความเสี่ยงผิดพลาดมากที่สุด และทดสอบอย่างไรจึงมั่นใจว่าถูกต้อง
2. ถ้า reset เกิดขึ้นกลางลำดับ input หรือกลางการนับ วงจรของกลุ่มกลับสู่สถานะใด และมี output ใดติดค้างหรือไม่
3. สมการหรือบล็อกใดเป็นส่วนที่กำหนด output สำคัญที่สุด เช่น `DISPENSE`, `Z`, `CARRY_T`, หรือ segment display จงอธิบายจาก state/table
4. ถ้าต่อวงจรจริงด้วยไอซี 74xx ปัญหา timing หรือ switch bounce จะกระทบส่วนใดของวงจรมากที่สุด
5. การเลือก state encoding ของกลุ่มช่วยให้สมการง่ายขึ้นหรือทำให้วงจรอ่านง่ายขึ้นอย่างไร
6. Test case ใดตรวจพบข้อผิดพลาดของวงจรได้ดีที่สุด และข้อผิดพลาดนั้นเกิดจากอะไร
7. ถ้าต้องเพิ่มฟังก์ชันอีก 1 อย่าง เช่น pause, cancel, overlapping, ทอนเงินเพิ่ม หรือแสดงผลหลายหลัก ต้องเพิ่ม state หรือบล็อกใด
8. สมาชิกแต่ละคนได้เรียนรู้อะไรจากการเชื่อมต่อระหว่าง state diagram, table และวงจรจริง
