## ทฤษฎีย่อเฉพาะการทดลองนี้

เรจิสเตอร์ (register) คือกลุ่มของ flip-flop ที่ใช้เก็บข้อมูลหลายบิตพร้อมกัน
ในใบงานนี้ใช้ D flip-flop 4 ตัวเพื่อเก็บข้อมูล 4 บิต โดยถือว่าเอาต์พุตเรียงเป็น `Q3 Q2 Q1 Q0`
และให้ `Q0` เป็นบิตขวาสุดหรือบิตน้ำหนักต่ำสุด (least significant bit)

เมื่อ D flip-flop ทำงานที่ขอบขาขึ้นของ clock (rising edge)
ค่าที่อยู่ที่ขา `D` ก่อนขอบ clock จะถูกคัดลอกไปยัง `Q`
หลังขอบ clock ค่าที่ `Q` จะคงอยู่จนกว่าจะมีขอบ clock ถัดไป

```text
ก่อน clock ↑:  D = 1, Q = 0
หลัง clock ↑: D = 1, Q = 1

ก่อน clock ↑:  D = 0, Q = 1
หลัง clock ↑: D = 0, Q = 0
```

เรจิสเตอร์ 4 บิตแบบโหลดขนานต้องมี D flip-flop 4 ตัวใช้ clock เส้นเดียวกัน
ถ้าโหลดค่า `D3 D2 D1 D0 = 1011` แล้วกด clock หนึ่งครั้ง
สถานะใหม่ต้องเป็น `Q3 Q2 Q1 Q0 = 1011`

```text
      D3 ──► [DFF] ──► Q3
      D2 ──► [DFF] ──► Q2
      D1 ──► [DFF] ──► Q1
      D0 ──► [DFF] ──► Q0
                 ▲
CLK ─────────────┴──────── clock ร่วมทุกตัว
```

ชิฟต์เรจิสเตอร์ (shift register) เพิ่มแนวคิดการ "ส่งต่อบิต" จาก flip-flop หนึ่งไปยัง flip-flop ถัดไปทุกครั้งที่ clock ทำงาน
ในใบงานนี้ให้ทดลองการ shift right โดยนิยามว่า serial input เข้า `Q3` แล้วข้อมูลเลื่อนไปทาง `Q0`
ดังนั้นสมการสถานะถัดไปคือ

| บิตสถานะใหม่ | ค่าที่รับเข้าจาก |
|---|---|
| `Q3(next)` | `SI` |
| `Q2(next)` | `Q3(now)` |
| `Q1(next)` | `Q2(now)` |
| `Q0(next)` | `Q1(now)` |

```text
SI ──► DFF3 ──► DFF2 ──► DFF1 ──► DFF0 ──► SO
        Q3       Q2       Q1       Q0
         ▲        ▲        ▲        ▲
         └────────┴────────┴────────┘
                    CLK ร่วม
```

คำย่อที่ใช้ในใบงานนี้มีความหมายเฉพาะด้านการรับส่งข้อมูลดังนี้

| แบบ | ชื่อเต็ม | ทางเข้า | ทางออก | ภาพการใช้งาน |
|---|---|---|---|---|
| SISO | Serial-In Serial-Out | ทีละ 1 บิต | ทีละ 1 บิต | หน่วงข้อมูลอนุกรม 4 clock |
| SIPO | Serial-In Parallel-Out | ทีละ 1 บิต | อ่านพร้อมกัน 4 บิต | รับ serial data แล้วแสดง LED หลายดวง |
| PISO | Parallel-In Serial-Out | โหลด 4 บิตพร้อมกัน | ส่งออกทีละ 1 บิต | แปลงข้อมูลจากสวิตช์หลายตัวเป็น serial stream |
| PIPO | Parallel-In Parallel-Out | โหลด 4 บิตพร้อมกัน | อ่านพร้อมกัน 4 บิต | เก็บข้อมูล 4 บิตแบบ register ธรรมดา |

ตัวอย่างการ shift right แบบ SIPO เมื่อเริ่มต้นที่ `0000` และป้อนลำดับ serial input เป็น `1, 0, 1, 1`

| Clock | `SI` ก่อนกด clock | `Q3 Q2 Q1 Q0` หลัง clock | ความหมาย |
|---:|:---:|:---:|---|
| 0 | - | `0000` | ล้างค่าเริ่มต้น |
| 1 | 1 | `1000` | บิตแรกเข้าที่ `Q3` |
| 2 | 0 | `0100` | `1` เดิมเลื่อนไป `Q2` |
| 3 | 1 | `1010` | บิตใหม่เข้า `Q3`, บิตเก่าเลื่อนต่อ |
| 4 | 1 | `1101` | ได้คำขนานจาก serial stream |

จากตารางนี้ถ้ามอง LED ที่ `Q3 Q2 Q1 Q0`
จะเห็นว่า LED ไม่ได้แสดงเลขนับขึ้นหรือลง แต่แสดงตำแหน่งของบิตที่ถูกเลื่อนตาม clock

สำหรับ PISO การโหลดขนานใช้ multiplexor เลือกระหว่างข้อมูลโหลด (`P3 P2 P1 P0`) กับข้อมูล shift
ถ้าโหลด `P3 P2 P1 P0 = 1011` แล้ว shift right โดยเติม `SI = 0`
serial output ที่อ่านจาก `Q0` จะออกตามลำดับ `1, 1, 0, 1` เมื่ออ่านหลังแต่ละ clock shift

| ขั้น | โหมด | `Q3 Q2 Q1 Q0` หลัง clock | `SO=Q0` |
|---:|---|:---:|:---:|
| 0 | clear | `0000` | 0 |
| 1 | load `1011` | `1011` | 1 |
| 2 | shift, `SI=0` | `0101` | 1 |
| 3 | shift, `SI=0` | `0010` | 0 |
| 4 | shift, `SI=0` | `0001` | 1 |
| 5 | shift, `SI=0` | `0000` | 0 |

ring counter ใช้ shift register แล้วต่อเอาต์พุตท้ายกลับเข้าต้นทาง
ถ้าเริ่มต้นด้วย one-hot state `1000`
การ shift right พร้อม feedback `Q0 -> D3` จะได้สถานะวนรอบ 4 สถานะ

```text
Q0 ───────────────┐
                  ▼
        DFF3 -> DFF2 -> DFF1 -> DFF0
        Q3     Q2     Q1     Q0
```

| Clock | Ring counter `Q3 Q2 Q1 Q0` |
|---:|:---:|
| 0 | `1000` |
| 1 | `0100` |
| 2 | `0010` |
| 3 | `0001` |
| 4 | `1000` |

Johnson counter หรือ twisted-ring counter ใช้ feedback กลับด้าน
ถ้าเริ่มต้นที่ `0000` และต่อ `NOT Q0 -> D3`
จะได้สถานะไม่ซ้ำ 8 สถานะก่อนวนกลับ

```text
Q0 ──► NOT ───────┐
                  ▼
        DFF3 -> DFF2 -> DFF1 -> DFF0
        Q3     Q2     Q1     Q0
```

| Clock | Johnson counter `Q3 Q2 Q1 Q0` |
|---:|:---:|
| 0 | `0000` |
| 1 | `1000` |
| 2 | `1100` |
| 3 | `1110` |
| 4 | `1111` |
| 5 | `0111` |
| 6 | `0011` |
| 7 | `0001` |
| 8 | `0000` |

## ส่วนที่ 2: สร้างเรจิสเตอร์ 4 บิตแบบโหลดขนานจาก D flip-flop (30 นาที)

วงจรนี้เป็นฐานของ PIPO และเป็นจุดเริ่มต้นก่อนเพิ่มวงจร shift
ให้สร้าง register ที่โหลดข้อมูล 4 บิตพร้อมกันเมื่อ `LOAD = 1`
และคงค่าเดิมเมื่อ `LOAD = 0`

### สมการอินพุตของ D flip-flop

ใช้ 2-to-1 multiplexer หน้า D flip-flop แต่ละตัว

| บิต | ถ้า `LOAD=1` | ถ้า `LOAD=0` | สมการแบบ MUX |
|---|---|---|---|
| `D3` | `P3` | `Q3` | `D3 = LOAD ? P3 : Q3` |
| `D2` | `P2` | `Q2` | `D2 = LOAD ? P2 : Q2` |
| `D1` | `P1` | `Q1` | `D1 = LOAD ? P1 : Q1` |
| `D0` | `P0` | `Q0` | `D0 = LOAD ? P0 : Q0` |

```text
             ┌─────────────── feedback ───────────────┐
             │                                         │
P3 ──► MUX ──┴──► DFF3 ──► Q3                         │
P2 ──► MUX ─────► DFF2 ──► Q2                         │
P1 ──► MUX ─────► DFF1 ──► Q1                         │
P0 ──► MUX ─────► DFF0 ──► Q0                         │
       ▲             ▲                                 │
       │             │                                 │
     LOAD          CLK                                 │
```

### ขั้นตอนใน Logisim Evolution

1. เข้า subcircuit `REG4_LOAD`
2. วาง input pin 4 ตัว ตั้งชื่อ `P3`, `P2`, `P1`, `P0`
3. วาง input pin 1 ตัว ตั้งชื่อ `LOAD`
4. วาง input pin 1 ตัว ตั้งชื่อ `CLK`
5. วาง D flip-flop 4 ตัวจากหมวด **Memory -> D Flip-Flop**
6. ตั้งค่า trigger ของ D flip-flop เป็น rising edge ถ้าโปรแกรมให้เลือกได้
7. วาง multiplexer 4 ตัวจากหมวด **Plexers -> Multiplexer**
8. ตั้งค่า multiplexer แต่ละตัวเป็น `Select Bits = 1` และ `Data Bits = 1`
9. ต่อ `P3` เข้า input 1 ของ MUX หน้า DFF3
10. ต่อ `Q3` feedback เข้า input 0 ของ MUX หน้า DFF3
11. ต่อ output ของ MUX หน้า DFF3 เข้า `D` ของ DFF3
12. ทำซ้ำกับ `P2/Q2`, `P1/Q1`, `P0/Q0`
13. ต่อ `LOAD` เข้าขา select ของ MUX ทั้ง 4 ตัว
14. ต่อ `CLK` เข้าขา clock ของ D flip-flop ทั้ง 4 ตัว
15. ต่อ output `Q3 Q2 Q1 Q0` ออกไปยัง LED และ probe
16. เพิ่ม label ที่สาย feedback ทุกเส้นเพื่อให้เห็นว่าเมื่อไม่โหลด register ต้องจำค่าเดิม

### ตัวอย่างทดสอบที่ต้องทำ

เริ่มต้นด้วยการ clear หรือตั้งค่า register ให้เป็น `0000`
จากนั้นทดลองโหลดและ hold ตามตาราง

| ขั้น | `LOAD` | `P3 P2 P1 P0` | การกด clock | Expected `Q3 Q2 Q1 Q0` | Actual |
|---:|:---:|:---:|---|:---:|:---:|
| 0 | - | - | clear | `0000` | |
| 1 | 1 | `1011` | 1 ครั้ง | `1011` | |
| 2 | 0 | `0100` | 1 ครั้ง | `1011` | |
| 3 | 1 | `0100` | 1 ครั้ง | `0100` | |
| 4 | 0 | `1111` | 2 ครั้ง | `0100` | |

จุดที่ต้องสังเกตคือขั้นที่ 2 และ 4
แม้เปลี่ยน input parallel แล้ว output ต้องไม่เปลี่ยนถ้า `LOAD=0`
ถ้า output เปลี่ยน แสดงว่า feedback หรือ select ของ MUX ต่อผิด

## ส่วนที่ 3: สร้าง SISO 4 บิตและตรวจการหน่วงข้อมูล 4 clock (25 นาที)

SISO ใช้ดูการเคลื่อนที่ของบิตอนุกรมอย่างชัดเจนที่สุด
ในใบงานนี้ให้อ่าน serial output จาก `Q0`

### ขั้นตอนใน Logisim Evolution

1. เข้า subcircuit `SISO4`
2. วาง input pin `SI`
3. วาง input pin `CLK`
4. วาง D flip-flop 4 ตัวเรียงจากซ้ายไปขวาเป็น DFF3, DFF2, DFF1, DFF0
5. ต่อ `SI` เข้า `D` ของ DFF3
6. ต่อ `Q3` เข้า `D` ของ DFF2
7. ต่อ `Q2` เข้า `D` ของ DFF1
8. ต่อ `Q1` เข้า `D` ของ DFF0
9. ต่อ `Q0` ไปยัง output pin ชื่อ `SO`
10. ต่อ `Q3 Q2 Q1 Q0` ไปยัง LED 4 ดวงเพื่อดูค่าภายในด้วย
11. ต่อ `CLK` เข้าขา clock ของ D flip-flop ทั้งหมด
12. ก่อนทดลองให้ reset หรือสร้างสถานะเริ่มต้น `0000`

### ลำดับข้อมูลที่ใช้ป้อน

ให้ป้อน `SI` ตามลำดับนี้ก่อนกด clock แต่ละครั้ง

```text
Clock ที่จะกด:  1 2 3 4 5 6 7 8
SI ก่อน clock: 1 0 1 1 0 0 1 0
```

### Timing diagram ที่คาดหวัง

```text

Q3  :  1    0    1    1    0    0    1    0
Q2  :  0    1    0    1    1    0    0    1
Q1  :  0    0    1    0    1    1    0    0
Q0  :  0    0    0    1    0    1    1    0
SO  :  0    0    0    1    0    1    1    0
```

คำตอบควรสัมพันธ์กับจำนวน D flip-flop ที่ต่ออนุกรมกัน

## ส่วนที่ 4: สร้าง SIPO 4 บิตและอ่านข้อมูลจาก LED พร้อมกัน (25 นาที)

SIPO ใช้วงจรภายในเหมือน SISO แต่สนใจ parallel output ทุกบิตพร้อมกัน
สถานการณ์ใช้งานคือรับข้อมูลทีละบิตจากสายเดียว แล้วแสดงหรือประมวลผลเป็นคำ 4 บิต

### ขั้นตอนใน Logisim Evolution

1. คัดลอกโครงจาก `SISO4` ไปเป็น `SIPO4`
2. คง input `SI` และ `CLK`
3. เพิ่ม output pin 4 ตัวชื่อ `P_OUT3`, `P_OUT2`, `P_OUT1`, `P_OUT0`
4. ต่อ `Q3` ไปยัง `P_OUT3`
5. ต่อ `Q2` ไปยัง `P_OUT2`
6. ต่อ `Q1` ไปยัง `P_OUT1`
7. ต่อ `Q0` ไปยัง `P_OUT0`
8. ต่อ LED 4 ดวงกับ `Q3 Q2 Q1 Q0`
9. ตั้งสีหรือวางตำแหน่ง LED ให้ `Q3` อยู่ซ้ายสุดและ `Q0` อยู่ขวาสุด
10. ทดลองด้วย manual clock เท่านั้นในรอบแรก เพื่อไม่ให้พลาดการบันทึกสถานะ

### ตัวอย่างรับ serial word `1011`

| Clock | `SI` | `Q3 Q2 Q1 Q0` | คำอธิบาย |
|---:|:---:|:---:|---|
| 0 | - | `0000` | เริ่มต้น |

## ส่วนที่ 5: สร้าง PISO 4 บิตด้วย MUX หน้า D flip-flop (30 นาที)

PISO ต้องมีสองโหมดคือ load และ shift
เมื่อ `LOAD=1` ให้โหลดข้อมูลขนาน `P3 P2 P1 P0`
เมื่อ `LOAD=0` ให้ shift right และเติม `SI` ที่ `Q3`

### สมการ PISO แบบ shift right

| บิต | `LOAD=1` | `LOAD=0` |
|---|---|---|
| `D3` | `P3` | `SI` |
| `D2` | `P2` | `Q3` |
| `D1` | `P1` | `Q2` |
| `D0` | `P0` | `Q1` |

```text
LOAD=1: P3 P2 P1 P0 ──► DFF3 DFF2 DFF1 DFF0
LOAD=0: SI ───────────► DFF3 -> DFF2 -> DFF1 -> DFF0 -> SO
```

### ขั้นตอนใน Logisim Evolution

1. เข้า subcircuit `PISO4`
2. วาง input pin `P3`, `P2`, `P1`, `P0`
3. วาง input pin `SI`
4. วาง input pin `LOAD`
5. วาง input pin `CLK`
6. วาง D flip-flop 4 ตัว
7. วาง MUX 4 ตัว โดยแต่ละตัวมี select 1 บิต
8. ต่อ `LOAD` เข้าขา select ของ MUX ทั้งหมด
9. สำหรับ MUX หน้า DFF3 ให้ต่อ input 1 เป็น `P3` และ input 0 เป็น `SI`
10. สำหรับ MUX หน้า DFF2 ให้ต่อ input 1 เป็น `P2` และ input 0 เป็น `Q3`
11. สำหรับ MUX หน้า DFF1 ให้ต่อ input 1 เป็น `P1` และ input 0 เป็น `Q2`
12. สำหรับ MUX หน้า DFF0 ให้ต่อ input 1 เป็น `P0` และ input 0 เป็น `Q1`
13. ต่อ output MUX แต่ละตัวเข้าขา `D` ของ D flip-flop ตำแหน่งเดียวกัน
14. ต่อ `CLK` เข้าขา clock ของ flip-flop ทั้งหมด
15. ต่อ `Q0` เป็น output pin ชื่อ `SO`
16. ต่อ `Q3 Q2 Q1 Q0` ไปยัง LED เพื่อดูสถานะภายในก่อน serial output จะออกครบ

### การทดลอง PISO ด้วยข้อมูล `1011`

1. ตั้ง `P3 P2 P1 P0 = 1011`
2. ตั้ง `LOAD = 1`
3. กด clock 1 ครั้งเพื่อโหลดข้อมูล
4. บันทึกว่า `Q3 Q2 Q1 Q0 = 1011`
5. ตั้ง `LOAD = 0`
6. ตั้ง `SI = 0` เพื่อเติมศูนย์เข้าเรจิสเตอร์ขณะ shift
7. กด clock ทีละครั้งและอ่าน `SO`

| ขั้น | โหมด | `P3 P2 P1 P0` | `SI` | Expected `Q3 Q2 Q1 Q0` | Expected `SO=Q0` | Actual |
|---:|---|:---:|:---:|:---:|:---:|:---:|
| 0 | clear | - | - | `0000` | 0 | |
| 1 | load | `1011` | X | `1011` | 1 | |
| 2 | shift | X | 0 | `0101` | 1 | |
| 3 | shift | X | 0 | `0010` | 0 | |
| 4 | shift | X | 0 | `0001` | 1 | |
| 5 | shift | X | 0 | `0000` | 0 | |

ข้อผิดพลาดที่พบบ่อยคืออ่าน `SO` ก่อนกด clock หรืออ่านจาก `Q3` แทน `Q0`
ให้ระบุในรายงานว่ากลุ่มของนักศึกษาอ่าน serial output จากบิตใดและเพราะเหตุใด

## ส่วนที่ 6: ทดลอง PIPO และเปรียบเทียบกับ REG4_LOAD (15 นาที)

## ส่วนที่ 7: สร้าง ring counter จาก shift register (25 นาที)

ring counter ต้องมีสถานะเริ่มต้นที่มีบิต 1 เพียงตัวเดียว
ในการทดลองนี้ใช้ `1000`
หากเริ่มที่ `0000` วงจรจะค้างที่ `0000` ตลอด เพราะ feedback จาก `Q0` ก็เป็น 0

### ขั้นตอนใน Logisim Evolution

1. เข้า subcircuit `RING4`
2. วาง D flip-flop 4 ตัวแบบเดียวกับ `SISO4`
3. ต่อ `Q3 -> D2`
4. ต่อ `Q2 -> D1`
5. ต่อ `Q1 -> D0`
6. ต่อ `Q0 -> D3` เพื่อ feedback กลับต้นทาง
7. ต่อ `CLK` เข้าทุก flip-flop
8. เพิ่มวิธีกำหนดค่าเริ่มต้นเป็น `1000`
9. วิธีที่ 1: ใช้ preset/clear ของ D flip-flop ถ้าใช้ได้ใน Logisim
10. วิธีที่ 2: เพิ่ม MUX หน้า D flip-flop เหมือน PIPO แล้วใช้ `LOAD_INIT=1` โหลด `1000` หนึ่ง clock
11. หลังตั้งค่าเริ่มต้นแล้วให้ `LOAD_INIT=0`
12. กด clock ทีละครั้งและบันทึกสถานะ

### ตารางบันทึกลำดับ ring counter

|---:|:---:|:---:|---|:---:|
| 0 | `1000` | | `Q3` | |
| 1 | `0100` | | `Q2` | |
| 2 | `0010` | | `Q1` | |
| 3 | `0001` | | `Q0` | |
| 4 | `1000` | | `Q3` | |
| 5 | `0100` | | `Q2` | |
| 6 | `0010` | | `Q1` | |
| 7 | `0001` | | `Q0` | |
| 8 | `1000` | | `Q3` | |

### ประเด็นวิเคราะห์เฉพาะ ring counter

- จำนวน flip-flop 4 ตัวให้จำนวนสถานะที่ใช้งานจริง 4 สถานะเมื่อเป็น one-hot ring
- มีสถานะอื่นที่เป็น invalid เช่น `0000`, `1100`, `1010`
- ถ้าไม่มีวงจรแก้ invalid state วงจรอาจไม่กลับเข้าสู่ลำดับ one-hot เอง
- เอาต์พุตแต่ละบิตมี duty cycle 1/4 ถ้าเริ่มจาก one-hot state

## ส่วนที่ 8: สร้าง Johnson counter และเปรียบเทียบกับ ring counter (25 นาที)

Johnson counter ใช้ D flip-flop จำนวนเท่ากันกับ ring counter
แต่ใช้ feedback แบบกลับค่า `NOT Q0 -> D3`
ทำให้ได้จำนวนสถานะเป็นสองเท่าของจำนวน flip-flop

### ขั้นตอนใน Logisim Evolution

1. เข้า subcircuit `JOHNSON4`
2. คัดลอกวงจรจาก `RING4`
3. ลบสาย feedback `Q0 -> D3`
4. วาง NOT gate 1 ตัวจากหมวด **Gates -> NOT Gate**
5. ต่อ `Q0` เข้า NOT gate
6. ต่อ output ของ NOT gate เข้า `D3`
7. ตั้งสถานะเริ่มต้นเป็น `0000`
8. กด clock ทีละครั้งและบันทึกสถานะ 8 clock
9. เปรียบเทียบจำนวนสถานะก่อนวนซ้ำกับ ring counter

### ตารางบันทึกลำดับ Johnson counter

| 0 | `0000` | | 0 | |
| 1 | `1000` | | 1 | |
| 2 | `1100` | | 2 | |
| 3 | `1110` | | 3 | |
| 4 | `1111` | | 4 | |
| 5 | `0111` | | 3 | |
| 6 | `0011` | | 2 | |
| 7 | `0001` | | 1 | |
| 8 | `0000` | | 0 | |

### ตารางเปรียบเทียบ ring counter กับ Johnson counter

| รายการ | Ring counter 4 บิต | Johnson counter 4 บิต |
|---|---|---|
| feedback | `Q0 -> D3` | `NOT Q0 -> D3` |

## ส่วนที่ 9: ตัวเลือกเสริม ต่อไอซี 74194/74195 universal shift register

5. ต่อ serial input เข้าขา 2

## คำถามท้ายใบงาน

1. ใน SIPO ขนาด 4 บิต ถ้าป้อน serial input เป็น `1, 0, 1, 1` แล้วอ่าน parallel output หลัง clock ที่ 4 เหตุใดจึงได้ `1101` ไม่ใช่ `1011`
2. PISO ใช้แปลง parallel-to-serial อย่างไร จงอธิบายโดยอ้างอิงขั้น load และขั้น shift ของข้อมูล `1011`
3. ถ้าต้องการให้ SISO หน่วงข้อมูลอนุกรม 8 clock ต้องปรับวงจรอย่างไร และต้องใช้ D flip-flop กี่ตัว
4. ring counter 4 บิตกับ Johnson counter 4 บิตต่างกันด้าน feedback และจำนวนสถานะอย่างไร
5. เพราะเหตุใด ring counter ที่เริ่มจาก `0000` จึงไม่สามารถสร้างลำดับ one-hot ได้เอง
6. อธิบายว่า output ของ ring counter สามารถใช้หารความถี่ clock ได้อย่างไร และ duty cycle ของแต่ละ output ประมาณเท่าใด
7. ถ้าวงจร PISO อ่าน serial output จาก `Q3` แทน `Q0` ลำดับบิตที่ออกจะต่างจากตารางอย่างไร
8. เมื่อใช้ไอซี 74194 จริง เหตุใดต้องตรวจว่า `CLR/MR` เป็น active-low และต้องไม่ปล่อยขาควบคุม `S0`, `S1`, `CLR` ลอย

## เกณฑ์ประเมินใบงาน

| เกณฑ์ | คะแนน | รายละเอียด |
|---|:---:|---|
| เรจิสเตอร์ 4 บิตแบบโหลดขนาน | 15 | โหลด `P3 P2 P1 P0` ได้ถูกต้องและ hold ค่าเดิมเมื่อ `LOAD=0` |
| SISO/SIPO | 20 | ป้อน serial input ตามลำดับและแสดงสถานะ `Q3 Q2 Q1 Q0` ถูกทุก clock |
| PISO/PIPO | 20 | โหลดขนานและส่งออกอนุกรม/อ่านขนานถูกต้อง พร้อมอธิบายทิศทางบิต |
| Ring และ Johnson counter | 20 | ลำดับสถานะถูกต้องครบตามตาราง และอธิบาย feedback ได้ |
| ตารางบันทึกผลและหลักฐาน | 15 | ตาราง A, B, C กรอกครบ มีภาพวงจรหรือไฟล์ `.circ` ที่ตรวจซ้ำได้ |
| คำถามท้ายใบงาน | 10 | ตอบเฉพาะเจาะจงกับวงจรที่สร้าง ไม่ตอบแบบท่องจำกว้าง ๆ |

## ข้อควรระวังด้าน timing และวงจรจริง

- เปลี่ยนค่า `SI`, `LOAD`, `P3 P2 P1 P0` ให้เรียบร้อยก่อนกด clock
- อย่าเปลี่ยน input พร้อมกับขอบ clock เพราะอาจละเมิด setup time หรือ hold time
- ใน Logisim ให้ใช้ manual clock ตอนบันทึกตาราง เพื่อเห็นสถานะทีละ clock
- ในวงจรจริงต้องต่อ VCC/GND ของไอซีทุกตัวก่อนต่อสัญญาณอื่น
- ขา control ของไอซี TTL ห้ามปล่อยลอย ให้ต่อ HIGH หรือ LOW ผ่านสวิตช์/ตัวต้านทานตามเหมาะสม
- LED ทุกดวงบน breadboard ต้องมีตัวต้านทานจำกัดกระแส
- ถ้าใช้ 74LS ควรใช้แหล่งจ่าย 5 V และตรวจระดับ logic จาก datasheet
- ถ้า output ใน Logisim ไม่ตรงตาราง ให้ตรวจทิศทาง shift ก่อนตรวจอย่างอื่น เพราะมักสลับ `Q3` กับ `Q0`

## การส่งงาน

ส่งหลักฐานต่อไปนี้ผ่านช่องทางที่ประกาศในรายวิชา

1. ไฟล์ Logisim Evolution `lab14-shift-registers.circ`
2. ภาพหน้าจอวงจร `REG4_LOAD`, `SISO4`, `SIPO4`, `PISO4`, `RING4`, `JOHNSON4`
3. ตารางบันทึกผล A, B และ C ที่กรอกครบ
4. คำตอบคำถามท้ายใบงาน 8 ข้อ
5. ถ้าทดลองไอซีจริง ให้แนบภาพ breadboard และระบุเบอร์ไอซีที่ใช้

### Checklist ก่อนส่ง

- [ ] ชื่อสัญญาณใน Logisim ตรงกับใบงาน เช่น `SI`, `SO`, `LOAD`, `CLK`, `Q3..Q0`
- [ ] ใช้ manual clock เพื่อบันทึกสถานะครบทุก clock
- [ ] ไม่สลับลำดับบิตระหว่าง `Q3 Q2 Q1 Q0` กับ `Q0 Q1 Q2 Q3`
- [ ] ตาราง SISO/SIPO มีสถานะเริ่มต้น clock 0
- [ ] ตาราง PISO ระบุลำดับ `SO` หลังโหลดข้อมูลแล้ว
- [ ] ตาราง ring และ Johnson มีสถานะครบถึง clock 8
- [ ] คำตอบท้ายใบงานอ้างอิงผลจากวงจรที่สร้างเอง
