# 🔬 Lab 12: FSM Design — ออกแบบวงจรไฟจราจร

## วัตถุประสงค์

- ออกแบบ FSM ไฟจราจร 3 สถานะด้วย Moore Machine
- ทำครบทุกขั้นตอน: State Diagram → State Table → Excitation Table → K-Map → สมการ
- เตรียมสมการสำหรับต่อวงจรจริงใน Lab 13

## โจทย์

ออกแบบวงจรควบคุมไฟจราจรอย่างง่าย โดยมี **3 สถานะ** วนซ้ำ:

```
GREEN → YELLOW → RED → GREEN → ...
```

**Output:** LED 3 ดวง (R, Y, G) — ติดทีละ 1 ดวงตามสถานะ

**ใช้ D Flip-Flop** (เพราะง่ายที่สุด: D = Q⁺)

---

## ขั้นตอนที่ 1: State Diagram (10 นาที)

### วาด State Diagram (Moore Machine)

```
    ┌────────────┐
    │            ↓
┌────────┐  ┌────────┐  ┌────────┐
│  S0    │  │  S1    │  │  S2    │
│ GREEN  │→ │ YELLOW │→ │  RED   │
│ G=1    │  │ Y=1    │  │ R=1    │
└────────┘  └────────┘  └────────┘
    ↑                        │
    └────────────────────────┘
```

- ไม่มี Input (เปลี่ยน state ทุก Clock pulse)
- Output อยู่ใน State (Moore)

---

## ขั้นตอนที่ 1.1: เทคนิคการลดทอนสถานะ (State Reduction) (10 นาที) ⭐

ในกรณีที่ FSM มีสถานะจำนวนมาก เราสามารถลดทอนสถานะที่ซ้ำซ้อนได้โดยใช้ **ตารางนัยพัวพัน (Implication Table)**

**โจทย์:** หากสถานะ S0 และ S1 ให้ Output เหมือนกัน และเมื่อได้รับ Input เดียวกันจะเปลี่ยนไปที่สถานะเดียวกันเสมอ เราสามารถทำอะไรได้?
   - [ ] ลบสถานะหนึ่งทิ้ง
   - [ ] ยุบรวม (Collapse) ทั้งสองสถานะเข้าด้วยกันเพื่อลดจำนวน Flip-Flop

## ขั้นตอนที่ 2: State Assignment (5 นาที)

ใช้ 2 FF (Q₁, Q₀) เพราะมี 3 states → $\lceil \log_2 3 \rceil = 2$ FF

| State | Q₁ | Q₀ | G | Y | R |
|:---:|:---:|:---:|:---:|:---:|:---:|
| S0 (GREEN) | 0 | 0 | 1 | 0 | 0 |
| S1 (YELLOW) | 0 | 1 | 0 | 1 | 0 |
| S2 (RED) | 1 | 0 | 0 | 0 | 1 |
| (unused) | 1 | 1 | X | X | X |
