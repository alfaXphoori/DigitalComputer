## ทฤษฎีย่อเฉพาะเคาน์เตอร์

เคาน์เตอร์ (counter) คือวงจรลำดับที่เปลี่ยนสถานะตามจังหวะ clock โดยสถานะของ flip-flop หลายตัวรวมกันแทนเลขฐานสอง เช่น `Q2Q1Q0 = 101` หมายถึงค่าทศนิยม `5`

ในการทดลองนี้ให้กำหนด `Q0` เป็นบิตต่ำสุด (least significant bit) และ `Q2` เป็นบิตสูงสุดของเคาน์เตอร์ 3 บิต

| สัญญาณ | ความหมายในแลปนี้ | ข้อสังเกตระหว่างทดลอง |
|---|---|---|
| `CLK` | clock หลักจากปุ่มหรือ clock generator | ใช้ manual tick ก่อนทุกครั้งเพื่อมองลำดับสถานะ |
| `Q0` | บิตน้ำหนัก 1 | toggle ทุก clock ใน binary up counter |
| `Q1` | บิตน้ำหนัก 2 | toggle เมื่อ `Q0` เปลี่ยนจาก 1 ไป 0 |
| `Q2` | บิตน้ำหนัก 4 | toggle เมื่อ `Q1Q0` เปลี่ยนจาก 11 ไป 00 |
| `CLR` หรือ `Reset` | สั่งล้างค่า flip-flop เป็น 0 | ระวัง active-high หรือ active-low ตามอุปกรณ์ที่เลือก |

เคาน์เตอร์แบบอะซิงโครนัส หรือ ripple counter ใช้ output ของ flip-flop ตัวก่อนหน้าเป็น clock ของตัวถัดไป ดังนั้น flip-flop แต่ละตัวไม่ได้เปลี่ยนพร้อมกันจริง

```text
CLK ─────► FF0 ─Q0────► FF1 ─Q1────► FF2 ─Q2────► output
          T=1          T=1          T=1
```

เมื่อเคาน์เตอร์ ripple เปลี่ยนจาก `011` เป็น `100` ตามเลขฐานสองจริง บิต `Q0` ต้องเปลี่ยนก่อน จากนั้น `Q1` จึงเปลี่ยน และสุดท้าย `Q2` จึงเปลี่ยน ช่วงสั้น ๆ ระหว่างทางอาจเห็นสถานะชั่วคราว เช่น `010` หรือ `000` ขึ้นกับทิศทางขอบสัญญาณและ delay ของ flip-flop

เคาน์เตอร์แบบซิงโครนัส (synchronous counter) ให้ flip-flop ทุกตัวรับ clock เส้นเดียวกัน แล้วใช้วงจรตรรกะเลือกว่าบิตใดควร toggle ใน clock ถัดไป

```text
                 ┌──────── toggle logic ────────┐
CLK ─────────────┼────► FF0 ───── Q0 ────────────┤
                 ├────► FF1 ───── Q1 ────────────┤
                 └────► FF2 ───── Q2 ────────────┘
```

สำหรับ binary up counter 3 บิตที่ใช้ T flip-flop สมการ toggle คือ

```text
T0 = 1
T1 = Q0
T2 = Q1 · Q0
```

ความหมายของสมการคือ `Q0` toggle ทุก clock, `Q1` toggle เมื่อบิตล่างกว่า (`Q0`) เป็น 1, และ `Q2` toggle เมื่อบิตล่างกว่าทั้งสองบิต (`Q1Q0`) เป็น 11

ลำดับนับมาตรฐานของเคาน์เตอร์ 3 บิตคือ

| Clock count | Q2 | Q1 | Q0 | Binary | Decimal |
|---:|:---:|:---:|:---:|:---:|---:|
| 0 | 0 | 0 | 0 | 000 | 0 |
| 1 | 0 | 0 | 1 | 001 | 1 |
| 2 | 0 | 1 | 0 | 010 | 2 |
| 3 | 0 | 1 | 1 | 011 | 3 |
| 4 | 1 | 0 | 0 | 100 | 4 |
| 5 | 1 | 0 | 1 | 101 | 5 |
| 6 | 1 | 1 | 0 | 110 | 6 |
| 7 | 1 | 1 | 1 | 111 | 7 |
| 8 | 0 | 0 | 0 | 000 | 0 |

## ตัวอย่างที่ 1: อ่านลำดับ ripple counter 3 บิต

ต้องการสร้างเคาน์เตอร์นับขึ้น 0 ถึง 7 ด้วย T flip-flop 3 ตัว

กำหนดให้ flip-flop ทำงานที่ขอบตก (falling edge) เพื่อให้ต่อ `Q0` ไปเป็น clock ของ FF1 และ `Q1` ไปเป็น clock ของ FF2 ได้ตรงไปตรงมา

```text
Manual CLK ──► CLK(FF0)
Q0        ──► CLK(FF1)
Q1        ──► CLK(FF2)
T0=T1=T2=1
Reset ทุกตัวเริ่มที่ 0
```

เมื่อ clock หลักเดิน 1 ครั้ง FF0 toggle ก่อน ถ้า `Q0` เปลี่ยนจาก 1 เป็น 0 จะสร้าง falling edge ให้ FF1 toggle ต่อ และถ้า `Q1` เปลี่ยนจาก 1 เป็น 0 จะสร้าง falling edge ให้ FF2 toggle ต่อ

ตาราง expected state เมื่อนับจาก `000` คือ

| Tick | ก่อน tick | หลัง ripple เสถียร | เหตุการณ์สำคัญ |
|---:|:---:|:---:|---|
| 1 | 000 | 001 | FF0 toggle |
| 2 | 001 | 010 | FF0 toggle 1→0 แล้ว FF1 toggle |
| 3 | 010 | 011 | FF0 toggle |
| 4 | 011 | 100 | FF0, FF1, FF2 toggle ต่อกันเป็นทอด |
| 5 | 100 | 101 | FF0 toggle |
| 6 | 101 | 110 | FF0 แล้ว FF1 toggle |
| 7 | 110 | 111 | FF0 toggle |
| 8 | 111 | 000 | FF0, FF1, FF2 toggle ต่อกันเป็นทอด |

ช่วงเปลี่ยน `011 -> 100` เป็นจุดที่ควรสังเกต delay มากที่สุด เพราะต้องเปลี่ยนครบทั้ง 3 flip-flop

```text
เวลาเพิ่มไปทางขวา

CLK:  ___|‾‾‾|___|‾‾‾|___|‾‾‾|___|‾‾‾|___
Q0 :  ___|‾‾‾‾‾‾|______|‾‾‾‾‾‾|______
Q1 :  _________|‾‾‾‾‾‾‾‾‾‾‾|_________
Q2 :  ______________________|‾‾‾‾‾‾‾‾
           001     010     011   100
                               ^ ripple หลายชั้น
```

## ตัวอย่างที่ 2: หา toggle logic ของ synchronous counter 3 บิต

ต้องการให้ flip-flop ทุกตัวรับ `CLK` เดียวกัน และนับขึ้นแบบ binary

เริ่มจาก state table ของ present state และ next state

| Present Q2Q1Q0 | Next Q2Q1Q0 | Q0 toggle? | Q1 toggle? | Q2 toggle? |
|:---:|:---:|:---:|:---:|:---:|
| 000 | 001 | 1 | 0 | 0 |
| 001 | 010 | 1 | 1 | 0 |
| 010 | 011 | 1 | 0 | 0 |
| 011 | 100 | 1 | 1 | 1 |
| 100 | 101 | 1 | 0 | 0 |
| 101 | 110 | 1 | 1 | 0 |
| 110 | 111 | 1 | 0 | 0 |
| 111 | 000 | 1 | 1 | 1 |

จากตารางจะได้

```text
T0 = 1
T1 = Q0
T2 = Q1Q0
```

วงจรที่ต้องสร้างใน Logisim คือ

```text
                         ┌───────┐
CLK ────────────────────►│ T FF0 │── Q0 ───────────────┐
T0=1 ───────────────────►│       │                      │
                         └───────┘                      │
                                                        ▼
                         ┌───────┐                  ┌─────┐
CLK ────────────────────►│ T FF1 │── Q1 ───────────►│ AND │── T2
T1=Q0 ──────────────────►│       │                  └─────┘
                         └───────┘                     ▲
                                                       Q0
                         ┌───────┐
CLK ────────────────────►│ T FF2 │── Q2
T2=Q1Q0 ────────────────►│       │
                         └───────┘
```

จุดสังเกตสำคัญคือ `Q0`, `Q1`, `Q2` จะเปลี่ยนหลังขอบ clock เดียวกันเกือบพร้อมกัน ไม่มีการนำ `Q0` ไปเป็น clock ของตัวถัดไป

## ตัวอย่างที่ 3: ออกแบบ mod-6 counter ด้วย synchronous excitation

mod-6 counter ต้องมีสถานะใช้งาน 6 สถานะ คือ `000` ถึง `101` แล้วกลับ `000`

กำหนดให้สถานะ `110` และ `111` เป็นสถานะไม่ต้องการ (unused states) และต้องกลับ `000` เพื่อให้วงจร self-start ได้

| Present Q2Q1Q0 | Next Q2Q1Q0 | D2 | D1 | D0 |
|:---:|:---:|:---:|:---:|:---:|
| 000 | 001 | 0 | 0 | 1 |
| 001 | 010 | 0 | 1 | 0 |
| 010 | 011 | 0 | 1 | 1 |
| 011 | 100 | 1 | 0 | 0 |
| 100 | 101 | 1 | 0 | 1 |
| 101 | 000 | 0 | 0 | 0 |
| 110 | 000 | 0 | 0 | 0 |
| 111 | 000 | 0 | 0 | 0 |

ถ้าใช้ D flip-flop จะต่อ `D2D1D0` เท่ากับ next state โดยตรง จึงลดรูปสมการจากตารางได้

K-map สำหรับ `D0` มีค่า 1 ที่ present state `000`, `010`, `100`

```text
D0 K-map
       Q1Q0
       00 01 11 10
Q2=0   1  0  0  1
Q2=1   1  0  0  0

กลุ่มที่เลือก:
1) Q1=0,Q0=0 ครอบ 000 และ 100 ได้เทอม Q1'Q0'
2) Q2=0,Q0=0 ครอบ 000 และ 010 ได้เทอม Q2'Q0'

D0 = Q0'(Q1' + Q2')
```

K-map สำหรับ `D1` มีค่า 1 ที่ present state `001`, `010`

```text
D1 K-map
       Q1Q0
       00 01 11 10
Q2=0   0  1  0  1
Q2=1   0  0  0  0

D1 = Q2'Q1'Q0 + Q2'Q1Q0'
   = Q2'(Q1 XOR Q0)
```

K-map สำหรับ `D2` มีค่า 1 ที่ present state `011`, `100`

```text
D2 K-map
       Q1Q0
       00 01 11 10
Q2=0   0  0  1  0
Q2=1   1  0  0  0

D2 = Q2'Q1Q0 + Q2Q1'Q0'
```

สมการ synchronous mod-6 ด้วย D flip-flop คือ

```text
D0 = Q0'(Q1' + Q2')
D1 = Q2'(Q1 XOR Q0)
D2 = Q2'Q1Q0 + Q2Q1'Q0'
```

ในใบงานนี้จะให้ทำ mod-6 แบบ decode reset เป็นวิธีหลัก เพราะต่อเร็วและเห็นปัญหา timing ชัดเจน จากนั้นให้เปรียบเทียบกับสมการ synchronous ข้างต้นในคำถามท้ายแลป

## ส่วนที่ 2: สร้าง ripple counter 3 บิต (35 นาที)

ภารกิจของส่วนนี้คือสร้างเคาน์เตอร์อะซิงโครนัสที่นับ `0` ถึง `7` แล้วกลับ `0`

ใช้ T flip-flop ได้โดยตรง หรือใช้ JK flip-flop โดยต่อ `J=1` และ `K=1` เพื่อให้ทำงานเหมือน T flip-flop

### วิธี A: ใช้ T flip-flop

1. เข้า subcircuit `ripple_3bit`
2. ไปที่ **Memory > T Flip-Flop**
3. วาง T flip-flop 3 ตัว เรียกเป็น `FF0`, `FF1`, `FF2`
4. ตั้ง label output ของ `FF0` เป็น `Q0`
5. ตั้ง label output ของ `FF1` เป็น `Q1`
6. ตั้ง label output ของ `FF2` เป็น `Q2`
7. ต่อ **Constant 1** เข้าขา `T` ของ flip-flop ทั้ง 3 ตัว
8. ตั้ง trigger ของ flip-flop เป็น falling edge ถ้าต้องการให้วงจรนับขึ้นด้วยการต่อ `Q` ไป clock ตัวถัดไป
9. ต่อ `CLK` เข้าขา clock ของ `FF0`
10. ต่อ `Q0` เข้าขา clock ของ `FF1`
11. ต่อ `Q1` เข้าขา clock ของ `FF2`
12. ต่อ `RESET` เข้าขา clear/reset ของ flip-flop ทั้ง 3 ตัว
13. ถ้า clear เป็น active-low ให้ใส่ NOT gate ตามความจำเป็น และบันทึกไว้ในรายงาน
14. วาง LED หรือ probe ที่ `Q0`, `Q1`, `Q2`
15. ต่อ `Q2Q1Q0` เข้า Hex Digit Display ตามวิธีในส่วนที่ 1

```text
             ┌─────────┐       ┌─────────┐       ┌─────────┐
CLK ────────►│ CLK FF0 │ Q0 ──►│ CLK FF1 │ Q1 ──►│ CLK FF2 │
1 ──────────►│ T       │       │ T       │       │ T       │
RESET ──────►│ CLR     │       │ CLR     │       │ CLR     │
             └────┬────┘       └────┬────┘       └────┬────┘
                  Q0                Q1                Q2
```

### วิธี B: ใช้ JK flip-flop

1. ถ้าใช้ **Memory > J-K Flip-Flop** ให้ต่อ `J=1` และ `K=1` ทุกตัว
2. การต่อ clock เหมือนวิธี A
3. ตรวจว่าเมื่อ `J=K=1` flip-flop toggle ทุกครั้งที่ได้รับขอบ clock
4. บันทึกในรายงานว่าใช้ T flip-flop หรือ JK flip-flop

### การทดสอบ ripple counter

1. กด `RESET` เพื่อให้ได้สถานะ `000`
2. ปล่อย `RESET`
3. กด **Simulate > Tick Once** 8 ครั้ง
4. อ่าน `Q2Q1Q0` หลังวงจรนิ่งแล้วทุกครั้ง
5. บันทึกเลขที่ Hex display แสดง
6. ซูมเข้าใกล้ช่วงเปลี่ยน `011 -> 100`
7. ถ้าใช้ timing diagram หรือ probe ที่เห็น delay ให้บันทึกภาพช่วงนี้
8. ถ้ามอง delay ไม่ชัด ให้เพิ่ม propagation delay ของ flip-flop หรือใช้ tick ช้าและสังเกตการเปลี่ยนผ่านของ probe

## ส่วนที่ 3: สร้าง synchronous counter 3 บิต (35 นาที)

ภารกิจของส่วนนี้คือสร้างเคาน์เตอร์ที่นับเหมือน ripple counter แต่ flip-flop ทุกตัวรับ clock เดียวกัน

สมการที่ใช้คือ

```text
T0 = 1
T1 = Q0
T2 = Q1 · Q0
```

### ขั้นตอนต่อวงจรใน Logisim

1. เข้า subcircuit `sync_3bit`
2. วาง T flip-flop 3 ตัว ชื่อ `FF0`, `FF1`, `FF2`
3. ต่อ clock หลัก `CLK` เข้าขา clock ของ flip-flop ทั้ง 3 ตัวพร้อมกัน
4. ต่อ `RESET` เข้าขา clear/reset ของ flip-flop ทั้ง 3 ตัว
5. ต่อ **Constant 1** เข้าขา `T` ของ `FF0`
6. ต่อ `Q0` เข้าขา `T` ของ `FF1`
7. วาง **AND gate** แบบ 2 input
8. ต่อ `Q1` และ `Q0` เข้า AND gate
9. ต่อ output ของ AND gate เข้าขา `T` ของ `FF2`
10. วาง LED หรือ probe ที่ `Q2`, `Q1`, `Q0`
11. ต่อ Hex Digit Display แบบเดียวกับ ripple counter
12. จัดสาย clock เป็นเส้นแนวนอนเส้นเดียวแล้วแตกสาขาลง flip-flop ทุกตัว เพื่อให้เห็นว่าเป็น synchronous จริง

```text
CLK ─────────┬──────────────┬──────────────┐
             ▼              ▼              ▼
          ┌─────┐        ┌─────┐        ┌─────┐
1 ───────►│T FF0│ Q0 ───►│T FF1│ Q1 ─┐  │T FF2│── Q2
          └──┬──┘        └──┬──┘     │  └──┬──┘
             │              │        ▼     ▲
             Q0             Q1     [AND]───┘
                                      ▲
                                      Q0
```

### การทดสอบ synchronous counter

1. กด `RESET` ให้สถานะเริ่มที่ `000`
2. ปล่อย `RESET`
3. เดิน clock 8 ครั้งด้วย **Tick Once**
4. บันทึกสถานะหลังขอบ clock แต่ละครั้ง
5. เปรียบเทียบกับ ripple counter ใน tick เดียวกัน
6. สังเกตช่วง `011 -> 100` ว่า output เปลี่ยนหลัง clock เดียวกัน ไม่ได้ส่ง clock ต่อกันเป็นทอด
7. บันทึกภาพวงจรที่เห็นว่า `CLK` ไปถึง flip-flop ทั้ง 3 ตัวโดยตรง

## ส่วนที่ 4: สร้าง mod-6 counter ด้วย decode reset (40 นาที)

ภารกิจของส่วนนี้คือทำให้เคาน์เตอร์นับ `0,1,2,3,4,5` แล้วกลับ `0`

วิธีหลักในแลปนี้คือใช้เคาน์เตอร์ 3 บิต แล้ว decode สถานะ `110` ซึ่งเป็นเลข 6 เพื่อสั่ง reset กลับ `000`

ลำดับที่ต้องการคือ

```text
000 -> 001 -> 010 -> 011 -> 100 -> 101 -> 000 -> ...
 0      1      2      3      4      5      0
```

สถานะที่ใช้ trigger reset คือ

```text
Q2Q1Q0 = 110
Reset_condition = Q2 · Q1 · Q0'
```

### ขั้นตอนต่อวงจรแบบ decode reset

1. เข้า subcircuit `mod6_decode_reset`
2. คัดลอกแนวคิดจาก `ripple_3bit` หรือสร้าง T flip-flop 3 ตัวใหม่
3. ต่อให้เป็น binary counter 3 บิตก่อน และทดสอบให้สามารถนับ 0 ถึง 7 ได้
4. วาง **NOT gate** 1 ตัว เพื่อสร้าง `Q0_NOT`
5. ต่อ `Q0` เข้า NOT gate
6. วาง **AND gate** 3 input หรือใช้ AND 2 input ต่อเป็นสองชั้น
7. ต่อ `Q2`, `Q1`, และ `Q0_NOT` เข้า AND gate
8. ตั้งชื่อ output ของ AND gate ว่า `RESET6`
9. ต่อ `RESET6` เข้าขา clear/reset ของ flip-flop ทั้ง 3 ตัว
10. ถ้าขา clear ของ flip-flop เป็น active-low ให้ต่อ `RESET6` ผ่าน NOT gate ก่อนเข้าขา clear
11. วาง LED หรือ probe เพิ่มที่สัญญาณ `RESET6`
12. ต่อ `Q2Q1Q0` เข้า Hex Digit Display
13. กด reset เริ่มต้นให้สถานะเป็น `000`
14. เดิน clock ทีละจังหวะและบันทึกว่าสถานะ `110` ปรากฏค้างหรือถูกล้างเร็วมาก

```text
Q2 ─────────────┐
                ▼
Q1 ───────────► AND ───── RESET6 ─────► CLR ของ FF0, FF1, FF2
                ▲
Q0 ─► NOT ──────┘

RESET6 = Q2Q1Q0'
```

### จุดที่ต้องสังเกตเป็นพิเศษ

ถ้าสร้าง mod-6 จาก ripple counter การเปลี่ยนจาก `101` ไป `110` อาจไม่ได้เกิดพร้อมกันครบทุกบิต สัญญาณ decode `Q2Q1Q0'` อาจเป็น pulse สั้นมาก และอาจมี glitch จากสถานะชั่วคราว

ใน Logisim pulse นี้มักพอเห็นจาก probe `RESET6` ถ้าเปิด clock ช้า หรือดูด้วย timing diagram

ถ้าสร้าง mod-6 จาก synchronous counter แล้วนำ `RESET6` ไป asynchronous clear ก็ยังต้องระวัง glitch จาก decoder แต่ delay จาก chain clock จะน้อยกว่า ripple

## ส่วนที่ 5: ทดลองสถานะผิดปกติของ mod-6 (15 นาที)

เคาน์เตอร์ที่ดีควรกลับเข้าสู่ลำดับนับที่ถูกต้องได้ แม้เริ่มจากสถานะที่ไม่ได้ใช้งาน

ใน mod-6 นี้สถานะ `110` และ `111` ไม่อยู่ในลำดับปกติ

### วิธีทดสอบใน Logisim

1. หยุด clock อัตโนมัติถ้าเปิดอยู่
2. ใช้ poke tool ตั้งค่า flip-flop หรือใช้ preset/clear ให้ได้สถานะ `110`
3. สังเกตว่า `RESET6` ทำงานทันทีหรือไม่
4. บันทึกว่าวงจรกลับ `000` โดยไม่ต้องรอ clock หรือรอ clock ถัดไป
5. ตั้งค่าเริ่มต้นเป็น `111`
6. เดิน clock 1 ถึง 2 ครั้ง
7. บันทึกว่าวงจรกลับเข้าลำดับ `000 -> 001 -> ...` ได้หรือค้างสถานะผิด

### ตารางทดสอบสถานะผิดปกติ

| สถานะเริ่มต้น | Expected behavior | Actual behavior | ต้องใช้กี่ clock จึงกลับลำดับปกติ | หมายเหตุ |
|:---:|---|---|---:|---|
| 110 | reset เป็น 000 เพราะ decode ตรง `Q2Q1Q0'` | | | |
| 111 | ไม่ถูก decode โดยสูตร `Q2Q1Q0'`; ต้องดู clock ถัดไป | | | |

ถ้าเริ่มที่ `111` แล้วใช้ ripple counter แบบ falling-edge, tick ถัดไปอาจทำให้ผ่าน `000` หรือเกิด reset ตามการต่อ clock และ clear ที่เลือก ให้บันทึกตามผลจริง ไม่ต้องแก้ให้ตรงกับตารางถ้าวงจรของกลุ่มใช้ trigger ต่างจากที่กำหนด แต่ต้องอธิบายได้

## สรุปผลที่ต้องวิเคราะห์ในรายงาน

ให้เขียนวิเคราะห์เป็นย่อหน้า ไม่ใช่ตอบเพียงว่า "ทำได้" หรือ "ทำไม่ได้"

1. ripple counter และ synchronous counter มีลำดับเลขสุดท้ายเหมือนกันหรือไม่
2. ช่วง tick ใดของ ripple counter เห็น delay ชัดที่สุด และเพราะเหตุใด
3. สัญญาณ `RESET6` ของ mod-6 เป็นระดับค้างหรือ pulse สั้น
4. ถ้าใช้ output ของ ripple counter ไปขับ decoder หรือ display โดยตรง อาจเห็นเลขผิดชั่วคราวช่วงใด
5. วงจร mod-6 ของกลุ่มเริ่มจาก `110` และ `111` แล้วกลับลำดับปกติอย่างไร
6. ถ้าทำ 7490 ให้เปรียบเทียบการใช้ไอซีสำเร็จรูปกับการสร้างจาก flip-flop ใน Logisim

## คำถามท้ายใบงาน

1. อธิบายความแตกต่างระหว่าง asynchronous counter และ synchronous counter โดยอ้างอิงจากวงจร `ripple_3bit` และ `sync_3bit` ที่สร้างจริง
2. ใน ripple counter ช่วงเปลี่ยนจาก `011` เป็น `100` มี flip-flop กี่ตัวที่ต้องเปลี่ยน และเหตุใดจึงเกิด ripple delay มากกว่าช่วง `010` เป็น `011`
3. ถ้าต้องการ mod-10 counter จากเคาน์เตอร์ 4 บิต ควร decode สถานะใดเพื่อ reset และเขียนสมการ reset condition
4. วงจร mod-6 แบบ decode `110` มีโอกาสเกิด glitch อย่างไรเมื่อสร้างจาก ripple counter และ glitch นี้อาจกระทบอุปกรณ์ปลายทางใดบ้าง
5. ถ้าใช้ synchronous mod-6 ด้วย D flip-flop ตามสมการในตัวอย่างที่ 3 ข้อดีเมื่อเทียบกับ asynchronous clear คืออะไร
6. ทำไมการต่อ input reset/preset ของไอซี 74xx ค้างลอยจึงทำให้ counter ทำงานผิดลำดับได้
7. ถ้า Hex display แสดงเลข `6` สั้น ๆ ใน mod-6 ทั้งที่ expected sequence คือ `0-5` ให้ตีความว่าเป็นความผิดทางตรรกะหรือเป็นผลจาก delay จงอธิบายเงื่อนไขที่แยกสองกรณีนี้
8. ใน CPU หรือไมโครคอนโทรลเลอร์ วงจรใดมีพฤติกรรมคล้าย counter และเหตุใดวงจรนั้นควรเป็น synchronous มากกว่า ripple

| Synchronous counter 3 บิต | 20 | ใช้ clock เดียวกันทุก flip-flop ต่อ `T0=1`, `T1=Q0`, `T2=Q1Q0` ถูกต้อง และเปรียบเทียบ timing กับ ripple ได้ |
| Mod-6 counter | 25 | นับ `0-5-0` ได้จริง มีสัญญาณ decode/reset ชัดเจน ทดสอบสถานะ `110` และ `111` พร้อมวิเคราะห์ glitch |
| ตารางและ timing diagram | 15 | กรอกตาราง ripple, synchronous, mod-6 ครบ วาดหรือเติม timing diagram อ่านได้ และมีภาพวงจรประกอบ |
| คำถามท้ายใบงาน | 10 | ตอบโดยอ้างอิงผลทดลองของตนเอง ไม่ตอบเป็นนิยามทั่วไป |
| ความเป็นระเบียบและวินัยการต่อวงจร | 10 | ตั้งชื่อสัญญาณชัดเจน ไม่ปล่อย input สำคัญลอย แยก subcircuit ตามที่กำหนด ส่งไฟล์และรายงานครบ |

คะแนนเสริมสูงสุด 5 คะแนนสำหรับกลุ่มที่ทดลองไอซี 7490 ได้จริง พร้อมภาพวงจร breadboard ตารางนับ `0-9` และคำอธิบายปัญหา contact bounce หรือ reset pin
