## โจทย์หลักของแลป

ให้ลดรูปฟังก์ชัน 4 ตัวแปรต่อไปนี้ด้วยวิธีตาราง Quine-McCluskey

```text
F(A,B,C,D) = Σm(0,1,2,5,6,7,8,9,10,14)
```

ลำดับบิตในแลปนี้กำหนดตายตัวเป็น `A B C D`

การตรวจวงจรต้องเทียบกับตารางความจริงเดิม ไม่ใช่เทียบเฉพาะกับสมการที่ลดรูปแล้ว
เพราะสมการที่ลดรูปผิดอาจยังต่อวงจรได้ถูกตามสมการผิดนั้น

## ทฤษฎีย่อเฉพาะวิธีตาราง

วิธี Quine-McCluskey เป็นวิธีลดรูปฟังก์ชันบูลีนแบบเป็นขั้นตอน
ต่างจาก K-map ที่อาศัยการมองกลุ่มช่องที่ติดกันบนแผนภาพ
วิธีตารางจะเปรียบเทียบเทอมด้วยเลขฐานสองและรวมเทอมที่ต่างกันเพียง 1 บิต

กฎการรวมที่ใช้ตลอดแลปคือ

```text
0000 + 0001 = 000-
```

เครื่องหมาย `-` หมายถึงตัวแปรตำแหน่งนั้นถูกตัดออกจากพจน์
ถ้า pattern คือ `000-` และตัวแปรคือ `A B C D`
พจน์ที่ได้คือ `A'B'C'`

```text
pattern: 0 0 0 -
ตัวแปร:  A B C D
พจน์:    A'B'C'
```

การรวมรอบถัดไปต้องระวังตำแหน่ง dash ให้ตรงกัน
ตัวอย่างที่รวมได้และรวมไม่ได้มีดังนี้

```text
000- กับ 100-  รวมได้  -> -00-
00-0 กับ 10-0  รวมได้  -> -0-0
000- กับ 00-0  รวมไม่ได้ เพราะ dash อยู่คนละตำแหน่ง
0-01 กับ 1-10  รวมไม่ได้ เพราะต่างกันมากกว่า 1 ตำแหน่ง
```

ศัพท์ที่ต้องใช้ในรายงานมีความหมายเฉพาะดังนี้

| คำ | ความหมายในแลปนี้ | ตัวอย่างจากโจทย์ |
|---|---|---|
| Minterm | แถวอินพุตที่ F ต้องมีค่า 1 | `m14 = 1110` |
| Implicant | พจน์ที่ครอบคลุมมินเทอมของ F อย่างน้อย 1 ตัว | `0-01` ครอบคลุม `m1,m5` |
| Prime implicant | Implicant ที่รวมให้ใหญ่กว่านี้ไม่ได้แล้ว | `-00-` |
| Essential PI | PI ที่ครอบคลุมมินเทอมบางตัวเพียงแถวเดียวใน PI chart | ถ้า `m14` มีเฉพาะ `--10` ครอบคลุม |
| PI chart | ตารางแถวเป็น PI คอลัมน์เป็นมินเทอมของ F | ใช้เลือกคำตอบสุดท้าย |
| Minimal SOP | ผลรวมของพจน์ AND ที่เลือกแล้วครอบคลุมมินเทอมครบ | `B'C' + CD' + A'BD` |

### 3.3 หา essential prime implicant

ดูทีละคอลัมน์
ถ้าคอลัมน์ใดมี `x` เพียงตัวเดียว PI แถวนั้นต้องถูกเลือก

```text
m9  มีเฉพาะ -00- ครอบคลุม  -> -00- เป็น EPI
m14 มีเฉพาะ --10 ครอบคลุม  -> --10 เป็น EPI
```

เลือก `-00-` แล้วครอบคลุม `0,1,8,9`
เลือก `--10` แล้วครอบคลุม `2,6,10,14`

มินเทอมที่ยังเหลือหลังเลือก EPI คือ

```text
m5 และ m7
```

PI ที่ครอบคลุมทั้ง `m5` และ `m7` พร้อมกันคือ `01-1`
จึงเลือกเพิ่มอีกหนึ่งพจน์

```text
ชุด PI สุดท้าย = { -00-, --10, 01-1 }
```

### 3.4 แปลง pattern เป็นสมการ

```text
-00-  = B'C'
--10  = CD'
01-1  = A'BD
```

ดังนั้นคำตอบ minimal SOP คือ

```text
F(A,B,C,D) = B'C' + CD' + A'BD
```

ตารางสรุปคำตอบสำหรับรายงาน

| PI ที่เลือก | มินเทอมที่ครอบคลุม | พจน์บูลีน | เหตุผลที่เลือก |
|---|---|---|---|
| `-00-` | 0,1,8,9 | `B'C'` | EPI จากคอลัมน์ `m9` |
| `--10` | 2,6,10,14 | `CD'` | EPI จากคอลัมน์ `m14` |
| `01-1` | 5,7 | `A'BD` | ครอบคลุมมินเทอมที่เหลือทั้งสองตัว |

## ส่วนที่ 4: สร้างวงจร minimal SOP ใน Logisim Evolution (45 นาที)

วงจรที่จะสร้างต้องตรงกับสมการ

```text
F = B'C' + CD' + A'BD
```

### 4.1 ตั้งค่าไฟล์และอินพุต

1. เปิด Logisim Evolution
2. เลือก `File > New`
3. บันทึกไฟล์เป็น `lab06-tabulation-qm.circ`
4. เลือกเครื่องมือ `Input Pin`
5. วาง input pin 4 ตัวทางซ้ายของพื้นที่วงจร
6. ตั้งชื่อ pin จากบนลงล่างเป็น `A`, `B`, `C`, `D`
7. ตั้งค่า `Data Bits = 1` และ `Facing = East` ทุก pin
8. เลือกเครื่องมือ `Output Pin`
9. วาง output pin ทางขวาและตั้งชื่อ `F_QM`

ผังการไหลของสัญญาณควรเป็นแบบนี้

```text
A ─┬──────────────┐
   │              │
B ─┼─ NOT ─ B' ─┐ │
   │            │ │
C ─┼─ NOT ─ C' ─┼─ AND(B'C') ─┐
   │            │             │
D ─┼─ NOT ─ D' ─┼─ AND(CD')  ─┼─ OR ─ F_QM
   │            │             │
A'─┘            └─ AND(A'BD) ─┘
```

### 4.2 วาง NOT gate

1. เลือกหมวด `Gates`
2. วาง `NOT Gate` 4 ตัว
3. ต่อ `A` เข้า NOT ตัวที่ 1 เพื่อสร้าง `A'`
4. ต่อ `B` เข้า NOT ตัวที่ 2 เพื่อสร้าง `B'`
5. ต่อ `C` เข้า NOT ตัวที่ 3 เพื่อสร้าง `C'`
6. ต่อ `D` เข้า NOT ตัวที่ 4 เพื่อสร้าง `D'`
7. ใช้ `Text Tool` ใส่ label ที่สาย output ของ NOT เป็น `A'`, `B'`, `C'`, `D'`

แม้สมการสุดท้ายใช้ `A'`, `B'`, `C'`, `D'` ไม่ครบทุกเส้นในทุกพจน์
ให้ติด label ให้ครบเพื่ออ่านวงจรง่ายและลดความผิดพลาดตอนต่อสาย

### 4.3 สร้างพจน์ AND ทั้งสาม

1. วาง `AND Gate` ตัวที่ 1
2. ตั้ง `Number of Inputs = 2`
3. ต่อ `B'` และ `C'` เข้ากับ AND ตัวที่ 1
4. ใส่ label output ว่า `P1 = B'C'`
5. วาง `AND Gate` ตัวที่ 2
6. ตั้ง `Number of Inputs = 2`
7. ต่อ `C` และ `D'` เข้ากับ AND ตัวที่ 2
8. ใส่ label output ว่า `P2 = CD'`
9. วาง `AND Gate` ตัวที่ 3
10. ตั้ง `Number of Inputs = 3`
11. ต่อ `A'`, `B`, และ `D` เข้ากับ AND ตัวที่ 3
12. ใส่ label output ว่า `P3 = A'BD`

ถ้า AND ตัวที่ 3 ยังมีเพียง 2 input ให้คลิก gate แล้วปรับ property `Number of Inputs`
อย่าต่อ AND 2 input สองตัวโดยไม่ตั้งชื่อสายกลาง
เพราะจะตรวจยากว่าผิดที่พจน์ `A'B` หรือผิดที่การ AND กับ `D`

### 4.4 รวมด้วย OR gate

1. วาง `OR Gate` 1 ตัวทางขวาของ AND ทั้งสาม
2. ตั้ง `Number of Inputs = 3`
3. ต่อ `P1`, `P2`, `P3` เข้า OR gate
4. ต่อ output ของ OR gate ไปยัง output pin `F_QM`
5. เพิ่ม `Probe` ที่สาย `P1`, `P2`, `P3`, และ `F_QM`

โครงวงจรหลังต่อเสร็จควรอ่านได้ดังนี้

```text
        B' ─────┐
                AND ─ P1 ─┐
        C' ─────┘         │
                          │
        C  ─────┐         │
                AND ─ P2 ─┼── OR ── F_QM
        D' ─────┘         │
                          │
        A' ─────┐         │
        B  ─────┼─ AND ─ P3
        D  ─────┘
```

## ส่วนที่ 6: เปรียบเทียบกับ K-map ของฟังก์ชันเดียวกัน (25 นาที)

ใช้ลำดับ Gray code เหมือนบท K-map
แถวคือ `AB = 00, 01, 11, 10`
คอลัมน์คือ `CD = 00, 01, 11, 10`

### 6.1 K-map ของโจทย์

```text
              CD
             00  01  11  10
          +---+---+---+---+
AB = 00   | 1 | 1 | 0 | 1 |
          +---+---+---+---+
AB = 01   | 0 | 1 | 1 | 1 |
          +---+---+---+---+
AB = 11   | 0 | 0 | 0 | 1 |
          +---+---+---+---+
AB = 10   | 1 | 1 | 0 | 1 |
          +---+---+---+---+
```

กลุ่มที่เลือกได้จาก K-map คือ

```text
กลุ่มที่ 1: m0, m1, m8, m9       -> B'C'
กลุ่มที่ 2: m2, m6, m10, m14     -> CD'
กลุ่มที่ 3: m5, m7               -> A'BD
```

คำตอบจาก K-map จึงเป็น

```text
F = B'C' + CD' + A'BD
```

ซึ่งตรงกับวิธีตาราง

### 6.2 ตารางเปรียบเทียบวิธีตารางกับ K-map

| ประเด็น | วิธีตาราง Quine-McCluskey | K-map |
|---|---|---|
| หลักการเลือกกลุ่ม | ใช้การเปรียบเทียบเลขฐานสองและ PI chart | ใช้การมองช่องติดกันแบบ Gray code |
| จุดที่ผิดง่ายในโจทย์นี้ | ลืม PI ที่ไม่ได้รวมต่อ เช่น `01-1` | มองข้ามกลุ่มสองช่อง `m5,m7` |
| หลักฐานในรายงาน | ตารางรวมรอบที่ 1, รอบที่ 2, PI chart | แผนภาพ K-map พร้อมวงกลุ่ม |
| เหมาะเมื่อจำนวนตัวแปรเพิ่ม | เหมาะกว่า เพราะทำเป็นขั้นตอนและเขียนโปรแกรมได้ | ยากขึ้นเมื่อมากกว่า 4 ตัวแปร |
| คำตอบของโจทย์นี้ | `B'C' + CD' + A'BD` | `B'C' + CD' + A'BD` |

## ส่วนเสริม: ต่อวงจรจริงด้วยไอซี 74xx

ส่วนนี้ใช้เมื่อผู้สอนให้ทดลองบน breadboard
ก่อนจ่ายไฟต้องตรวจ datasheet ของไอซีรุ่นที่ใช้จริง
ตัวอย่างนี้อ้างอิงไอซีตระกูล 74LS แบบ DIP-14 ทั่วไป

### อุปกรณ์ที่ใช้

| อุปกรณ์ | จำนวน | ใช้ทำอะไร |
|---|:---:|---|
| 74LS04 | 1 | สร้าง `A'`, `B'`, `C'`, `D'` |
| 74LS08 | 1 | AND สำหรับ `B'C'`, `CD'`, `A'B`, `A'BD` |
| 74LS32 | 1 | OR รวม `P1 + P2 + P3` |
| DIP switch 4 ช่อง | 1 | อินพุต `A,B,C,D` |
| LED + R 330 Ω | 1 ชุด | แสดงผล `F` |
| R pull-down หรือ pull-up | ตามวงจร | ไม่ปล่อยอินพุตลอย |

### การกำหนดขา 74LS04

```text
74LS04: pin 14 = VCC, pin 7 = GND

A  -> pin 1   ได้ A' จาก pin 2
B  -> pin 3   ได้ B' จาก pin 4
C  -> pin 5   ได้ C' จาก pin 6
D  -> pin 13  ได้ D' จาก pin 12
```

### การกำหนดขา 74LS08

```text
74LS08: pin 14 = VCC, pin 7 = GND

P1 = B'C'
B' -> pin 1
C' -> pin 2
P1 <- pin 3

P2 = CD'
C  -> pin 4
D' -> pin 5
P2 <- pin 6

T  = A'B
A' -> pin 9
B  -> pin 10
T  <- pin 8

P3 = A'BD
T  -> pin 12
D  -> pin 13
P3 <- pin 11
```

### การกำหนดขา 74LS32

```text
74LS32: pin 14 = VCC, pin 7 = GND

S1 = P1 + P2
P1 -> pin 1
P2 -> pin 2
S1 <- pin 3

F = S1 + P3
S1 -> pin 4
P3 -> pin 5
F  <- pin 6
```

## คำถามท้ายใบงาน

1. เหตุใดวิธีตารางจึงต้องจัดกลุ่มมินเทอมตามจำนวนเลข 1 ก่อนจับคู่
2. ในโจทย์นี้ `-00-` เป็น essential prime implicant เพราะมินเทอมใด และอธิบายจาก PI chart
3. ถ้าเลือก `0-01` แทน `01-1` หลังเลือก EPI แล้ว จะยังครอบคลุมมินเทอมทั้งหมดหรือไม่ เพราะเหตุใด
4. อธิบายความแตกต่างระหว่าง prime implicant กับ essential prime implicant โดยใช้ PI จากโจทย์นี้อย่างน้อย 2 ตัวอย่าง
5. เมื่อจำนวนตัวแปรเพิ่มเป็น 5 หรือ 6 ตัว วิธีตารางเหนือกว่า K-map ในแง่ใด และยังมีข้อเสียอะไรบ้าง
6. ถ้าในโจทย์มี don't care เพิ่ม เช่น `d(3,11,15)` ขั้นตอนใดของวิธีตารางจะเปลี่ยน และ PI chart ต้องใส่คอลัมน์ don't care หรือไม่
7. วงจร `F = B'C' + CD' + A'BD` มีพจน์ใดทำให้ `m14` เป็น 1 และถ้าสาย `D'` ต่อผิดจะเห็นความผิดพลาดที่มินเทอมใดบ้าง
8. เปรียบเทียบจำนวนพจน์และจำนวน literal ของสมการ canonical SOP เดิมกับ minimal SOP ที่ได้จากวิธีตาราง
